# Phase 5 상세 — 라이브 대시보드

> 팝업이 오픈된 이후 전시·팝업 성과를 실시간으로 모니터링하는 단계입니다. 운영자가 현장에서 즉각적인 의사결정을 내릴 수 있는 지표 중심 화면입니다.

---

## 1. Phase 목표

전시 기간 중 및 팝업 운영 기간 동안 **성과 지표를 실시간으로 파악**하고, 필요 시 즉각 대응할 수 있는 근거를 제공합니다. 최종적으로 사내 보고 및 파트너(기획사) 공유용 성과 증빙 리포트를 자동 생성합니다.

---

## 2. 대시보드 화면 구성

### 상단 KPI 카드 (항상 노출)

| KPI | 설명 | 업데이트 주기 |
| --- | --- | --- |
| **오늘 관람객** | 당일 QR 스캔 수 | 실시간 |
| **누적 관람객** | 전시 오픈 이후 총계 | 실시간 |
| **AI 큐레이터 활성율** | Level 2 진입 비율 (목표: 30%) | 1시간 |
| **초대장 발송 수** | Bridge 트리거 발동 총계 | 실시간 |
| **초대장 CTR** | 발송 대비 클릭율 (목표: 20%) | 1시간 |
| **팝업 방문 수** | 스핀오프 랜딩 페이지 방문 | 실시간 |
| **굿즈 결제 수** | 팝업 내 결제 완료 수 | 실시간 |
| **누적 매출** | 팝업 총 매출 (원) | 실시간 |

---

### 탭 1. 전시 성과

- **시간대별 방문자 추이** 차트 (일별·주별 토글)
- **작품별 반응 히트맵** (Phase 3 데이터 누적 버전)
- **감정 키워드 트렌드** (전시 기간 전체 흐름)
- **세그먼트 분포 파이차트** (반응 유형별 비율)
---

### 탭 2. 팝업 전환 퍼널

관람객이 팝업으로 이동하는 전환 흐름을 단계별로 시각화합니다.

```javascript
전체 관람객 (QR 스캔)
    ↓  AI 큐레이터 활성율
Level 2 진입 관람객
    ↓  Bridge 트리거 발동율
초대장 수신 관람객
    ↓  초대장 CTR
팝업 랜딩 방문
    ↓  구매 전환율
굿즈 결제 완료
```

- 각 단계별 이탈율 표시
- 목표치 대비 실제 수치 비교 (목표 라인 표시)
- 단계별 개선 제안 AI 코멘트 (예: "Level 2 진입율이 낮습니다. 현장 QR 위치를 작품 정면으로 이동을 권장합니다.")
---

### 탭 3. 매출 & RS 정산

| 항목 | 내용 |
| --- | --- |
| **일별 매출 추이** | 팝업 운영 기간 일별 매출 차트 |
| **굿즈별 판매 현황** | MD 아이템별 판매 수량·금액 |
| **객단가 추이** | 일별 평균 구매 금액 |
| **RS 정산 예상액** | 현재까지 누적 매출 기반 RS 배분 예상 금액 (고객사·AFTERGLOW 비율) |
| **목표 대비 달성율** | Phase 4에서 설정한 예상 수익 시뮬 대비 실제 달성율 |

---

### 탭 4. 성과 증빙 리포트 출력

운영자가 버튼 하나로 보고용 리포트를 자동 생성합니다.

**포함 내용:**

- 전시 기본 정보 (전시명·기간·장소)
- 핵심 KPI 요약 (관람객 수·전환율·매출)
- 작품별 반응 히트맵 이미지
- 전환 퍼널 차트
- 세그먼트 분석 요약
- RS 정산 내역
- 차기 전시 제언 (AI 자동 생성 1문단)
**출력 형식:** PDF / PPT 선택 가능

**공유 권한:** Tier 2(고객사 어드민)에게 자동 공유 알림 발송

## 3. 알림 & 이상 감지

| 이벤트 | 알림 조건 | 알림 방식 |
| --- | --- | --- |
| **CTR 급락** | 초대장 CTR이 목표 대비 50% 미만 | 인앱 + 이메일 |
| **매출 목표 달성** | 예상 수익 시뮬 기본 시나리오 달성 | 인앱 축하 알림 |
| **인기 작품 변동** | 히트맵 1위 작품이 변경될 때 | 인앱 |
| **팝업 종료 D-3** | 팝업 종료 3일 전 | 이메일 |
| **데이터 이상** | 수집 데이터 급감 (기술 오류 의심) | 인앱 + 이메일 긴급 |

## 4. 검토 의견 & 보강 제안

**1. 실시간 vs 배치 처리 범위 명확화**

전체를 실시간으로 처리하면 비용이 높아집니다. KPI 카드 일부와 방문자 수는 실시간, 히트맵·세그먼트 분석 등 무거운 집계는 1시간 배치 처리로 구분하고 "마지막 업데이트 시각"을 명시하는 것이 현실적입니다.

**2. 모바일 최적화 필요**

운영자가 현장에서 스마트폰으로 확인하는 경우가 많습니다. 탭 1의 KPI 카드와 퍼널 차트는 모바일 뷰에서도 한눈에 보이도록 설계해야 합니다.

**3. 데이터 비교 기준 제공**

수치만으로는 "잘 되고 있는 건지" 판단이 어렵습니다. 업계 벤치마크(미디어아트 전시 평균 전환율 등) 또는 이전 전시 대비 비교 기준을 함께 표시하면 운영자 판단에 도움이 됩니다.

**4. Tier 2(고객사) 뷰 별도 고려**

고객사 어드민은 운영자 대시보드의 "읽기 전용" 버전을 볼 수 있어야 합니다. 동일 화면에서 권한에 따라 편집 요소만 숨기는 방식이 효율적입니다.
