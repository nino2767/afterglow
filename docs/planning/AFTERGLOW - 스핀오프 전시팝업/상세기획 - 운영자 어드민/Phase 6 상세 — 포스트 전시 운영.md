# Phase 6 상세 — 포스트 전시 운영

> 전시가 종료된 후에도 팝업이 단독으로 운영되거나, 전시 데이터가 차기 기획에 활용되는 단계입니다. AFTERGLOW의 핵심 가치 중 하나인 "IP 수익 수명 연장"이 완성되는 구간입니다.

---

## 1. Phase 목표

전시 종료 후 두 가지를 동시에 달성합니다.

1. **팝업 단독 운영 (최대 20일)**: 전시를 놓친 신규 방문자 유입 + 전시 관람객 재방문 유도
1. **데이터 자산화**: 이번 전시 데이터를 차기 전시 기획의 근거로 영구 보존
---

## 2. 타임라인 구조

| 구간 | 기간 (예시) | 운영 내용 |
| --- | --- | --- |
| **전시 + 팝업 동시 운영** | D-day ~ D+60일 | 관람 직후 팝업 전환 → 전환율 최대 |
| **팝업 단독 연장 운영** | D+61 ~ D+80일 | 전시 미관람자 신규 유입 + 관람객 재방문 |
| **데이터 마감 & 자산화** | D+80일 이후 | 최종 리포트 생성, 데이터 DB 영구 저장 |

---

## 3. Step별 상세

### Step 1. 팝업 단독 운영 모드 전환

**전시 종료 D-7 시점 자동 알림:**

- "전시 종료 7일 전입니다. 팝업 단독 운영 기간을 설정해주세요."
- 운영 기간 설정 (최대 20일)
- 팝업 단독 마케팅 소재 자동 생성 제안 ("전시는 끝났지만, 여운은 계속됩니다" 카피)
**단독 운영 기간 특이사항:**

- B2C 웹앱 AI 큐레이터는 비활성화 (전시장이 없으므로)
- 팝업 랜딩 페이지만 유지 → 직접 방문·예약 기능으로 전환
- 기존 초대장 미클릭자에게 재발송 캠페인 가능
---

### Step 2. 최종 데이터 자산화 리포트

전시 종료 후 AI가 자동으로 생성하는 최종 리포트입니다.

| 섹션 | 내용 |
| --- | --- |
| **전시 성과 요약** | 총 관람객·AI 활성율·팝업 전환율·총 매출 |
| **관람객 취향 데이터** | 세그먼트별 선호 패턴, 핵심 감정 키워드 TOP 10 |
| **작품별 반응 아카이브** | 작품별 체류·감정·AI 대화 데이터 전체 |
| **팝업 성과 분석** | 컨셉별 전환율·객단가·MD 판매 순위 |
| **RS 최종 정산** | 전체 매출 기반 최종 RS 정산 내역 |
| **차기 전시 제언** | AI가 이번 데이터를 분석해 자동 생성하는 다음 전시 방향 제안 |

**저장 위치:** AFTERGLOW 플랫폼 누적 DB + 고객사 어드민 영구 보관

---

### Step 3. 유사 전시 비교 분석

AFTERGLOW가 축적한 전시 데이터 DB를 기반으로 비교 분석을 제공합니다.

| 비교 축 | 내용 |
| --- | --- |
| **장르별 비교** | 미디어아트 / 몰입형 / 혼합 장르 평균 대비 이번 전시 위치 |
| **계절·요일별 패턴** | 유사 전시의 계절별 관람객 패턴과 비교 |
| **팝업 전환율 비교** | 같은 컨셉 유형 팝업의 평균 전환율 대비 이번 성과 |
| **MD 적중률 비교** | 유사 세그먼트에서 잘 팔린 굿즈 유형 분석 |

> 💡 이 기능은 AFTERGLOW가 전시 데이터를 많이 축적할수록 정교해집니다. 초기(MVP)에는 자사 데이터가 없으므로, 외부 업계 리포트 기반 벤치마크로 대체하고 "축적 중" 상태로 표시하는 것이 현실적입니다.

---

### Step 4. 차기 전시 기획 연결

| 기능 | 내용 |
| --- | --- |
| **차기 전시 인사이트** | 이번 전시에서 가장 반응이 좋았던 요소를 차기 전시 기획 방향으로 자동 제안 |
| **타겟 세그먼트 보고서** | "다음 전시에서 집중해야 할 관람객 유형" AI 분석 결과 |
| **재계약 제안 자료** | 성과 데이터 + 차기 제안을 묶은 파트너십 갱신 제안서 자동 생성 |

**재계약 제안서 자동 생성:**

- 기획사 담당자에게 이메일 자동 발송 옵션
- 포함 내용: 이번 기획 성과 + AFTERGLOW 기여 분석 + 다음 전시 협력 제안

## 4. 완료 처리 & 프로젝트 아카이브

| 상태 | 조건 | 이후 처리 |
| --- | --- | --- |
| **팝업 종료** | 팝업 운영 기간 종료 | B2C 웹앱 비활성화, 랜딩 페이지 "종료" 표시 |
| **데이터 마감** | 최종 리포트 생성 완료 | 프로젝트 상태 → "완료" 아카이브 |
| **RS 정산 완료** | 고객사 정산 확인 | 고객사 어드민에 정산 완료 표시 |

아카이브된 프로젝트는 읽기 전용으로 영구 보존되며, 유사 전시 비교 분석의 데이터 소스로 활용됩니다.

---

## 5. 검토 의견 & 보강 제안

**1. 팝업 단독 운영 시 B2C 진입 경로 재설계 필요**

전시가 끝나면 QR 스캔 경로가 없어집니다. 팝업 단독 운영 기간에는 SNS 링크·카카오 채널·이전 방문자 재알림 등 별도 진입 경로를 운영자가 설정할 수 있어야 합니다.

**2. 초기 데이터 부족 문제 — 유사 전시 비교 기능**

첫 번째 파트너사와의 파일럿 단계에서는 비교 데이터가 없습니다. "업계 평균" 또는 "목표 대비" 비교로 대체하고, 데이터 축적 후 자동으로 업그레이드되는 구조임을 명시해야 합니다.

**3. 데이터 보존 정책 및 동의 재확인**

B2C 관람객의 개인 행동 데이터를 영구 보존하기 위해서는 최초 동의 범위 내에서만 가능합니다. 온보딩 단계의 consent_scope에 "차기 서비스 개선 목적 보관 동의" 항목 포함 여부를 법무 검토해야 합니다.

**4. 재계약 제안서 타이밍 자동화**

전시 종료 후 자동으로 재계약 제안서를 발송하면 역효과가 날 수 있습니다. 운영자 또는 고객사 담당자가 "발송" 버튼을 직접 누르는 방식으로 설계하고, 자동 초안 생성 + 수동 발송 구조를 권장합니다.
