# Phase 2 상세 — 개별 작품 학습 (AI 세팅)

> Phase 1에서 등록된 전시 자산을 바탕으로 AI 큐레이터의 응답 품질을 결정하는 핵심 세팅 단계입니다. 이 Phase가 탄탄할수록 B2C 관람객의 몰입도와 데이터 품질이 올라갑니다.

---

## 1. Phase 목표

작품별 AI 지식베이스를 구축하여 B2C 웹앱의 AI 큐레이터가 **할루시네이션 없이 정확하고 깊이 있는 해설**을 제공할 수 있는 상태를 만드는 것. 동시에 감정 태그 초기값을 세팅하여 B2C 데이터 수집의 정확도를 높입니다.

---

## 2. Step별 상세

### Step 1. 작품별 에셋 업로드

| 입력 항목 | 타입 | 설명 | 필수 |
| --- | --- | --- | --- |
| **작품 이미지** | JPG/PNG | Vision LLM 인식용 고해상도 이미지 | 필수 |
| **작품 영상 클립** | MP4 / URL | 인터랙티브·미디어아트의 경우 움직임 포함 | 권장 |
| **도슨트 스크립트** | TXT / PDF | RAG 학습의 핵심 소스. 작가 의도·작품 배경 포함 | 필수 |
| **작가 노트** | TXT / PDF | 심층 Q&A 품질 향상용 | 권장 |
| **참고 자료 링크** | URL | 관련 기사·인터뷰·평론 | 선택 |

**업로드 UX 방향:**

- Phase 1에서 등록된 작품 목록이 자동 연동되어 작품 카드 형태로 표시
- 각 작품 카드에 학습 완료 여부 배지 표시 (미완료 / 학습 중 / 완료)
- 드래그앤드롭 멀티 업로드 지원
---

### Step 2. 도슨트 스크립트 입력 & RAG 세팅

**입력 방식 2가지:**

**방법 A — 직접 입력 (텍스트 에디터)**

- 작품별 텍스트 에디터에서 직접 작성 또는 붙여넣기
- AI가 입력 내용을 분석해 Q&A 시나리오 자동 제안 → 운영자 검토·승인
**방법 B — 파일 업로드 (PDF/TXT)**

- 업로드 후 AI가 자동 파싱하여 작품별로 분리·매핑
- 파싱 결과 확인 화면에서 수정 가능
> ⚠️ **MVP 범위**: 텍스트 직접 입력(방법 A) 우선 구현. 파일 업로드 파싱은 v1.0에서 추가.

**RAG 품질 지표 (학습 완료 후 표시):**

| 지표 | 설명 |
| --- | --- |
| **커버리지 점수** | 입력된 스크립트가 예상 질문 유형을 얼마나 커버하는지 (0~100%) |
| **예상 Q&A 수** | 학습 데이터 기반으로 응답 가능한 예상 질문 수 |
| **보강 권장 항목** | AI가 추가 입력을 권장하는 항목 (예: "작가 생애 정보 부족") |

---

### Step 3. 감정 태그 후보 세팅

B2C 앱에서 관람객이 선택하는 감정 Chip UI의 초기값을 운영자가 사전 설정합니다.

**세팅 방식:**

- Asset Analyzer가 Phase 1 분석 결과를 기반으로 감정 태그 후보 자동 추천
- 운영자가 추천 태그 검토 후 추가·삭제·수정
- 작품 전체 공통 태그 + 작품별 개별 태그 2레벨로 관리
**태그 카테고리 (예시):**

| 카테고리 | 예시 태그 |
| --- | --- |
| 감정 | 압도적인, 평온한, 몽환적인, 쓸쓸한, 경이로운 |
| 분위기 | 고요한, 역동적인, 신비로운, 따뜻한, 차가운 |
| 매체 반응 | 소리가 인상적인, 빛이 아름다운, 움직임이 신기한 |
| 서사 | 이야기가 궁금한, 작가가 궁금한, 더 보고싶은 |

> 💡 태그는 Phase 3 데이터 분석의 핵심 축이 됩니다. 초기 세팅이 부실하면 수집 데이터 품질이 낮아지므로 최소 10개 이상 권장.

---

### Step 4. AI 지식베이스 학습 실행

- **"AI 지식베이스 학습 시작"** 버튼 클릭
- 백그라운드 학습 진행 (작품 수에 따라 5~30분 소요 예상)
- 학습 완료 시 알림 발송 (이메일 + 인앱)
- 학습 완료 후 **테스트 채팅** 기능으로 응답 품질 직접 확인 가능
**테스트 채팅 기능:**

- 운영자가 관람객 입장에서 직접 질문 입력
- AI 응답 확인 후 품질 미달 시 스크립트 보강 후 재학습
- 테스트 결과 저장 및 팀 공유 가능

## 3. 완료 조건 & 다음 단계 진입

| 항목 | 최소 조건 (MVP) | 권장 조건 |
| --- | --- | --- |
| 작품 학습 완료 | 전체 작품의 50% 이상 | 전체 작품 100% |
| 도슨트 스크립트 | 작품당 200자 이상 | 작품당 500자 이상 + 작가 노트 |
| 감정 태그 | 공통 태그 5개 이상 | 카테고리별 3개 이상, 총 15개 이상 |

최소 조건 충족 시 "전시 오픈 & B2C 활성화" 버튼 활성화 → Phase 3 진입.

---

## 4. 검토 의견 & 보강 제안

**1. 테스트 채팅이 핵심 — MVP에 반드시 포함**

운영자가 학습 결과를 직접 확인하지 못하면 AI 품질에 대한 신뢰가 생기지 않습니다. 간단한 채팅 인터페이스라도 MVP에 포함해야 합니다.

**2. 학습 소요 시간 관리**

작품이 많을 경우 학습 시간이 길어질 수 있어 운영자 이탈 가능성이 있습니다. 대안: 작품 단위로 부분 학습 허용 → 일부 학습 완료된 작품부터 B2C 활성화.

**3. 재학습 정책 명확화 필요**

스크립트 수정 후 재학습 시 기존 데이터에 미치는 영향을 명시해야 합니다. 전시 오픈 이후 재학습 시 서비스 중단 없이 처리되는지 여부도 중요합니다.

**4. Tier 2(고객사) vs Tier 3(운영자) 역할 분리**

도슨트 스크립트 입력은 전문성이 필요하므로 고객사 어드민(Tier 2)이 담당하고, 감정 태그 세팅은 현장 감각이 필요하므로 운영자(Tier 3)가 담당하는 구조가 현실적입니다.
