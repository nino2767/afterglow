# Phase 1 상세 — 마스터 자산 등록 (기본 세팅)

> 운영자가 처음 프로젝트를 열었을 때 마주하는 첫 번째 화면. 이 Phase가 탄탄할수록 AI 학습(Phase 2)과 데이터 분석(Phase 3)의 품질이 올라갑니다.

---

## 1. Phase 목표

전시 IP 자산을 시스템에 등록하여 **AI 학습 기반**과 **B2C 데이터 수집 인프라**를 동시에 세팅하는 것. 운영자가 이 Phase를 마치면 AI 큐레이터가 가동될 준비 상태가 됩니다.

---

## 2. Step별 상세 (화면 흐름)

### Step 1. 프로젝트 기본 정보 입력

| 입력 항목 | 타입 | 설명 | 필수 |
| --- | --- | --- | --- |
| **전시명** | text | 예: 빛의 심연: 언더워터 미디어아트 | 필수 |
| **전시 장소** | text | 예: 피플리 갤러리 성수점 | 필수 |
| **전시 기간** | date range | 시작일 ~ 종료일 (팝업 운영 기간 별도 입력) | 필수 |
| **팝업 운영 예정 기간** | date range | 전시 종료 후 최대 21일 | 선택 |
| **예상 월 관람객 수** | number | 수익 시뮬레이션 자동 계산에 사용 | 선택 |
| **전시 장르** | select | 미디어아트 / 몰입형 / 사진 / 회화 / 혼합 | 필수 |

> 💡 전시 기간은 Phase 5 라이브 대시보드의 데이터 수집 기간과 연동됩니다. 팝업 운영 예정 기간은 스핀오프 타임라인 자동 설정에 사용됩니다.

---

### Step 2. 전시 자산 업로드

> 📌 **댓글 검토 반영**: "도록 등 전시 관련 자료 넣을 수 있는 건 다 넣을 수 있어야 할 듯" → 아래와 같이 업로드 가능 자료 범위를 확장합니다.

| 자료 유형 | 포맷 | 역할 | 필수 |
| --- | --- | --- | --- |
| **메인 포스터** | JPG / PNG | 세계관·톤앤매너 추출 기준 | 필수 |
| **전시 기획서** | PDF | Asset Analyzer 핵심 분석 소스 | 필수 |
| **도록 (Catalog)** | PDF | 작품 설명·작가 노트 자동 추출 | 권장 |
| **작가 인터뷰·작가 노트** | PDF / TXT | RAG 학습용 심층 컨텍스트 | 권장 |
| **전시 소개 영상** | URL (YouTube / Vimeo) | 비전-텍스트 변환 보조 | 선택 |
| **언론 보도 자료** | URL / PDF | 세계관 보강 및 마케팅 카피 초안 참고 | 선택 |
| **작품 이미지 에셋** | ZIP (JPG/PNG 일괄) | Phase 2 작품별 AI 인식 학습용 | 권장 |

**업로드 UX 방향:**

- 드래그앤드롭 멀티 업로드 지원
- 파일별 업로드 상태 표시 (분석 중 / 완료 / 오류)
- 업로드 완료 시 Asset Analyzer가 자동 가동되어 핵심 키워드·톤앤매너·세계관 요약 결과를 화면에 노출 → 운영자가 검토·수정 후 확정
---

### Step 3. 공간 구조 입력

> 📌 **댓글 검토 반영**: "어떤식으로 입력되야 하나? 설계도? 평면도??" / "상상이 잘 안되어서 구체화 필요"

현실적으로 전시 기획사가 CAD 파일을 보유한 경우는 드뭅니다. 따라서 **3가지 입력 방법**을 제공하여 어떤 상황에서도 진입 가능하게 합니다.

**방법 A — 평면도 이미지 업로드 + 섹션 태깅 (권장)**

1. 평면도 이미지(JPG/PNG) 업로드
1. 이미지 위에서 드래그로 섹션 영역 지정
1. 섹션명 입력 (예: 입구홀 / 1관-빛의 굴절 / 2관-심해 / 포토존 / 출구)
1. 각 섹션에 QR 배치 위치 핀 지정
**방법 B — 섹션 리스트 직접 입력 (간소화, MVP 기본)**

1. "+ 섹션 추가" 버튼 클릭
1. 섹션명 / 예상 체류 동선 순서(숫자) / QR 배치 여부 입력
1. 섹션 간 동선 순서를 드래그로 정렬
**방법 C — 템플릿 선택 (신규 기획사 온보딩용)**

- 전시 유형별 기본 공간 구조 템플릿 제공
- 예: "몰입형 미디어아트 4관 구조" 템플릿 선택 후 섹션명만 수정
> ⚠️ **MVP 범위**: 방법 B(섹션 리스트)를 우선 구현하고, 방법 A(이미지 태깅)는 v1.0에서 추가. 방법 C(템플릿)는 v2.0에서 추가.

## 3. Asset Analyzer 결과 화면

Step 2 업로드 완료 후 AI가 자동 분석한 결과를 운영자가 확인·수정하는 화면입니다.

| 분석 결과 항목 | 내용 예시 | 운영자 액션 |
| --- | --- | --- |
| **핵심 키워드** | #심해 #빛의굴절 #신비로움 #고요함 | 추가/삭제 가능 |
| **톤앤매너** | 딥블루 / 에메랄드 그린 / 몽환적·느린 호흡 | 수정 가능 |
| **세계관 요약** | AI가 생성한 1-2문단 요약 | 수정 후 확정 |
| **추출된 작가 목록** | 자동 감지된 작가명 리스트 | 추가/삭제 가능 |
| **AI 추천 감정 태그** | #압도적인 #편안한 #몽환적인 #쓸쓸한 | Phase 2로 연동 |

> 💡 이 결과물이 Phase 2(AI 학습)의 초기값이 되고, Phase 4(스핀오프 기획)의 컨셉 카드 데이터 근거로 활용됩니다.

**[댓글] "도록 등 전시 관련 자료 넣을 수 있는 건 다 넣을 수 있어야 할 듯. 기간도 필요."**

→ Step 2에서 6가지 자료 유형으로 확장. 전시 기간·팝업 기간은 Step 1에 명시.

### 🔍 추가 보강 제안 (내 의견)

**1. Asset Analyzer 결과 확인 화면이 없었음 → 추가 필요**

AI가 분석한 결과를 운영자가 검토·수정할 수 있는 화면이 없으면 "블랙박스" 느낌. 운영자가 결과를 확인하고 수정하는 Step이 있어야 신뢰도가 생기고, 이후 AI 제안에 대한 납득도가 높아집니다.

**2. QR 배포 방식이 미정 → 현장 현실 고려 필요**

QR을 PDF로 출력해서 현장에 붙이는 방식은 현실적이지만, 교체/훼손 시 대응 방법이 필요합니다. 대안: QR은 고정 URL(영구적), 내용만 업데이트되는 방식으로 설계하면 현장 관리 부담 감소.

**3. 여러 전시를 동시에 운영하는 기획사 케이스**

피플리처럼 여러 전시를 동시에 운영하는 기획사는 프로젝트 목록 화면이 필요합니다. "전시 목록 → 전시 선택 → Phase 1" 진입 구조로 설계해야 합니다.

**4. 운영자 권한별 접근 차이**

고객사 어드민(Tier 2)이 Step 1~2를 담당하고, 현장 운영자(Tier 3)가 Step 3~4를 담당하는 경우가 현실적. 권한별 접근 가능 Step을 명시해야 합니다.
