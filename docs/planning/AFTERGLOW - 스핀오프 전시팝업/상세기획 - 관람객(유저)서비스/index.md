# 상세기획 - 관람객(유저)서비스

## 1. 방향

AFTERGLOW B2C 관람객 경험은 **"낮은 허들, 깊은 여운, 자연스러운 전환"** 3가지 원칙으로 설계합니다.

- **낮은 허들**: 별도 앱 설치 없이 QR 스캔만으로 진입. 사진 한 장만 찍어도 아카이빙되는 부담 없는 시작.
- **깊은 여운**: 행동 데이터를 축적해 관람 종료 시 나만의 여운 리포트 자동 생성. 전시가 끝난 후에도 감동이 남도록.
- **자연스러운 전환**: 강요 없이 데이터 기반 개인화 초대장으로 스핀오프 팝업으로 연결. 관람 경험이 소비로 이어지는 루프 완성.
> 핵심 전략: 행동의 허들은 낮추되(사진만 찍어도 아카이빙), 여운의 깊이는 더하여(나만의 리포트), 최종적으로 스핀오프라는 다음 스테이지로 관람객을 안내합니다.

---

## 2. 목표 및 기대효과

| 구분 | 목표 | 기대효과 |
| --- | --- | --- |
| **관람 몰입도** | AI 큐레이터 활성화율 (Level 2 진입) 30% 이상 | 전시 만족도 향상 → 긍정 후기·SNS 확산 |
| **B2C 전환율** | 여운 리포트 수령 후 스핀오프 초대장 클릭률 20% 이상 | 전시 종료 후 추가 매출 발생 |
| **데이터 유효성** | 수집된 감정 태그와 실제 굿즈 구매 의향 일치율 검증 | B2B 운영자 컨셉 도출 정확도 향상 |
| **재방문 유도** | 스핀오프 팝업 방문자 중 재관람 의향률 40% 이상 | 전시 IP 수명 연장 및 팬덤 형성 |

## 3. IA (정보 구조)

![image]()

5개 Phase × 화면 × 주요 기능 구조: **온보딩 → 카메라 뷰어(Home) → 스냅 정보카드(Level 1) → AI 큐레이터 채팅(Level 2) → 브릿지 리포트 → 스핀오프 랜딩**

수집된 B2C 행동 데이터는 전량 B2B 운영자 대시보드(Phase 3·4·5·6)로 연동됩니다.

## 4. 유저 플로우 (User Flow)

![image]()

5개 Phase를 가로로 펼친 전체 플로우입니다. 각 단계별 분기(CTA 선택, 카메라 거부, AI Push 트리거, 관련 작품 추천 수락/거절, 초대장 클릭 여부)와 예외 경로가 포함되어 있습니다.

- **정방향 플로우**: QR 스캔 → 온보딩 → 촬영 → AI 대화 → 리포트 → 스핀오프 방문 → 굿즈 결제
- **분기 경로**: 카메라 거부(텍스트 모드 전환), 저장하고 닫기(카메라 복귀), AI Push 트리거(Level 2 자동 유도), 초대장 미클릭(GPS 푸시 발송)
- **데이터 루프**: 모든 단계의 행동 데이터 → B2B 운영자 대시보드 실시간 연동

### Phase 2-1. 상호작용 Level 1 (📷 스냅 캡처)

| 사용자 Task (액션) | 서비스 대응 (화면/AI 액션) | 수집 데이터 | B2B 연동 |
| --- | --- | --- | --- |
| 1. 마음에 드는 작품을 뷰파인더로 찰칵 찍는다 | 사진 자동 저장 + 하단 정보 카드(Bottom Sheet) 팝업 | 촬영 작품, 촬영 시각, 촬영 위치 | 운영자 Phase 3 작품별 반응 히트맵 |
| 2. 작품명 및 한 줄 요약 확인 | 작품 정보 카드 노출 (작품명, 작가명, 한 줄 요약) | 정보 카드 열람 시간 | 운영자 Phase 3 체류 데이터 |
| 3. AI 자동 태그 확인 (예: #푸른색 #고독) | 사진 분석 기반 키워드 자동 노출 | 노출된 태그 목록 | 운영자 Phase 3 반응 키워드 랭킹 |
| 4. **저장 / 공유 / 좋아요** 액션 | 저장 → 도슨트 북에 추가, 공유 → SNS 공유 시트, 좋아요 → 선호 표시 | 저장·공유·좋아요 여부 및 횟수 | 운영자 Phase 3 콘텐츠 반응 데이터 |
| 5. CTA 선택: [저장하고 닫기] 또는 [AI에게 더 물어보기 💬] | 닫기 → 카메라 뷰어 복귀 / 물어보기 → Level 2 진입 | CTA 선택 기록 | 운영자 Phase 3 관심도 분석 |
|  |  |  |  |

> 💡 **Push 트리거 (AI 능동 개입):** 같은 작품을 2회 이상 촬영하거나, 정보 카드를 15초 이상 열람하면 AI가 "이 작품이 특별히 마음에 드시나 봐요. 더 깊은 이야기 들어보실래요?" 알림을 능동적으로 전송 → Level 2 자연 유도

> ⚠️ **예외 처리:** AI가 작품을 인식하지 못할 경우 → "아직 이 작품을 학습하지 못했어요. 작품 옆 QR을 스캔해 보세요" 안내 + QR 수동 입력 대체 경로

> ⚠️ **예외 처리:** 네트워크 끊김 시 → 마지막 로드된 작품 정보 로컬 캐시 유지 + "연결이 불안정해요. 저장된 정보를 보여드릴게요" 안내, 재연결 시 자동 동기화

### Phase 3. 브릿지 (💌 여운 리포트 & 초대장)

| 사용자 Task (액션) | 서비스 대응 (화면/AI 액션) | 수집 데이터 | B2B 연동 |
| --- | --- | --- | --- |
| 1. 전시 종료 시점, 내가 찍은 사진들이 엮인 **'나만의 여운 리포트'** 확인 | 관람 동선 기반 디지털 도슨트 북 자동 생성 | 리포트 열람 여부, 열람 시간 | 운영자 Phase 5 라이브 대시보드 |
| 2. 나의 취향 분석 결과 확인 | 수집된 감정 태그 기반 취향 분석 (예: "S님은 '차분한 블루' 테마에 가장 많이 머무르셨네요") | 취향 분석 결과 확인 여부 | 운영자 Phase 3 세그먼트 데이터 |
| 3. **스핀오프 초대장 확인** | 개인화 초대장 UI 노출 (예: "S님이 좋아하신 '블루' 테마를 커스텀 굿즈로 만나보세요") | 초대장 노출 여부 | 운영자 Phase 5 초대장 CTR |
| 4. **[스핀오프 팝업 예약하기] 또는 [나만의 맞춤 굿즈 확인하기]** 클릭 | 예약 페이지 또는 굿즈 프리뷰 페이지로 이동 | 클릭 여부, 전환 여부 | 운영자 Phase 5 팝업 방문율 |

> 💡 **리포트 생성 트리거 조건:**

> - **자동 트리거:** GPS/비콘 기반 전시장 이탈 감지 시 푸시 알림 발송 → "스핀오프 전시가 500m 거리에 있습니다"

> - **수동 트리거:** 웹앱 내 "관람 종료" 버튼 클릭

> - **최소 조건:** 2개 이상의 작품 촬영 시 리포트 생성 가능 (1개 이하일 경우 "더 많은 작품을 만나보세요" 안내)

> ⚠️ **예외 처리:** 사용자가 촬영 없이 전시를 나갈 경우 → QR 스캔 기록 기반 간소화 리포트 제공 + "다음에는 카메라로 찍어보세요" 재방문 유도 메시지

## 📱 B2C 웹앱 와이어프레임 (UI/UX 정보 구조)

> 각 화면이 어떤 Phase의 Task를 실행하는지 매핑하여 정리합니다.

### 1. [Home] 메인 화면 (카메라 뷰어) → Phase 1 완료 후 진입, Phase 2-1 실행

> **핵심 의도:** 거추장스러운 메뉴 없이 바로 미디어아트를 찍을 수 있도록 유도

- **배경 (Background):** 전체 화면 카메라 뷰파인더 (실시간 화면)
- **상단 영역 (Top Bar):**
    - 좌측: 내 프로필 (나의 전시 여정)
    - 우측: 알림 아이콘 (AI가 보내는 팁이나 Push 트리거 알림)
- **중앙 영역 (Center):**
    - 포커스 가이드라인 (작품을 가운데 맞추도록 돕는 옅은 UI)
    - **작품 미인식 시 안내 텍스트** ("작품에 카메라를 맞춰주세요")
- **하단 영역 (Bottom Action):**
    - 중앙: **핵심 액션 버튼 (대형 촬영 셔터 버튼 📸)**
    - 좌측 하단: QR 수동 스캔 버튼 (카메라 인식 실패 대비)
    - 우측 하단 (FAB): '나의 도슨트 북(리포트)' 미리보기 썸네일 (찍을 때마다 숫자가 카운팅됨)
### 2. [Level 1] 상호작용 화면 (스냅 & 정보 카드) → Phase 2-1 실행

> **핵심 의도:** 사진을 찍는 즉시 하단에서 '바텀 시트(Bottom Sheet)'가 자연스럽게 올라와 작품 정보를 제공

- **배경:** 방금 찍은 사진이 백그라운드로 깔리며 살짝 블러(Blur) 처리됨
- **하단 팝업 (Bottom Sheet):**
    - **작품 정보:** 작품명, 작가명, 한 줄 요약 (예: "현대인의 고독을 표현한 푸른 방")
    - **AI 자동 태그:** 사진 분석 기반 키워드 노출 (예: #푸른색 #고독 #반고흐)
    - **경량 인터랙션 바:** ❤️ 좋아요 | 📥 저장 | 📤 공유
    - **CTA (Call to Action) 버튼:**
        - [저장하고 닫기] (계속 관람하기)
        - [AI에게 더 물어보기 💬] (Level 2로 진입하는 트리거)
### 3. [Level 2] 딥다이브 화면 (AI 큐레이터 채팅) → Phase 2-2 실행

> **핵심 의도:** 작품에 대해 더 깊이 알고 싶은 고관여 사용자에게 인터랙티브한 대화와 감정 태그 수집 제공

- **상단 (Header):** 채팅방 타이틀 (예: '푸른 방' 도슨트 진행 중) 및 뒤로 가기 버튼
- **중앙 (Chat Area):**
    - 내가 찍은 사진이 말풍선 형태로 업로드됨
    - AI의 심층 해설 말풍선 (RAG 기반 정확한 정보)
    - **감정 태그 선택지 (Chip UI):** "이 작품을 보니 어떤 느낌이 드나요?"
        - 버튼 예시: [압도적인] [편안한] [쓸쓸한] [직접 입력]
    - **관련 작품 추천 카드:** "이 작품과 비슷한 감성의 작품이 있어요" (수락 시 해당 작품 위치 안내)
- **하단 (Input Area):**
    - 추가 질문 입력창 및 마이크 아이콘(음성 입력 지원)
    - **오프라인 표시:** 네트워크 끊김 시 "오프라인 모드 — 저장된 정보를 보여드려요" 배너
### 4. [Bridge] 여운 리포트 & 초대장 화면 → Phase 3 실행

> **핵심 의도:** 관람 종료 후 성취감을 주고, 스핀오프 팝업으로의 전환(전시 IP 확장)을 이끌어냄

- **상단 (Header):** "S님의 전시 여운 리포트가 도착했습니다 💌"
- **중앙 (Report Body):**
    - **관람 요약:** 내가 찍은 작품 사진들 (가로 스와이프 갤러리)
    - **관람 통계:** 총 관람 시간, 촬영 작품 수, AI 대화 횟수
    - **나의 취향 분석:** 수집된 감정 태그 기반 결과 (예: "S님은 이번 전시에서 '차분한 블루' 테마에 가장 많이 머무르셨네요.")
- **하단 (Spinoff Invitation - 가장 중요 ⭐️):**
    - **초대장 UI:** 시각적으로 눈에 띄는 티켓 형태의 디자인
    - **개인화 카피:** "S님이 좋아하신 '블루' 테마를 커스텀 굿즈로 만나보세요."
    - **CTA 버튼:** [스핀오프 팝업 예약하기] 또는 [나만의 맞춤 굿즈 확인하기]

## 💡 UI/UX 설계 팁

- **매끄러운 전환 (Seamless Transition):** Level 1에서 정보 카드를 볼 때, 사용자가 언제든 화면을 아래로 스와이프하면 카드가 닫히고 즉시 카메라 뷰어(Home)로 돌아갈 수 있어야 관람 흐름이 끊기지 않습니다.
- **스핀오프 초대장의 위치:** 리포트 화면뿐만 아니라, 사용자가 전시장을 나서는 순간 위치 기반(GPS/비콘)으로 **"스핀오프 전시가 500m 거리에 있습니다"**라는 푸시 알림을 주는 것도 훌륭한 브릿지 전략이 됩니다.
- **오프라인 대응:** 전시장 내 네트워크가 불안정할 수 있으므로, Level 1 정보 카드와 기본 작품 데이터는 앱 진입 시 사전 로드(프리캐시)하여 오프라인에서도 기본 경험이 보장되어야 합니다.
