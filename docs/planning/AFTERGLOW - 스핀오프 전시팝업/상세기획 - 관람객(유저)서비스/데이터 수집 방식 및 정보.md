# 데이터 수집 방식 및 정보 

# 1. 관람 반응 데이터 수집

### - 기능 정의

관람 반응 데이터 수집은 전시 현장에서 관람객이 남기는 행동, 선택, 체류, 저장, 공유, 감정 반응을 작품·구간·주제 단위로 수집하는 기능이다.

단순히 “어떤 작품이 인기 있었는가”를 확인하는 것이 아니라, 관람객이 전시 안에서 **어디에 오래 머물렀는지, 어떤 정보를 더 보고 싶어 했는지, 어떤 감정으로 반응했는지, 어떤 요소를 이후에도 저장하거나 공유했는지**를 구조화하여 전시 이후의 스핀오프 제안 근거로 활용하는 것이 목적이다.

---

# 1-1. 고객 관점의 핵심 경험

관람자는 별도의 복잡한 절차 없이 전시를 관람하는 과정에서 자연스럽게 반응을 남긴다.

관람 중에는 작품 앞 QR을 스캔하거나, 마음에 드는 작품을 저장하거나, 감정 태그를 선택하거나, 더 깊은 설명을 확인하는 식으로 참여한다. 관람 종료 후에는 본인이 반응했던 작품과 키워드를 기반으로 개인화된 관람 리포트나 후속 콘텐츠 추천을 받을 수 있다.

즉 관람자에게는 “데이터 수집”이 아니라, **내가 본 전시를 더 깊게 기억하고 이어갈 수 있는 개인화 경험**으로 제공되어야 한다.

---

# 1-2. 오프라인 전시 현장 구현 방식

## A. 전시 입장 시점: 관람 세션 생성

### 구현 방식

전시 입장 시 관람객에게 AFTERGLOW 기반의 관람 세션을 생성하도록 유도한다.

현장에서는 아래 방식 중 하나로 진입할 수 있다.

| 방식 | 설명 | 초기 적용 적합도 |
| --- | --- | --- |
| 입장 티켓 QR 연동 | 예매/입장 티켓의 QR을 스캔하면 관람 세션 생성 | 높음 |
| 전시장 입구 QR 스캔 | 입구 배너나 리플렛의 QR을 통해 모바일 웹 진입 | 높음 |
| NFC 태그 진입 | 입구에 비치된 NFC 태그 터치로 진입 | 중간 |
| 전용 앱 연동 | 별도 앱에서 전시 관람 모드 시작 | 초기에는 낮음 |

초기 PoC 단계에서는 **앱 설치 없이 모바일 웹 기반 QR 진입 방식**이 가장 적합하다. 관람객 입장에서 진입 장벽이 낮고, 전시장 운영사 입장에서도 별도 장비 부담이 적기 때문이다.

### 관람자 화면 예시

입구 QR 스캔 후 다음과 같은 안내 화면을 제공한다.

> “오늘의 전시에서 인상 깊었던 작품과 감정을 기록해보세요.

> 관람 후 나만의 전시 리포트와 후속 추천을 받아볼 수 있어요.”

### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| exhibition_id | 전시 고유 ID |
| visitor_session_id | 익명 또는 회원 기반 관람 세션 ID |
| entry_time | 관람 시작 시간 |
| entry_channel | 티켓 QR, 입구 QR, 제휴 채널 등 |
| consent_status | 데이터 활용 동의 여부 |
| visitor_type | 일반 관람객, 초대권, VIP, 단체 관람 등 |

### 기획 포인트

초기 진입 단계에서 개인정보 입력을 과도하게 요구하면 참여율이 낮아질 수 있다. 따라서 초기에는 **익명 세션 기반으로 관람을 시작하고, 관람 종료 후 리포트 저장 또는 후속 추천 수신 단계에서 회원 전환이나 연락처 입력을 유도하는 방식**이 적합하다.

## B. 작품/섹션별 QR 반응 수집

### 구현 방식

각 작품 또는 섹션 옆에 QR 코드를 배치한다. 관람객이 QR을 스캔하면 해당 작품의 상세 설명, 작가 이야기, 제작 배경, 관련 이미지, 오디오 해설, 후속 콘텐츠가 노출된다.

이때 QR 스캔 자체가 작품에 대한 명시적 관심 신호로 기록된다.

### QR 배치 단위

| 배치 단위 | 활용 목적 |
| --- | --- |
| 작품별 QR | 개별 작품에 대한 관심도 측정 |
| 섹션별 QR | 특정 주제/구간에 대한 관심도 측정 |
| 작가별 QR | 작가 서사 및 인물 중심 반응 측정 |
| 포토존 QR | 촬영/공유 중심 반응 측정 |
| 굿즈존 QR | 구매 관심 및 확장 가능성 측정 |

### 관람자 화면 예시

작품 QR 스캔 시 다음과 같은 구성을 제공한다.

| 화면 요소 | 설명 |
| --- | --- |
| 작품명/작가명 | 기본 정보 |
| 1분 요약 설명 | 현장에서 빠르게 읽을 수 있는 짧은 설명 |
| 더 깊게 보기 | 상세 해설, 작가 서사, 시대 배경 |
| 감정 태그 선택 | 이 작품을 보고 느낀 감정 선택 |
| 저장하기 | 나중에 다시 볼 작품으로 저장 |
| 공유하기 | SNS 또는 링크 공유 |
| 관련 작품 보기 | 같은 작가/주제/시대의 다른 작품 연결 |

### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| qr_scan_count | 작품/섹션별 QR 스캔 수 |
| unique_scan_count | 중복 제외 스캔 수 |
| repeat_scan_count | 동일 세션의 재스캔 수 |
| content_view_depth | 기본 설명만 봤는지, 상세 설명까지 봤는지 |
| dwell_after_scan | QR 스캔 후 콘텐츠 체류 시간 |
| save_action | 저장 여부 |
| share_action | 공유 여부 |
| related_click | 관련 작품/작가 클릭 여부 |

### 스핀오프 활용 의미

QR 스캔이 많은 작품은 단순 인기 작품으로 볼 수 있지만, 상세 설명까지 많이 열람된 경우에는 **시각적 호감보다 서사적 관심이 강한 작품**으로 해석할 수 있다.

예를 들어 특정 작품의 QR 스캔 수는 보통 수준이지만, 상세 설명 열람률과 저장률이 높다면 해당 작품은 대중적 흡입력은 낮아도 **깊이 관람하는 고객층에게 강하게 반응한 작품**으로 볼 수 있다. 이는 스핀오프 기획에서 소형 기획전, 작가 서사 콘텐츠, 도슨트형 프로그램으로 확장할 수 있는 근거가 된다.

## C. 작품별 체류 시간 수집

### 구현 방식

작품별 체류 시간은 현장 환경과 예산 수준에 따라 단계적으로 구현한다.

| 구현 방식 | 설명 | 장점 | 한계 |
| --- | --- | --- | --- |
| QR 기반 간접 체류 | QR 스캔 후 해당 콘텐츠 페이지 체류 시간 측정 | 초기 구현 쉬움 | 실제 작품 앞 체류와는 차이 있음 |
| 비콘/BLE 기반 측정 | 관람객 모바일과 근거리 신호로 구간 체류 추정 | 구간 단위 분석 가능 | 동의 및 장비 필요 |
| Wi-Fi 위치 기반 | 전시장 내 Wi-Fi 접속 위치로 동선 추정 | 넓은 공간 분석 가능 | 정확도 제한 |
| 카메라/센서 기반 | 작품 앞 밀집도, 체류 인원 측정 | 작품 단위 분석 가능 | 개인정보·설치비 고려 필요 |
| 인터랙티브 존 로그 | 터치스크린, 키오스크 사용 시간 측정 | 명확한 행동 데이터 | 설치 구역 한정 |

초기에는 **QR 기반 콘텐츠 체류 + 섹션 단위 센서/비콘** 조합이 현실적이다. 이후 전시 규모와 고객사 요구에 따라 카메라/센서 기반 분석을 추가할 수 있다.

### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| section_entry_time | 특정 구간 진입 추정 시간 |
| section_exit_time | 특정 구간 이탈 추정 시간 |
| section_dwell_time | 구간 체류 시간 |
| artwork_content_dwell_time | 작품 상세 페이지 체류 시간 |
| average_dwell_time | 작품/섹션별 평균 체류 시간 |
| high_dwell_ratio | 평균 대비 오래 머문 관람객 비율 |
| pass_through_ratio | 빠르게 지나간 비율 |

### 고객 관점 해석

관람객은 체류 시간을 별도로 입력하지 않아도 된다. 대신 시스템은 관람 흐름 속에서 자연스럽게 체류 데이터를 수집한다.

고객사는 이를 통해 어떤 구간이 관람객을 오래 붙잡았는지, 어느 구간에서 빠르게 이탈했는지 확인할 수 있다.

### 스핀오프 활용 의미

체류 시간이 긴 구간은 몰입도가 높은 구간으로 볼 수 있다. 다만 체류 시간이 길다고 무조건 긍정적이라고 해석해서는 안 된다.

예를 들어 체류 시간이 긴 이유는 다음처럼 다를 수 있다.

| 체류 시간 증가 원인 | 해석 |
| --- | --- |
| 작품 몰입 | 긍정적 관심 |
| 설명 난이도 높음 | 이해 부담 가능성 |
| 동선 정체 | 공간 운영 이슈 |
| 포토존 대기 | 촬영 목적 관심 |
| 인터랙션 대기 | 참여형 콘텐츠 관심 |

따라서 체류 시간은 단독 지표가 아니라 QR 스캔, 저장, 감정 태그, 사진 촬영, 공유 행동과 함께 해석해야 한다.

## D. 사진 촬영 반응 수집

### 구현 방식

오프라인 전시에서 사진 촬영 여부는 관람객의 강한 관심 신호가 될 수 있다. 다만 개인 사진을 직접 수집하는 방식은 개인정보 이슈가 있으므로, 초기에는 관람객이 직접 업로드하거나 포토존 QR을 활용하는 방식이 적합하다.

### 적용 방식

| 방식 | 설명 |
| --- | --- |
| 포토존 QR 인증 | 포토존 옆 QR을 스캔해 촬영/저장 페이지 진입 |
| 촬영 후 해시태그 이벤트 | 특정 해시태그 공유 시 참여 인정 |
| 사진 업로드형 리포트 | 관람객이 인상 깊은 장면을 직접 업로드 |
| AR 필터/프레임 제공 | 작품별 프레임을 제공하고 저장/공유 유도 |
| 촬영 가능 구역 선택 기록 | 관람 후 “가장 찍고 싶었던 구간” 선택 |

### 관람자 화면 예시

> “이 장면이 마음에 들었다면 나의 관람 리포트에 저장해보세요.”

또는

> “가장 기억에 남는 장면을 선택하면 전시 종료 후 나만의 감상 카드로 받아볼 수 있어요.”

### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| photo_zone_scan | 포토존 QR 스캔 여부 |
| photo_upload | 사진 업로드 여부 |
| sns_share | SNS 공유 여부 |
| hashtag_participation | 해시태그 이벤트 참여 여부 |
| frame_save | AR 프레임 저장 여부 |
| visual_preference_tag | 선택한 이미지/색감/공간 분위기 |

### 스핀오프 활용 의미

사진 촬영 반응이 높은 구간은 관람객이 **소장하고 싶어 하는 장면**, **타인에게 보여주고 싶은 장면**, **SNS 확산 가능성이 높은 장면**으로 해석할 수 있다.

이는 후속 팝업, 굿즈, 포토존 중심 전시, 몰입형 공간 연출 기획의 근거가 된다.

## E. 감정 태그 및 선호 키워드 수집

### 구현 방식

작품 상세 화면 또는 섹션 종료 지점에서 관람객이 간단하게 감정 태그를 선택할 수 있도록 한다.

중요한 점은 관람 중 흐름을 방해하지 않도록 선택지를 짧고 직관적으로 제공하는 것이다.

### 감정 태그 예시

| 분류 | 태그 예시 |
| --- | --- |
| 정서 반응 | 따뜻한, 쓸쓸한, 압도적인, 불안한, 평온한 |
| 몰입 반응 | 오래 보고 싶은, 더 알고 싶은, 다시 보고 싶은 |
| 시각 반응 | 색감이 좋은, 구성이 인상적인, 빛이 아름다운 |
| 서사 반응 | 이야기가 궁금한, 관계성이 흥미로운, 시대 배경이 궁금한 |
| 확장 반응 | 굿즈로 갖고 싶은, 영상으로 보고 싶은, 강연으로 듣고 싶은 |

### 화면 구성 예시

작품 설명 하단에 다음 질문을 제공한다.

> “이 작품을 보고 어떤 느낌이 들었나요?”

선택지는 5~7개 내외로 제한한다.

예시:

- 따뜻했어요
- 쓸쓸했어요
- 압도적이었어요
- 더 알고 싶었어요
- 다시 보고 싶어요
- 사진으로 남기고 싶어요
### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| emotion_tag | 선택 감정 태그 |
| preference_keyword | 선호 키워드 |
| tag_selection_time | 태그 선택 시점 |
| tag_by_artwork | 작품별 감정 반응 |
| tag_by_section | 섹션별 감정 반응 |
| tag_combination | 함께 선택된 태그 조합 |

### 스핀오프 활용 의미

감정 태그는 스핀오프 컨셉을 도출하는 데 매우 중요한 데이터가 된다.

예를 들어 특정 전시에서 “쓸쓸한”, “편지”, “관계성”, “시대 배경” 키워드가 반복적으로 높게 나타난다면, 후속 전시는 단순히 작가 대표작 중심이 아니라 **서간문, 관계 서사, 시대적 고립감**을 중심으로 기획할 수 있다.

## F. 저장, 좋아요, 공유 반응 수집

### 구현 방식

관람객이 작품 또는 콘텐츠를 나중에 다시 볼 수 있도록 저장 기능을 제공한다. 저장은 관람객에게 실질적인 효용이 있으면서도, 서비스 입장에서는 강한 선호 데이터가 된다.

### 주요 액션

| 액션 | 의미 |
| --- | --- |
| 저장 | 다시 보고 싶은 관심 |
| 좋아요 | 즉각적 호감 |
| 공유 | 타인에게 알리고 싶은 확산성 |
| 더 알아보기 클릭 | 정보 탐색 욕구 |
| 관련 콘텐츠 클릭 | 확장 탐색 가능성 |
| 굿즈 보기 클릭 | 상품화 가능성 |

### 관람자 화면 예시

> “마음에 든 작품을 저장하면 관람 후 한 번에 다시 볼 수 있어요.”

전시 종료 후에는 다음과 같이 연결한다.

> “쏠님이 오래 머문 작품과 저장한 키워드를 바탕으로 후속 추천을 준비했어요.”

### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| save_count | 작품별 저장 수 |
| like_count | 작품별 좋아요 수 |
| share_count | 작품별 공유 수 |
| detail_click_count | 상세 설명 클릭 수 |
| related_content_click | 관련 콘텐츠 클릭 수 |
| post_visit_reopen | 전시 종료 후 다시 열람 여부 |

### 스핀오프 활용 의미

저장률이 높은 작품은 관람객의 사후 관심이 높은 콘텐츠다. 공유율이 높은 작품은 확산 가능성이 높고, 더 알아보기 클릭률이 높은 작품은 심화 콘텐츠로 확장 가능성이 높다.

따라서 각 액션은 다음처럼 다르게 해석해야 한다.

| 데이터 | 해석 |
| --- | --- |
| 좋아요 높음 | 즉각적 호감 |
| 저장 높음 | 지속 관심 |
| 공유 높음 | 확산성 |
| 상세 보기 높음 | 서사/정보 탐색 욕구 |
| 굿즈 클릭 높음 | 상품화 가능성 |
| 재열람 높음 | 전시 이후 연결 가능성 |

## G. 관람 동선 및 이탈 구간 수집

### 구현 방식

관람 동선은 작품별 QR 스캔 순서, 섹션별 진입 로그, 비콘/센서 데이터, 인터랙션 참여 순서를 조합해 추정한다.

초기에는 정밀한 위치 추적보다 **QR 스캔 순서 기반의 간접 동선 분석**부터 시작하는 것이 적합하다.

### 수집 데이터

| 데이터 항목 | 설명 |
| --- | --- |
| visit_sequence | 관람 순서 |
| section_visit_rate | 섹션별 방문율 |
| drop_off_section | 이탈 추정 구간 |
| skipped_section | 스캔/체류가 적은 구간 |
| reentry_section | 재방문한 구간 |
| last_engaged_section | 마지막으로 반응한 구간 |

### 고객사 활용 관점

고객사는 이 데이터를 통해 관람객이 전시 의도대로 이동했는지, 특정 구간에서 흥미가 떨어졌는지, 마지막까지 몰입이 유지되었는지를 확인할 수 있다.

### 스핀오프 활용 의미

이탈이 많은 구간은 스핀오프 확장 대상이라기보다 개선 대상일 수 있다. 반대로 재방문이 많은 구간이나 관람 후반부에도 반응이 유지된 구간은 후속 콘텐츠로 확장할 가능성이 높다.

# 1-3. 데이터 수집 설계 원칙

## A. 관람 흐름을 방해하지 않아야 함

전시 현장에서는 관람객이 작품에 집중해야 하므로, 데이터 수집을 위한 액션이 과도하면 안 된다. QR 스캔, 저장, 감정 태그 등은 모두 3초 이내에 완료 가능한 수준으로 설계하는 것이 적합하다.

## B. 관람객에게 즉시 보상이 있어야 함

관람객이 데이터를 남기는 이유가 명확해야 한다.

예를 들어 다음과 같은 보상을 제공할 수 있다.

| 참여 행동 | 관람객 보상 |
| --- | --- |
| 작품 저장 | 관람 후 나만의 작품 리스트 제공 |
| 감정 태그 선택 | 나의 전시 감상 키워드 제공 |
| QR 스캔 | 더 깊은 작품 설명 제공 |
| 포토존 참여 | 감상 카드 또는 이미지 프레임 제공 |
| 관람 완료 | 개인화 전시 리포트 제공 |

## C. 익명 기반 수집부터 시작해야 함

초기에는 개인정보 수집을 최소화하고, 관람 세션 단위로 데이터를 쌓는 것이 좋다. 이후 관람객이 리포트를 저장하거나 후속 추천을 받기 원할 때 회원가입, 이메일, 휴대폰 번호, 카카오 연동 등을 선택적으로 제공한다.

## D. 데이터는 작품·섹션·주제 단위로 태깅되어야 함

각 작품과 섹션에는 사전에 메타데이터가 부여되어야 한다.

예시:

| 메타데이터 축 | 예시 |
| --- | --- |
| 작가 | 반고흐 |
| 시대 | 후기 인상주의 |
| 주제 | 고독, 우정, 자연 |
| 감정 | 쓸쓸함, 강렬함 |
| 매체 | 회화, 드로잉, 편지 |
| 서사 | 고갱과의 관계, 아를 시기 |
| 공간 유형 | 포토존, 몰입형 영상, 아카이브존 |

이 메타데이터가 있어야 수집된 행동 데이터를 단순 클릭 수가 아니라 “어떤 주제와 감정에 반응했는가”로 해석할 수 있다.

# 2) 반응 패턴 분석

## 2-1. 기능 정의

반응 패턴 분석은 전시 현장에서 수집된 행동 데이터를 작품·섹션·작가·감정·주제·매체 단위로 재분류하고, 관람객이 실제로 어떤 요소에 강하게 반응했는지 해석하는 기능이다.

핵심은 단순 인기 순위가 아니라, **몰입 신호, 탐색 신호, 저장 신호, 확산 신호, 이탈 신호를 조합해 스핀오프 가능성이 높은 반응 패턴을 도출하는 것**이다.

---

## 2-2. 분석 구조

반응 패턴은 크게 다섯 가지 신호로 구분한다.

| 분석 신호 | 의미 | 주요 데이터 |
| --- | --- | --- |
| 몰입 신호 | 오래 머물고 깊게 본 반응 | 체류 시간, 상세 보기, 재열람 |
| 호감 신호 | 즉각적으로 좋다고 표현한 반응 | 좋아요, 감정 태그, 저장 |
| 탐색 신호 | 더 알고 싶어 한 반응 | 더 알아보기, 관련 콘텐츠 클릭 |
| 확산 신호 | 타인에게 공유하고 싶은 반응 | 공유, 사진 촬영, 해시태그 |
| 이탈 신호 | 관심이 낮거나 흐름이 끊긴 반응 | 낮은 체류, 낮은 스캔, 중도 이탈 |

---

# 2-3. 주요 분석 기능 상세

## A. 작품별 반응 점수 분석

### 기능 설명

작품별로 수집된 반응 데이터를 종합해 반응 점수를 산출한다.

단순 조회 수만으로 인기 작품을 판단하지 않고, 체류 시간, 저장, 공유, 감정 태그, 상세 보기 등을 함께 반영한다.

### 예시 지표

| 지표 | 설명 |
| --- | --- |
| 관심도 점수 | QR 스캔, 상세 보기, 저장 기반 |
| 몰입도 점수 | 체류 시간, 콘텐츠 체류, 재열람 기반 |
| 확산성 점수 | 공유, 사진 촬영, SNS 반응 기반 |
| 서사 반응 점수 | 상세 설명, 작가 이야기, 관계 콘텐츠 클릭 기반 |
| 상품화 가능성 점수 | 굿즈 보기, 저장, 이미지 선호 기반 |

### 분석 결과 예시

| 작품 | 관심도 | 몰입도 | 확산성 | 주요 반응 |
| --- | --- | --- | --- | --- |
| 작품 A | 높음 | 높음 | 중간 | 작가 서사 관심 |
| 작품 B | 높음 | 낮음 | 높음 | 포토존/이미지 확산형 |
| 작품 C | 중간 | 높음 | 낮음 | 깊이 관람형 |
| 작품 D | 낮음 | 낮음 | 낮음 | 개선 필요 |

### 고객사 인사이트

작품 B는 SNS 확산형 콘텐츠로 적합하고, 작품 C는 대중성은 낮지만 깊은 관람을 유도하는 해설형 콘텐츠로 확장할 수 있다.

---

## B. 섹션별 몰입 구간 분석

### 기능 설명

전시를 구성하는 섹션별로 관람객 반응을 비교하여, 어떤 구간에서 몰입이 높고 어떤 구간에서 이탈이 발생했는지 분석한다.

### 분석 항목

| 항목 | 설명 |
| --- | --- |
| 섹션 진입률 | 해당 섹션에 도달한 관람객 비율 |
| 평균 체류 시간 | 섹션 내 머문 평균 시간 |
| QR 스캔율 | 섹션 내 정보 탐색 비율 |
| 저장률 | 섹션 내 작품 저장 비율 |
| 감정 태그 집중도 | 특정 감정 반응 집중 여부 |
| 이탈률 | 다음 섹션으로 이동하지 않은 비율 |

### 분석 결과 예시

| 섹션 | 주요 반응 | 해석 |
| --- | --- | --- |
| 작가의 초기 생애 | 체류 낮음, 스캔 낮음 | 정보 전달 방식 개선 필요 |
| 고갱과의 관계 | 체류 높음, 상세 보기 높음 | 관계 서사 반응 강함 |
| 아를의 색채 | 저장 높음, 사진 반응 높음 | 시각/색감 반응 강함 |
| 마지막 편지 | 감정 태그 높음, 재열람 높음 | 감정 몰입 반응 강함 |

### 스핀오프 활용

“고갱과의 관계”, “마지막 편지” 구간의 반응이 높다면 후속 전시는 작품 나열형보다 **관계 서사형, 편지 기반 몰입형 전시**로 제안할 수 있다.

---

## C. 반응 키워드 클러스터 분석

### 기능 설명

관람객이 선택한 감정 태그, 저장한 작품의 메타데이터, 상세 보기 콘텐츠의 주제를 묶어 핵심 반응 키워드 클러스터를 도출한다.

### 클러스터 예시

| 클러스터 | 포함 데이터 |
| --- | --- |
| 작가 관계성 | 고갱, 동료 작가, 편지, 갈등, 우정 |
| 감정 몰입 | 쓸쓸함, 고독, 위로, 불안 |
| 시각 선호 | 노란색, 강렬한 붓터치, 빛, 밤하늘 |
| 시대 배경 | 아를, 파리, 19세기, 병원, 편지 |
| 공간 경험 | 포토존, 몰입형 영상, 사운드, 조명 |

### 분석 결과 예시

> 전체 관람객 반응 중 “작가 관계성”과 “감정 몰입” 클러스터가 높게 나타남.

> 특히 20~30대 관람객은 “편지”, “고독”, “관계” 키워드가 포함된 작품과 설명 콘텐츠에 높은 저장률을 보임.

### 스핀오프 활용

이 경우 후속 스핀오프 방향은 다음처럼 제안할 수 있다.

| 스핀오프 방향 | 설명 |
| --- | --- |
| 서간문 기반 전시 | 작가의 편지와 내면을 중심으로 구성 |
| 관계 서사형 전시 | 고흐와 고갱, 동시대 작가 관계를 중심으로 확장 |
| 감정 몰입형 콘텐츠 | 고독, 위로, 불안을 주제로 한 오디오/영상 콘텐츠 |

---

## D. 관람객 세그먼트별 반응 분석

### 기능 설명

관람객을 행동 패턴에 따라 세그먼트로 나누고, 세그먼트별로 어떤 전시 요소에 반응했는지 분석한다.

초기에는 개인정보 기반 세그먼트보다 행동 기반 세그먼트가 적합하다.

### 행동 기반 세그먼트 예시

| 세그먼트 | 정의 | 특징 |
| --- | --- | --- |
| 깊이 관람형 | 체류 시간과 상세 보기 비율이 높은 관람객 | 작가 서사, 해설 콘텐츠 선호 |
| 이미지 소장형 | 저장과 사진 반응이 높은 관람객 | 색감, 포토존, 굿즈 반응 높음 |
| 공유 확산형 | 공유와 SNS 참여가 높은 관람객 | 화제성 있는 공간 선호 |
| 빠른 관람형 | 체류와 스캔이 낮은 관람객 | 핵심 요약, 직관적 동선 필요 |
| 서사 탐색형 | 관련 콘텐츠 클릭이 높은 관람객 | 관계성, 시대 배경, 작가 이야기 선호 |

### 분석 결과 예시

| 세그먼트 | 높은 반응 요소 | 후속 제안 |
| --- | --- | --- |
| 깊이 관람형 | 편지, 작가 해설, 시대 배경 | 도슨트형 강연, 아카이브 전시 |
| 이미지 소장형 | 색감, 포토존, 굿즈 | 포토 팝업, 한정 굿즈 |
| 공유 확산형 | 몰입형 영상, AR 프레임 | SNS 챌린지형 팝업 |
| 서사 탐색형 | 작가 관계, 갈등, 편지 | 관계 서사형 스핀오프 |

### 고객사 가치

고객사는 “전체 관람객이 무엇을 좋아했는가”뿐 아니라, **어떤 고객군에게 어떤 후속 경험을 제안해야 하는가**를 확인할 수 있다.

## E. 몰입 요소와 확산 요소 분리 분석

### 기능 설명

전시에서 오래 머문 작품과 많이 공유된 작품은 다를 수 있다. 따라서 몰입 요소와 확산 요소를 분리해서 분석해야 한다.

### 분석 기준

| 구분 | 대표 데이터 | 의미 |
| --- | --- | --- |
| 몰입 요소 | 체류 시간, 상세 보기, 재열람 | 깊은 관심 |
| 확산 요소 | 사진 촬영, 공유, 해시태그 | 외부 확산 가능성 |
| 구매 요소 | 굿즈 클릭, 저장, 이미지 선호 | 상품화 가능성 |
| 서사 요소 | 작가 이야기 클릭, 관련 콘텐츠 탐색 | 콘텐츠 확장 가능성 |

### 예시

| 작품/구간 | 몰입도 | 확산성 | 해석 |
| --- | --- | --- | --- |
| 작가의 편지 구간 | 높음 | 낮음 | 깊이 있는 후속 콘텐츠 적합 |
| 대형 미디어아트존 | 중간 | 높음 | 팝업/홍보 콘텐츠 적합 |
| 대표작 포토존 | 낮음 | 높음 | SNS 유입 장치 적합 |
| 작가 관계 설명 구간 | 높음 | 중간 | 스핀오프 전시 주제 적합 |

### 스핀오프 활용

몰입도와 확산성을 구분하면 후속 기획의 목적을 명확히 할 수 있다.

| 목적 | 적합한 스핀오프 |
| --- | --- |
| 깊은 관람 경험 확장 | 도슨트, 강연, 아카이브 전시 |
| 신규 고객 유입 | 포토 팝업, SNS 캠페인 |
| 상품 매출 확대 | 굿즈, 아트 상품, 한정판 |
| 브랜드 가치 강화 | 작가 서사형 콘텐츠, 다큐형 콘텐츠 |

## F. 이탈 구간 및 개선 포인트 분석

### 기능 설명

반응이 낮은 구간을 찾아 전시 구성의 개선 포인트를 제안한다.

이 기능은 스핀오프 제안뿐 아니라 현재 전시 운영 개선에도 활용할 수 있다.

### 분석 항목

| 항목 | 설명 |
| --- | --- |
| 낮은 체류 구간 | 관람객이 빠르게 지나간 구간 |
| 낮은 QR 스캔 구간 | 정보 탐색이 거의 없는 구간 |
| 낮은 저장 구간 | 사후 관심이 낮은 구간 |
| 높은 이탈 구간 | 다음 구간으로 넘어가지 않은 구간 |
| 반복 정체 구간 | 몰입이 아니라 혼잡 때문에 체류가 긴 구간 |

### 개선 제안 예시

| 발견된 문제 | 개선 방향 |
| --- | --- |
| 초반 작가 소개 구간 이탈 높음 | 텍스트 중심 설명을 이미지/타임라인형으로 전환 |
| 중간 섹션 QR 스캔 낮음 | QR 위치, 문구, 혜택 메시지 개선 |
| 마지막 구간 저장률 낮음 | 관람 종료 리포트와 연결되는 CTA 추가 |
| 포토존 체류는 높으나 공유 낮음 | 촬영 프레임, 해시태그 이벤트 추가 |

# 2-6. MVP 기준 우선 구현 범위

초기 런칭 또는 PoC 단계에서는 모든 데이터를 한 번에 수집하기보다, 구현 난이도 대비 활용 가치가 높은 항목부터 시작하는 것이 좋다.

## 1차 MVP 추천 범위

| 우선순위 | 기능 | 이유 |
| --- | --- | --- |
| 1 | 작품/섹션별 QR 스캔 | 설치 부담 낮고 관심 데이터 확보 가능 |
| 2 | 상세 보기/더 알아보기 클릭 | 서사 관심 분석 가능 |
| 3 | 저장하기 | 강한 선호 데이터 확보 가능 |
| 4 | 감정 태그 선택 | 스핀오프 컨셉 도출에 직접 활용 가능 |
| 5 | 관람 후 개인 리포트 | 관람객 참여 동기 강화 |
| 6 | 어드민 기본 분석 | 고객사 가치 확인 가능 |
