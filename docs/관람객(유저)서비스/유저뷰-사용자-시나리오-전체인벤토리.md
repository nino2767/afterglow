## A. 온보딩·입장

**목적**: 관람객이 진입 장벽 없이 30초 안에 서비스를 시작할 수 있어야 하며, 예외 상황(비동의·재방문·오류)에서도 온보딩이 막히지 않아야 한다.
**기대결과**: 어떤 경로로 들어와도 세션 생성 또는 명확한 대안 안내로 귀결되고, 데이터 무결성이 깨지지 않는다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| A1 | 개인정보 동의 거부 → "가이드 없이 관람" 안내 후 세션 미생성 | P1 |
| A2 | 온보딩 중 네트워크 불안정 → 입력값 로컬 보존, 재연결 시 복구 | P1 |
| A3 | 재방문 visitor(동일 기기) → 동의·닉네임 스킵, 키워드만 재확인 | P1 |
| A4 | 미성년 관람객 → 마케팅 opt-in만 연령 제한 | P2 |
| A5 | QR 위조·타 전시 QR → exhibition_id 검증 실패 안내 | P1 |
| A6 🆕 | 로그인 계정이 새 전시 QR 스캔 — 온보딩 스텝 축소 여부 **미정 갭** | P1 (갭) |

## B. 관람 중 (정보카드·AI채팅)

**목적**: 작품 인식·AI 대화가 각종 장애(권한거부·오프라인·AI실패)에도 끊기지 않고, 관람 경험이 유지돼야 한다.
**기대결과**: 어떤 장애 상황에서도 관람객은 대체 경로로 관람을 계속할 수 있고, 데이터(조회·감정)는 과장되거나 유실되지 않는다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| B1 | KB 미등록 작품 QR 스캔 → "준비 중" + 등록 작품 목록 유도 | P1 |
| B2 | 카메라 권한 거부 → 작품 목록(list 경로)으로 자동 대체 | P1 |
| B3 | 전시장 음영(오프라인) → 카드 캐시 표시, interaction 큐잉 | P2 |
| B4 | 동일 작품 연속 스캔 남용 → view 미중복, dwell만 누적 | P2 |
| B5 | 세션 없이 작품 QR 직접 진입 → 온보딩 후 카드로 복귀 | P2 |
| B6 | AI 응답 실패·타임아웃 3회 → Level 1 정보카드로 유도 | P1 |
| B7 | 대화 중 세션 만료 → 재스캔 후 대화 복구 | P2 |
| B8 | 부적절 발화 → 세계관 내 회피, 감정데이터 미기록 | P1 |
| B9 | 장시간 무응답 후 이탈 → 재접속 시 이어짐 | P2 |

## C. 관람 완료·리포트·브릿지

**목적**: 관람 데이터가 많든 적든 항상 리포트와 초대장(또는 대체 수단)으로 귀결돼, 전환 퍼널의 시작점이 끊기지 않아야 한다.
**기대결과**: API 장애·데이터 빈약·팝업 미오픈 등 어떤 상황에서도 빈 화면 없이 다음 단계로 이어진다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| C1 | 데이터 빈약(대화 0회) → 축약 리포트 + default 초대장 | P1 |
| C2 | `/session/complete` API 실패 → 로컬 캐시 부분 렌더 + 재시도 | P1 |
| C3 | 이미 완료된 세션 재진입 → 재발급 없이 기존 리포트 재표시 | P2 |
| C4 | 팝업 미운영 기간(오픈 전) → 오픈알림 opt-in으로 대체 | P1 |

## D. 스핀오프 랜딩·체크인

**목적**: 초대장의 생애주기(오픈전·만료·중복사용) 전반에서 관람객이 헤매지 않고, 퍼널 지표가 조작되지 않아야 한다.
**기대결과**: 어떤 초대장 상태에서도 명확한 다음 행동이 제시되고, 체크인·구매 이벤트는 정확히 유효 범위 내에서만 집계된다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| D1 | invite_id 만료(팝업 종료 후) → 아카이브 + 차기전시 opt-in | P1 |
| D2 | 팝업 오픈 전 접근 → 카운트다운 + 오픈알림 opt-in | P2 |
| D3 | 이미 체크인한 재방문 → 중복 `redeemed` 방지 | P1 (데이터 무결성) |
| D4 | 외부 커머스 링크 오류 → 대체 링크(현장구매 안내) | P2 |
| D5 | C경로에서 "관람했어요" 탭 → 세션조회 후 A경로 승격 | P2 |

## E. 팝업 현장 (큐레이터)

**목적**: 본전시 이력 유무·접근 경로(QR/리스트)와 무관하게 팝업 체험이 동일한 가치로 제공·집계돼야 한다.
**기대결과**: 모든 방문자가 정상적으로 큐레이터·존 콘텐츠를 이용하고, 접근 경로 차이가 데이터 집계를 왜곡하지 않는다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| E1 | main 세션 없는 현장 방문자 → 축약 온보딩, 기본 해설 모드 | P1 |
| E2 | 존 QR 미스캔, 리스트 탭만 탐색 → 동일하게 집계 | P2 |
| E3 | 팝업 혼잡·오프라인 → 존 해설 캐시 표시 | P2 |
| E4 | 체크인 없이 URL 직접 접근 → 화면5로 리다이렉트 | P1 |
| E5 | concept_type 6종별 큐레이터 톤 정상 동작 | P1(전체)/P2(개별) |
| E6 | community 롤 타관람객 문장 인용 opt-in 처리 | P2 |

## F. 계정·로그인

**목적**: 로그인은 선택 사항이어야 하며, 전환·중복가입·로그아웃 등 어떤 계정 상태 변화도 서비스 이용을 막지 않아야 한다.
**기대결과**: 게스트·회원 전환이 매끄럽고(리로딩 없음), 중복 계정이나 데이터 유실 없이 항상 이용 가능한 상태를 유지한다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| F1 | 비로그인 재접속(같은 기기) → 게스트로도 재열람 허용 | P1 |
| F2 | 카카오 가입 이메일로 중복 가입 시도 → 기존 계정 안내 | P2 |
| F3 | 로그아웃 후 게스트로 계속 이용 | P2 |
| F4 | 게스트→계정 병합 → 리로딩 없이 토스트만 표시 | P1 |

## G. 마이페이지·홈 (계정 모드)

**목적**: 계정 모드는 정보 과부하 없이 "지금 중요한 것"만 보여주고, 여러 기기·여러 전시를 다뤄도 일관돼야 한다.
**기대결과**: 홈은 항상 1건 이하의 우선순위 카드만 노출하고, 크로스디바이스 조회 결과가 어디서나 동일하다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| G1 | "지금 확인할 것" 0건, 리포트는 있음 → 섹션 미노출 | P2 |
| G2 | 여러 전시 초대장 동시 활성 → 최우선 1건만, 동순위는 임박순 | P1 |
| G3 | 화면9 종료 그룹 다수 누적 → 스크롤 성능 | P3 |
| G4 | 동일 계정 여러 기기 동시 로그인 → 조회 일관성 | P1 |

## H. 스핀오프 팝업 생애주기 (운영정책 연동)

**목적**: 팝업 운영 정책(21일 상한·리드타임 등)이 유저뷰에 자동 반영되고, B2B 영역과의 책임 경계가 명확해야 한다.
**기대결과**: 상태 전이는 수동 개입 없이 자동 반영되며, B2B 전용 예외는 유저뷰 동작에 영향을 주지 않는다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| H1 | 팝업 단독운영 21일 상한 도달 → 자동 전이, 재접속 시 반영 | P1 |
| H2 | 파일럿 조건 미달 전시에서 팝업 개설 시도 | P3 |
| H3 | 최소 리드타임 미달로 오픈 지연 | P3 |

## I. 데이터 무결성

**목적**: 감정·퍼널·유입경로 데이터가 승인·집계 규칙을 어기지 않고 정확하게 쌓여야 MVP 핵심 지표(전환율 20%)를 신뢰할 수 있다.
**기대결과**: 미승인 데이터는 절대 노출되지 않고, 열람·재방문·비개인화 유입이 핵심 지표를 오염시키지 않는다.

| ID | 시나리오 | 우선순위 |
|---|---|---|
| I1 | 감정 leaf `ai_extracted`·승인대기 → 관람객 화면 미노출 | P1 |
| I2 | 열람 모드에서 퍼널 이벤트 재발화 여부 → 미발화 확인 | P1 |
| I3 | C경로(`landing.organic_viewed`) 분리 집계 여부 | P1 |
