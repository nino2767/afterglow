---

## 1. 화면의 역할

| 관점 | 역할 |
|---|---|
| 관람객(B2C) | 30초 안에 부담 없이 시작 — 앱 설치·회원가입 없이 닉네임+키워드만 |
| 데이터 | VISITOR 생성/조회, SESSION 시작, 취향 시드(`initial_keywords`) 확보 |
| 퍼널 | `session.start` (전환율 20%의 **분모** 확정 지점) |
| 벤치마킹 차별화 | 큐피커류 "일방향 온보딩"이 아니라, 첫 입력부터 감정 데이터 수집 시작 |

**설계 원칙 — "30초 룰":** 전시 입구는 줄 서는 곳. QR 스캔부터 첫 작품 앞 도착까지 30초를 넘기면 이탈. 필수 입력은 **닉네임 1개 + 키워드 탭 2~3개**가 전부, 나머지는 전부 후순위·생략 가능.

## 2. 진입 · 이탈 플로우

```
[전시장 입구 QR 스캔] → 모바일 웹 오픈 (앱 설치 없음)
   └─ ① 웰컴 (전시 키 비주얼 + 서비스 한 줄 소개)
        └─ ② 동의 (개인정보·마케팅 opt-in)
             └─ ③ 닉네임 입력
                  └─ ④ 기대 키워드 선택 (감정 leaf 칩 탭)
                       └─ session.start 발화 → [화면2] 진입
이탈/우회
   ├─ 동의 거부 → "가이드 없이 관람하기" 안내 후 종료 (강요 없음)
   ├─ 재방문(기존 visitor) → 닉네임 인식 "다시 오셨네요" → ④만 갱신 후 즉시 입장
   └─ 출구 QR로 뒤늦게 진입(비온보딩) → 축약판(②+④만, 닉네임 자동생성) — 화면4 회수 경로
```

## 3. 화면 구성 (모바일, 스텝 4단 — 각 1스크린)

```
① 웰컴                          ④ 기대 키워드
┌──────────────────────┐      ┌──────────────────────┐
│ [전시 키 비주얼]        │      │ 오늘 어떤 기분을        │
│ 빛의 심연               │      │ 만나고 싶으세요?        │
│ "AI 큐레이터와 함께      │      │ ┌────┐┌────┐┌────┐ │
│  당신만의 관람을"        │      │ │고요함││몽환 ││강렬함│ │
│ [시작하기]              │      │ └────┘└────┘└────┘ │
└──────────────────────┘      │ │따뜻함││신비 ││해방감│ │
                               │ (전시별 EMOTION_LEAF   │
② 동의: 필수(서비스)/선택(마케팅) │  active 중 6~8개 노출) │
③ 닉네임: 1필드 + 랜덤 생성 버튼  │ [2~3개 선택] [입장 →]  │
                               └──────────────────────┘
```

### 구성 요소 상세

| 스텝 | 요소 | 동작 | 데이터 |
|---|---|---|---|
| ① | 웰컴 | 전시 키 비주얼(운영자 Phase 1 자산) + 1줄 가치 제안. CTA 1개만 | 읽기: EXHIBITION |
| ② | 동의 | 필수: 서비스 이용(세션 데이터 수집) / 선택: 마케팅(차기 전시·팝업 알림). 각 1줄 요약+전문 링크 | 쓰기: VISITOR.consent |
| ③ | 닉네임 | 1필드. "랜덤 이름 받기" 버튼(입력 부담 제거). 중복 허용(visitor_id가 키) | 쓰기: VISITOR.nickname |
| ④ | 키워드 | 해당 전시 EMOTION_LEAF(active) 중 운영자 지정 6~8개 칩. **2~3개 탭 선택.** `#`은 표시만, 저장은 `{emotion, axis}` | 쓰기: SESSION.initial_keywords |
| 완료 | — | `session.start` 발화 + 화면2 자동 진입 (완료 화면 없음 — 30초 룰) | §3 퍼널 1단계 |

## 4. 상태 · 엣지 케이스

| 상태 | 처리 |
|---|---|
| 동의 거부 | 세션 미생성. "가이드 없이 관람" 안내 + 언제든 QR 재스캔 가능 고지 |
| 네트워크 불안정 | 스텝 입력값 로컬 보존, 재시도. QR 재스캔 시 이어서 진행 |
| 재방문 visitor (동일 기기) | ①~③ 스킵, ④만 재확인 → 새 SESSION 생성 (전시 재방문도 새 세션) |
| 미성년 관람객 | 별도 분기 없음 — 수집 항목이 닉네임+감정 키워드뿐이라 최소화 원칙으로 커버. 마케팅 opt-in만 만 14세 미만 불가 문구 |
| QR 위조/타 전시 QR | entry_qr의 exhibition_id 검증 실패 시 안내 화면 |

## 5. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| QR → 웹앱, 4스텝 30초 온보딩 | 소셜 로그인·계정 연동 |
| 닉네임 + 키워드 2~3개 | 상세 취향 설문 |
| 재방문 인식 (기기 기반) | 크로스 기기 visitor 통합 |
| 한국어 | 다국어 (v3.0 §9 OUT) |

## 6. 남은 결정 포인트

1. **키워드 선택 강제 여부** — 0개로도 입장 허용할지. (현안: 최소 1개 강제 — 개인화 시드 없이는 화면3 첫마디·화면4 매칭이 전부 default로 떨어져 서비스 가치가 죽음. 대신 "잘 모르겠어요" 칩 1개를 두어 심리 부담 완화)
2. **랜덤 닉네임의 결** — 전시 세계관 단어 조합(예: "고요한 윤슬") vs 일반 조합. (현안: 세계관 조합 — persona_copy와 결 통일)
