# 유저뷰 상세기획 — 화면 4: 여운 리포트 + Bridge 초대장

> 버전: v1.0 | 작성일: 2026.07.02 | 상태: 📋 초안
> 근거: v3.0 §2(팝업 브릿지)·§5(Bridge Trigger)·§6(사용자 여정 3단계), 데이터 계약 SSOT, B2C-B2B API 명세 `/session/complete`
> 위치: 유저뷰 5화면 중 4번. **관람 경험을 "소장하고 싶은 것"으로 바꾸고, 팝업 전환(퍼널 3단계)을 여는 화면.** 서비스명 AFTERGLOW의 존재 이유가 구현되는 지점.

---

## 1. 화면의 역할

| 관점 | 역할 |
|---|---|
| 관람객(B2C) | 오늘의 관람을 "나만의 취향 리포트"로 요약받고, 여운을 이어갈 팝업 초대를 받음 |
| 데이터(Bridge) | TASTE_PROFILE 확정 → concept_type 매칭 → SPINOFF_INVITE 발급 (`invite.issued`) |
| 스토리 모듈 | 대화에서 쌓인 감정·서사를 전시 세계관 언어(persona_copy)로 되돌려줌 |
| 기획사(B2B) | 전환율 20% 목표(v3.0 9장)의 시작점 — 초대장 CTR이 여기서 결정됨 |

**설계 원칙:** 리포트가 주인공, 초대장은 선물. "당신을 분석했다"가 아니라 **"당신의 여운을 정리해뒀다"**로 읽혀야 함. 판매 화면처럼 보이는 순간 전환율이 죽는다.

## 2. 진입 · 이탈 플로우

```
진입 (3경로)
  A. 화면2/3 전역 하단 "관람 완료" 버튼 (주 경로)
  B. 출구 QR 스캔 (전시장 출구 배치 — 온보딩 QR과 별개 코드)
  C. 세션 무활동 90분 경과 → 다음 앱 접근 시 자동 진입 (놓친 사람 회수)

이탈
  ├─ [초대장 CTA] → 화면5 스핀오프 랜딩 (invite.landing_viewed)
  ├─ [리포트 저장/공유] → 이미지 저장 → 화면4 유지
  └─ [닫기] → 세션 종료. 초대장은 landing_url로 재접근 가능 (유효기간 내)
```

- 진입 시 `/session/complete` 호출 → 응답으로 리포트 데이터 + 초대장 수신. **이 시점에 `invite.issued` 발화** (퍼널 3단계).
- B(출구 QR) 진입도 동일 처리 — 세션 미보유자(비온보딩 관람객)는 온보딩 축약판으로 우회 (§5 엣지).

## 3. 화면 구성 (모바일, 세로 스크롤 1페이지)

```
┌──────────────────────────────┐
│  ✨ 오늘의 여운                │  ① 헤더
│  {nickname}님의 빛의 심연      │
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │   [취향 카드]              │ │  ② 페르소나 카드 (공유 단위)
│ │   persona_copy            │ │     "Deep Ocean Dreamer"
│ │   color_palette 그라디언트 │ │     축 비중 기반 배경색
│ │   top_keywords 3종 (#leaf)│ │
│ └──────────────────────────┘ │
│  [이미지로 저장] [공유]        │
├──────────────────────────────┤
│  📍 오늘 가장 머문 순간        │  ③ 관람 하이라이트
│  · '윤슬' — 12분             │     dwell 상위 1~2 작품
│  · 대화에서 가장 많이 말한 감정│     axis 상위 1종 + leaf
│    → #고요함 (serene)        │
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ 💌 초대장                  │ │  ④ Bridge 초대장
│ │ personalized_copy         │ │     concept_type 매칭 결과
│ │ "당신이 머문 그 푸른 빛,    │ │
│ │  팝업에서 향으로 만나보세요" │ │
│ │ [팝업 구경하기 →]          │ │     CTA → 화면5
│ │ 유효기간 · 위치 요약        │ │
│ └──────────────────────────┘ │
└──────────────────────────────┘
```

### 구성 요소 상세

| # | 요소 | 동작 | 데이터 |
|---|---|---|---|
| ② | 페르소나 카드 | persona_copy(전시별 네이밍) + axis 상위 2종 그라디언트 배경 + top_keywords 칩. **저장/공유 시 이 카드만 이미지화** (바이럴 단위) | TASTE_PROFILE: persona_copy, axis_distribution, color_palette, top_keywords |
| ③ | 하이라이트 | dwell 상위 작품 1~2 + 최다 감정 axis/leaf. 3줄 이내 | ARTWORK_INTERACTION, AI_CONVERSATION 집계 |
| ④ | 초대장 | personalized_copy(개인 대화 인용형) + CTA + 유효기간·위치. CTA 탭 → landing_url 이동 | SPINOFF_INVITE: concept_type, personalized_copy, landing_url |

- **공유 카드에 초대장 미포함** — 공유는 자기표현이지 광고 전달이 아님. 단 카드 하단에 전시명+AFTERGLOW 워터마크(자연 유입용).

## 4. Bridge 매칭 로직 (SSOT §5 구체화)

```
입력: TASTE_PROFILE (axis_distribution, segment_tag, top_keywords)
매칭: 전시의 활성 팝업 콘텐츠 풀(ConceptBot 확정안) 중
      concept_type별 axis 친화도 테이블로 스코어링 → 최고점 1종 선택
출력: SPINOFF_INVITE { concept_type, personalized_copy, landing_url, barcode }
```

### concept_type × axis 친화도 (v1 초기값 — 파일럿 데이터로 튜닝)

| concept_type | 주 친화 axis | 부 친화 |
|---|---|---|
| `story` | contemplative, melancholy | dreamy |
| `sensory` | serene, warm | dreamy |
| `archive` | melancholy, contemplative | serene |
| `community` | warm, thrill | awe |
| `role` | thrill, awe | tension |
| `time` | dreamy, contemplative | melancholy |

- 전시에 매칭 concept_type 팝업이 없으면 → 운영 중인 컨셉 중 차선책 매칭 (초대장 미발급 케이스 없음).
- **personalized_copy 생성 규칙:** [대화 속 실제 발화/leaf 1개 인용] + [매칭 콘텐츠 연결] + [행동 유도]. 예: "'고요한 블루'라고 하셨죠 — 그 색을 향으로 만든 인센스가 기다려요." AI 생성 + KB 톤가이드 적용, 30자 내외 2문장 상한.

## 4-1. 🆕 열람 모드 (계정 모드에서 재진입 시)

> 근거: 유저뷰 IA v1.0 §2. 화면8(마이리포트) 카드 탭 시 본 화면을 재사용하되 아래만 다르게 동작한다. 신규 화면이 아니다.

| 요소 | 라이브(가이드 모드) | 열람 모드 |
|---|---|---|
| 상단 배너 | 없음 | "보관함에서 보는 중" 얇은 배너 1줄 |
| 뒤로가기 | 화면2/3(이전 관람 화면)로 | **화면8(마이리포트)**로 |
| 페르소나 카드·하이라이트 | 동일 | 동일 (읽기 전용, 변화 없음) |
| 초대장 CTA | 정상 노출 | 유효기간 내면 동일 노출·클릭 가능 (초대장 자체는 계속 유효) |
| 이벤트 발화 | `invite.issued` 등 정상 발화 | **발화 없음** (조회일 뿐, 퍼널 오염 방지) |

## 5. 상태 · 엣지 케이스

| 상태 | 처리 |
|---|---|
| 데이터 빈약 (대화 0회, 체류만 존재) | 페르소나 카드 대신 "오늘의 발자국" 축약 리포트 (dwell 기반). 초대장은 default concept(운영자 지정 1종)으로 발급 |
| 비온보딩 관람객이 출구 QR 스캔 | 축약 온보딩(키워드 3개만) → 즉석 미니 리포트 + default 초대장. `session.start`+`invite.issued` 동시 발화 (놓친 관람객 회수 경로) |
| 팝업 미운영 기간 (전시 초반 등) | 초대장 = "오픈 알림 받기" (연락처 opt-in)로 대체. `invite.issued`는 동일 발화 |
| /session/complete 실패 | 로컬 캐시로 리포트 부분 렌더 + 초대장 영역 재시도 버튼 |
| 재진입 (이미 완료된 세션) | 발급된 리포트·초대장 그대로 재표시 (재발급 없음, 유효기간 표시) |
