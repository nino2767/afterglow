# 유저뷰 상세기획 — 화면 6: 팝업 AI 큐레이터 (스핀오프 현장 경험)

> 버전: v1.0 | 작성일: 2026.07.02 | 상태: 📋 초안
> 근거: v3.0 §2(2차 **체험 공간**)·§6(여정 4단계 "데이터 연동 체험")·컨셉 샘플 A/B/C, SSOT v1.1 §1·§3-1·§6
> 위치: 화면 5(랜딩)에서 **셀프 체크인 이후**의 현장 경험. 스핀오프 팝업을 "굿즈샵"이 아니라 **본전시의 여운이 이어지는 미니 전시**로 만드는 화면.

---

## 1. 화면의 역할

| 관점 | 역할 |
|---|---|
| 관람객(B2C) | 본전시에서 나눈 대화·취향을 **기억하는** AI가 팝업 존을 안내하고 체험을 해설 |
| 데이터 | `spinoff.session_start` / `zone_viewed` / `curator_chat` 발화, TASTE_PROFILE 갱신(main 프로필 이어받기) |
| 스토리 모듈 | 본전시 세계관이 팝업 존 스크립트로 연장 — "전시가 끝난 후에도 남는 빛"의 실체 |
| 기획사(B2B) | 존별 인기·체류 데이터 → Insight Reporter의 차기 전시 인풋. 커스텀 MD 구매 유도 접점 |

**설계 원칙:** 팝업 큐레이터의 첫마디가 **본전시의 기억**에서 시작해야 한다. "어서오세요"가 아니라 **"다시 만났네요 — 그 고요한 블루, 여기 있어요."** 이 연속성이 일반 팝업스토어와의 결정적 차이(v3.0 아르떼뮤지엄 대비 차별화 근거).

---

## 2. 진입 · 이탈 플로우

```
[화면5: 셀프 체크인 완료]
   └─ "팝업 큐레이터 시작하기" (체크인 완료 화면의 주 CTA)
        └─ [화면6] spinoff.session_start
             ├─ 존 QR 스캔 → 존 상세·해설 (zone_viewed)
             ├─ AI 대화 (curator_chat) — 화면3과 동일 UI 재사용
             ├─ 존 연계 MD → 구매 링크 (purchase.link_clicked, spinoff 소속)
             └─ "오늘의 여운 마무리" → 미니 리포트 → 종료
```

- **진입 조건**: `invite.redeemed` 상태 + main 세션 보유. 비보유자(C경로 현장 유입)는 축약 온보딩 → 취향 기억 없는 기본 모드로 동작.
- 팝업 세션은 SSOT v1.1대로 `exhibition_id = spinoff`의 **새 SESSION** — 단 TASTE_PROFILE은 visitor 기준으로 main 프로필을 이어받아 갱신.

## 3. 화면 구성

```
┌──────────────────────────────┐
│ 🌊 어비스 티 라운지            │  ① 팝업 헤더 (spinoff title)
│ "다시 만났네요, {nickname}님"  │     본전시 기억 기반 웰컴
├──────────────────────────────┤
│ 🗺 이 공간의 존                │  ② 존 맵 (리스트형, MVP)
│ ┌──────────────────────────┐ │
│ │ ● 딥블루 바 (sensory)     │ │   ← 내 axis 친화 존에
│ │   ✨ S님 취향 추천         │ │      "취향 추천" 뱃지
│ │ ○ 기억의 선반 (archive)   │ │
│ │ ○ 파도의 방 (story)       │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│ [존 상세 — 탭 or QR 스캔 시]   │  ③ 존 해설
│ 체험 설명 + AI 도슨트 한마디    │     본전시 대화 인용형
│ "아까 '윤슬'에서 12분 머무셨죠. │
│  이 바의 조명이 그 빛이에요."   │
│ 연계 MD 카드 → [구매]          │
├──────────────────────────────┤
│ 💬 큐레이터에게 물어보기        │  ④ AI 채팅 (화면3 UI 재사용)
├──────────────────────────────┤
│ [오늘의 여운 마무리]            │  ⑤ 미니 리포트 → 종료
└──────────────────────────────┘
```

### 구성 요소 상세

| # | 요소 | 동작 | 데이터 |
|---|---|---|---|
| ① | 웰컴 | main TASTE_PROFILE 기반 개인화 첫마디 (top_keyword 1개 인용) | 읽기: TASTE_PROFILE(main) |
| ② | 존 맵 | SPINOFF_ZONE 리스트. 내 axis_distribution과 존 concept_type 친화도(화면4 테이블 재사용) 상위 존에 "취향 추천" 뱃지 | 읽기: SPINOFF_ZONE, TASTE_PROFILE |
| ③ | 존 해설 | 존 QR 스캔 or 리스트 탭 → `spinoff.zone_viewed`. 해설은 존 docent_script + **본전시 데이터 인용**(dwell·감정) 결합. 연계 MD 카드 노출 | 쓰기: zone_viewed / 읽기: ARTWORK_INTERACTION(main) |
| ④ | AI 채팅 | 화면3 컴포넌트 재사용. RAG 소스 = spinoff KB(본전시 KB 상속 + 존 스크립트). `spinoff.curator_chat` | 쓰기: AI_CONVERSATION(spinoff), TASTE_PROFILE 갱신 |
| ⑤ | 미니 리포트 | 팝업에서의 반응 요약 1카드 + "차기 전시 소식 받기" opt-in | 쓰기: 세션 종료 |

## 4. 상태 · 엣지 케이스

| 상태 | 처리 |
|---|---|
| main 세션 없는 현장 방문자 | 축약 온보딩(키워드 3개) → 기억 인용 없는 기본 해설 모드. 존 맵·채팅은 동일 제공 |
| 존 QR 미스캔·리스트만 탐색 | 리스트 탭도 `zone_viewed`로 동일 집계 (viewed_via: qr\|list 구분) |
| 팝업 혼잡·오프라인 | 화면3과 동일 오프라인 처리. 존 해설은 최초 로드 시 캐시 |
| 체크인 없이 URL 직접 접근 | 화면5로 리다이렉트 (체크인 유도) |

## 5. 이벤트 · 데이터 계약 매핑 (SSOT v1.1 §3-1)

## 6. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| 존 리스트 맵 + 취향 추천 뱃지 | 실내 위치 기반 자동 해설 |
| 본전시 기억 인용 해설 (텍스트) | AI 편집 관람 영상·커스텀 패턴 실시간 생성 |
| 화면3 채팅 재사용 | 팝업 전용 대화 페르소나 분리 |
| 미니 리포트 + 차기 전시 opt-in | NFT·디지털 아카이브 발급 |

---

## 7. 남은 결정 포인트

1. **존 수 MVP 상한** — ConceptBot 산출 기준 존 3~5개 권장 (본전시 핵심 작품 3~5개 대칭). 파일럿 팝업 규모에 맞춰 확정 필요.
2. **미니 리포트의 무게** — 1카드 요약(현안) vs 화면4 수준 풀 리포트. 팝업 체류가 짧아 1카드가 적절하다고 봄.
