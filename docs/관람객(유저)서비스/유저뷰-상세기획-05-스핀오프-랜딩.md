# 유저뷰 상세기획 — 화면 5: 스핀오프 랜딩

> 버전: v1.0 | 작성일: 2026.07.02 | 상태: 📋 초안
> 근거: v3.0 §2(스핀오프 팝업)·§7(전시×팝업 오버랩)·§9(MVP IN: 스핀오프 랜딩 페이지 / OUT: 자체 결제), 데이터 계약 SSOT §3·§5
> 위치: 유저뷰 5화면 중 5번. **퍼널 4·5·6단계(`invite.landing_viewed` → `invite.redeemed` → `purchase.link_clicked`)가 모두 이 화면에서 발화.** 수익(Revenue) 모델의 B2C 접점.

---

## 1. 화면의 역할

| 관점 | 역할 |
|---|---|
| 관람객(B2C) | 내 취향으로 추천된 팝업 체험·굿즈를 확인하고, 방문·구매로 이동 |
| 데이터(Bridge) | 퍼널 4·5·6단계 이벤트 발화 — MVP 전환율 20% 판정(4단계/1단계)의 분자 확정 지점 |
| 기획사(B2B) | 전시 종료 후 수익 연장(D+80)의 실제 매출 접점. RS 정산 근거 데이터 생성 |
| AFTERGLOW | 단독 운영 기간 신규 고객 유입 채널 (v3.0 §7) |

**설계 원칙:** 초대장에서 이어진 **"나를 위해 준비된 곳"** 느낌 유지. 일반 이커머스 상세페이지가 아니라, 관람 여운의 연장선. 개인화 요소(내 감정·내 대화 인용)가 첫 화면에 반드시 보여야 함.

---

## 2. 진입 경로 (3종 — 개인화 수준이 다름)

| 경로 | 유입 | 개인화 | 퍼널 |
|---|---|---|---|
| **A. 초대장 CTA** | 화면4 → `landing_url?invite_id=` | 풀 개인화 (concept_type + personalized_copy + 추천 MD) | `invite.landing_viewed` |
| **B. 재방문** | 저장된 landing_url 직접 접근 | A와 동일 (invite_id 유효 시) | 재발화 없음 (최초 1회만 카운트) |
| **C. 일반 유입** | 팝업 홍보 QR·SNS·검색 (invite 없음) | 비개인화 — 전시 소개 + 컨셉 전체 + "관람하셨나요?" 세션 연결 유도 | 별도 이벤트 `landing.organic_viewed` (퍼널 외 참고 지표) |

- C 경로는 v3.0 §7 "단독 운영 기간 놓친 사람 유입"의 수용 지점. 초대장 없어도 팝업 정보·구매 링크는 제공하되, 퍼널 집계에선 분리 (전환율 20%는 A 기준).

## 3. 화면 구성 (모바일, 세로 스크롤)

```
┌──────────────────────────────┐
│ [키 비주얼 — concept_type별]   │  ① 히어로
│ "고요한 블루를 담은 향"        │     personalized_copy 재노출
│ {nickname}님을 위한 큐레이션   │     (A/B 경로만)
├──────────────────────────────┤
│ 💠 추천 체험 · 굿즈            │  ② 개인화 추천 (2~3종)
│ ┌────────┐ ┌────────┐       │     MD Matcher 결과 중
│ │AI 커스텀│ │사운드   │       │     내 axis 친화 상위
│ │인센스   │ │LP 코스터│       │
│ │₩18,000 │ │₩12,000 │       │
│ └────────┘ └────────┘       │
│ [예약/구매하러 가기 →]         │  ③ 구매 CTA → 외부 커머스
├──────────────────────────────┤
│ 📍 팝업 정보                  │  ④ 방문 정보
│ 위치 · 지도 · 운영기간(D+80)   │
│ ┌──────────────────────────┐ │
│ │ 🎫 도착하셨나요?           │ │  ⑤ 셀프 체크인
│ │ [팝업 도착 체크인]          │ │     탭 → invite.redeemed
│ │ (체크인 시 웰컴 혜택)       │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│ 나의 취향 카드 다시 보기        │  ⑥ 화면4 리포트 재진입 링크
└──────────────────────────────┘
```

### 구성 요소 상세

| # | 요소 | 동작 | 데이터 |
|---|---|---|---|
| ① | 히어로 | concept_type별 키 비주얼(운영자 Phase 4 산출물) + personalized_copy. C 경로는 전시 공통 비주얼 + 일반 카피 | SPINOFF_INVITE, 팝업 콘텐츠 풀 |
| ② | 추천 MD | invite의 concept_type 소속 MD 중 axis 친화 상위 2~3종. 가격·이미지·한줄 설명 | MD 라인업(ConceptBot 확정안), TASTE_PROFILE.axis_distribution |
| ③ | 구매 CTA | 탭 → **`purchase.link_clicked`** 발화 후 외부 커머스/예약 페이지로 이동 (v3.0 MVP OUT: 자체 결제 없음) | §3 퍼널 6단계 |
| ⑤ | 셀프 체크인 | 탭 → 위치 확인 다이얼로그("팝업에 도착하셨나요?") → 확인 시 **`invite.redeemed`** + 웰컴 혜택 바코드 표시 | §3 퍼널 5단계 (MVP=self_checkin) |
| ⑥ | 리포트 재진입 | 화면4 재표시 (재발급 없음) | — |

- **체크인 웰컴 혜택**: 허위 체크인 방지 겸 방문 동기. 현장에서만 쓸 수 있는 혜택(음료 1잔, 굿즈 5% 등 — 운영자 설정)으로, 버튼만 누르고 안 온 사람에겐 무의미하게 설계. 혜택 바코드가 사실상의 방문 검증.

## 4. 상태 · 엣지 케이스

| 상태 | 처리 |
|---|---|
| invite_id 만료 (팝업 종료 후) | "팝업은 끝났지만" 화면 — 아카이브 카드 재열람 + 차기 전시 알림 opt-in |
| 팝업 오픈 전 접근 | 카운트다운 + 오픈 알림 opt-in. `landing_viewed`는 정상 발화 |
| 이미 체크인한 재방문 | 체크인 버튼 → 혜택 바코드 재표시 (중복 `redeemed` 없음) |
| 외부 커머스 링크 오류 | 대체 링크(현장 구매 안내) 표시. `link_clicked`는 탭 시점 발화라 무관 |
| C 경로에서 "관람했어요" 탭 | 세션 조회(닉네임/QR 재스캔) → 있으면 A 경로로 승격, 없으면 화면4 축약 온보딩과 동일 |

## 5. 이벤트 · 데이터 계약 매핑

| 트리거 | 이벤트 | SSOT 근거 |
|---|---|---|
| A 경로 최초 진입 | `invite.landing_viewed` (invite당 1회) | §3 퍼널 4단계 |
| 셀프 체크인 확인 | `invite.redeemed` (invite당 1회) | §3 퍼널 5단계 |
| 구매/예약 CTA 탭 | `purchase.link_clicked` (복수 허용) | §3 퍼널 6단계 |
| C 경로 진입 | `landing.organic_viewed` (참고 지표) | 퍼널 외 |

공통 페이로드: `{ event, exhibition_id, session_id?, invite_id?, ts }` — C 경로는 session/invite 없이 발화 가능.

## 6. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| 개인화 랜딩 (A/B 경로) + 일반 랜딩 (C 경로) | AI 편집 관람 영상 확인 (v3.0 여정 4단계 일부) |
| 추천 MD 2~3종 + 외부 커머스 링크 | 자체 결제·장바구니 |
| 셀프 체크인 + 웰컴 혜택 바코드 | 현장 QR 스캔 검증 (`qr_scan` 모드, v2 고객사 옵션) |
| 팝업 정보·지도 | 방문 예약 시간 슬롯 |
| 오픈 전/종료 후 상태 화면 | 재고 실시간 연동 |
