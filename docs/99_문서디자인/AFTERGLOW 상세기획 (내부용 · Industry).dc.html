01재정의된 본질 (상위기획 결과)

한 줄 정의
AFTERGLOW는 전시 IP의 세계관을 확장해 '두 번째 작품(스핀오프)'으로 만들어내는 서비스다. 본전시를 봤든 안 봤든 그 자체로 성립하는 독립적인 작품이다.
“본전시의 연장 장치” → “세계관을 공유하는 독립된 두 번째 작품”
본전시 세계관의 확장(독립된 두 번째 작품) · 정보가 아닌 경험 · 여운·개인화는 정체성이 아닌 '강화 레이어'
⚠️ 우선순위 명확화 (2026.07.16): 1순위 정체성 = 본전시의 새로운 세계관(독립 작품). 여운·개인화는 이를 대체하지 않는다.
원점 (동기)
  • 영화/예능의 '스핀오프' 개념을 전시로 — 본편이 끝나도 세계관이 이어진다
  • 개인 경험: 전시 후 작가·작품을 검색해도 정보 파편만 있고 허전했음 (이해+감동+연결 모두 미해소)
  • 시장 공백(⚠️ 재검증 필요): "전시/미술은 IT 낙후"라는 초기 인상은 수정 필요. 관람 중 가이드 영역은 이미 경쟁 존재 — 오프라인 도슨트 보편화됨

핵심 동작: 전시 IP(작품, 작가 세계관)를 재료로, AI가 새로운 세계관 경험 — 이어지는 대화, 미니 스핀오프 전시, 커스텀 굿즈 — 을 생성한다

B2B의 위상: 본질이 아니라 전달 경로. 세계관의 주인이 기획사이므로 그들을 통해 관람객에게 닿는다.

03MVP 가설

“전시가 끝난 후에도 관람객은 그 세계관에 머물고 싶어하며, 스핀오프 경험(이어지는 대화 + 개인 아카이브 + 미니 오프라인 체험)이 '검색'보다 그 욕구를 더 잘 충족시킨다.”

검증 플로우

  1. 1관람 중 — 작품 사진 촬영이 기본 진입점 — 작품을 찍으면 AI가 사진을 인식해 설명 제공 + AI 큐레이터 대화 시작 (사진·대화 자동 저장)
  2. 2퇴장 시점 — 사진+대화가 자동으로 나만의 아카이브(리뷰 초안)로 생성 → 두 경로를 독립적으로 초대 (① 오프라인 미니 전시 ② 온라인 재접속)
  3. 3[오프라인 경로] — 미니 스핀오프 전시(전시장 내/인근 1개 공간): 굿즈숍이 아니라 '또 하나의 작은 전시'를 보는 경험 — 세계관이 확장된 새로운 장면
  4. 4[온라인 경로] — 관람 후 (D+1~7): 웹앱 재접속 → 세계관 대화 계속, 아카이브 확인·공유

온라인/오프라인 분리 검증 설계

두 경로는 서로 독립 — 부스를 안 가도 온라인 스핀오프는 온전히 즐길 수 있고, 그 역도 성립. 결과 교차표로 다음 방향을 정한다.

온라인 재접속부스 방문해석 → 다음 방향
↑↑풀 스핀오프 검증 → 정식 팝업 확장 + 디지털 병행
↑↓머무름 욕구는 디지털로 충족 → 디지털 스핀오프 중심, 팝업은 선택 상품
↓↑오프라인 체험이 본체 → 팝업 중심(v3.0 원형에 근접), 웹은 브릿지로 축소
↓↓본질 가설 재검토 (상위기획 복귀)

기능 범위 IN / OUT

IN
  • 웹앱형 AI 큐레이터 (핵심 작품 3~5개 대화 해설) — v3.0 유지, 단 진입은 사진 인식 기반(QR은 보조, 촬영 불가 시 작품 선택 폴백)
  • 사진 촬영 → 자동 아카이브/리뷰 초안 생성 — 신규
  • 관람 후에도 이어지는 세계관 대화 (작가·작품 심층 대화) — 신규, 본질 직결
  • 미니 스핀오프 전시 1개 공간 (세계관 기반 새 장면 관람 + 출구 굿즈 1~2종)
  • 상설 데모 전시 1개 (v3.0 '심해의 환상' 시뮬레이션의 실구현) — 홈 라인업 채움 + 고객사 세일즈 데모 + 장면 제작 파이프라인 테스트베드
  • 내부용 AI 기획 어시스턴트 (부스 컨셉·굿즈 생성용, 노출 안 함)
OUT
  • 정식 팝업 공간 운영 대행 (21일 연장 운영 등) → 검증 후
  • 네이티브 앱, 자체 결제, 실시간 도면 생성, 다국어 — v3.0 OUT 유지
  • 전시 검색 기능
  • 기획사용 대시보드 (파일럿에선 수동 리포트로 대체)

04비즈니스 모델 (가볍게)

  • MVP: 파트너 기획사 1곳 무료 파일럿 (IP 제공 ↔ 데이터 리포트 무상 제공)
  • 검증 후: v3.0의 RS 모델 재검토 — 단, 수익원 우선순위가 바뀔 수 있음 (팝업 매출 중심 → 관람객 구독/굿즈 + 기획사 리포트 구독 가능성). 재접속률 결과에 따라 조정
  • 부스 굿즈는 현장 결제/외부 링크 (v3.0 OUT 기준 유지)

05치명적 리스크 (트랙 A — 이것만)

  1. IP 2차 활용 라이선스: 작품 이미지·세계관을 AI가 재가공·개인화하는 것에 대한 기획사/작가의 명시적 동의 필수. 파일럿 협약서에 명시
  2. 개인정보 (사진·감정 데이터): 온보딩 시 수집·활용 동의 플로우 필수. 관람객 사진에 타인이 찍히는 경우의 처리 기준 필요

06파일럿 타임라인 (7주, v3.0 골격 유지)

  • 1~2주: 파트너 1곳 확정, IP 자산 확보, 라이선스 협약
  • 3~4주: AI 큐레이터 + 아카이브 웹앱 구축, 미니 스핀오프 전시 기획 (내부 AI 어시스턴트 활용)
  • 5~6주: 현장 운영 (QR 배치, 부스 오픈), 데이터 수집
  • 7주: 지표 분석 → 성공 기준 대비 판정 → 다음 단계 결정

판정 후 분기

  • 재접속률 ≥ 30%: 본질 검증 성공 → 오프라인 스핀오프 확장(정식 팝업) + B2B 모델 구체화
  • 10~30%: 상세기획 내 조정 (아카이브 UX, 대화 품질, 리마인드 방식 개선 후 재시도)
  • < 10%: 상위기획으로 복귀 — "관람 후에도 머물고 싶다"는 전제 자체 재검토

핵심 설계 원칙: Phase 1 스핀오프는 본전시 데이터 없이 성립해야 한다

  • 1순위 시점에는 관람객의 본전시 데이터(사진·대화)가 존재하지 않음
  • 따라서 Phase 1 스핀오프 = 본전시를 안 본 사람도 즐길 수 있는 IP 세계관 경험 (진짜 스핀오프의 정의와 일치)
  • 개인화(관람 데이터 연동)는 Phase 3에서 본전시 모듈이 붙을 때 켜지는 프리미엄 기능
  • 지표 조정: Phase 1은 "관람 후 재접속률" 대신 스핀오프 자체의 체류·재방문·공유로 측정. "관람 후 48시간 재접속 30%"는 Phase 3(A+B 결합) 지표로 재배치
