## 1. 스택 · 구조

> ⚠️ 개발 착수 가이드의 `(guide)`/`(account)` 2-way 라우트 그룹은 `유저뷰-모듈분담-및-Git워크플로우.md`에 의해 **대체**되어 **3-way(main/spinoff/account)**로 재편되었습니다.

- **Next.js App Router + TypeScript + Tailwind CSS** (`.agents/rules/03_uiux.md`·`04_developer.md`·`05_qa.md` 기준)
- 모바일 웹앱 (v3.0 §9: 네이티브 앱 OUT). 기준 뷰포트 390px, 데스크톱은 센터 컬럼.
- 디자인 토큰(`.agents/rules/03_uiux.md`와 완전 일치, 변경 없음): 골드 `#C9A84C` / 다크 `#0D0D0F` / 배경 `#F7F5F0` / 서체 Playfair Display + DM Sans. 감정 axis 색상은 어드민 Monitor의 `EMOTION_AXES` 팔레트를 이식.
- 아이콘: lucide-react (어드민과 통일, Next.js에서도 그대로 사용 가능)


## 2. 구현 순서 (마일스톤)

MVP는 목업 데이터 기반 정적 프로토타입 → 이후 API 바인딩 (어드민과 동일 전략).

| M | 범위 | 완료 기준 (DoD) |
|---|---|---|
| **M0 공통 기반** | lib/ssot.ts + lib/track.ts + lib/session.ts + 공통 컴포넌트 골격 | 이벤트가 콘솔/로컬 큐에 SSOT 규격으로 쌓임 |
| **M1 척추 A** | S1 온보딩 → S2 정보카드 | `session.start` 발화, 감정 칩이 `{emotion,axis}`로 저장, 30초 룰 통과 |
| **M2 척추 B** | S3 채팅 (mock 응답) | `curator.chat_start`, emotion_chips·depth_score 축적, 감정 칩 탭 동작 |
| **M3 전환부** | S4 리포트 → S5 랜딩 | `invite.issued`→`landing_viewed`→`redeemed`→`purchase.link_clicked` 전 퍼널이 mock으로 완주 가능 |
| **M4 팝업** | S6 팝업 큐레이터 | 존 리스트+취향 뱃지, `spinoff.*` 이벤트, S3 채팅 컴포넌트 재사용 확인 |
| **M4.5 계정 모드** 🆕 | S0 로그인·가입 → S7~S10 (마이페이지/마이리포트/초대장/홈) | `account.signup`/`login`/`visitor_merged` 발화, 게스트→계정 병합 데이터 정합, 열람 모드(S4/S5 크롬 전환) 동작, 홈 §A 우선순위 엔진 1건 노출 확인 |
| **M5 연결** | 실제 AI 응답(Claude API) + 어드민 mock 데이터 정합 확인 | 초대장 concept_type ↔ 어드민 ConceptBot enum 일치 |

- 순서 근거: 데이터가 흐르는 척추(1→2→3→4→5) 순. S6은 S3 재사용이라 마지막.
- 각 화면의 세부 사양·엣지 케이스는 해당 상세기획 문서의 §진입·이탈 / §상태·엣지 / §이벤트 매핑을 그대로 따른다.

## 4. `lib/track.ts` — 이벤트 규격 (SSOT §3·§3-1)

```js
// 퍼널: session.start | curator.chat_start | invite.issued |
//       invite.landing_viewed | invite.redeemed | purchase.link_clicked
// 팝업: spinoff.session_start | spinoff.zone_viewed | spinoff.curator_chat | landing.organic_viewed
export function track(event, extra = {}) {
  const p = { event, exhibition_id, session_id, ts: Date.now(), ...extra };
  queue.push(p);            // MVP: 로컬 큐 + console — 오프라인 시 재전송(화면2 §4)
}
// 1회성 이벤트(chat_start, landing_viewed, redeemed)는 track 내부에서 중복 발화 가드
```

## 5. 공통 컴포넌트

## 6. Claude API (S3·S6 대화)

- 아티팩트/프로토타입 단계: `claude-sonnet-4-6`, mock KB를 system prompt로 주입.
- 응답 규격: 본문 + `extracted_emotions:[{emotion,axis}]` + `depth_score:1~5` JSON — 화면3 §4-3.
- RAG 범위 제한(전시 KB 밖 금지)·세계관 내 회피는 system prompt에 명시. S6은 여기에 `curator_role_prompt`(페르소나 사양 §2) 추가.
