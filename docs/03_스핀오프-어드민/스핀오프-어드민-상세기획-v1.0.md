## 1. 개요 및 기획 목적

### 1.1. 목적
본 시스템은 B2B 고객사(전시 기획사, 갤러리, 작가)가 자신들의 전시 IP 원천 자산을 등록하고, 이를 바탕으로 AI가 독립된 **'두 번째 작품(스핀오프 전시)'**의 스토리라인, AI 페르소나 대화, 추천 굿즈 및 연출 프리셋을 자동으로 기획·생성·운영할 수 있도록 지원하는 관리용 백오피스 플랫폼이다.

### 1.2. 핵심 가치
*   **B2B 셀링포인트**: 기획 대행사 없이 어드민 업로드만으로 고품질의 스핀오프 웹 전시와 미니 오프라인 부스용 콘텐츠를 1시간 내에 생성하여 B2B 확장 비용을 절감한다.
*   **데이터 선순환 구조**: 본전시와 스핀오프 간의 관람객 유입, 체류, 공유, 구매 데이터를 어드민에서 통합 관리하여 기획사에게 정량화된 가치(B2B 리포트)를 제공한다.

## 2. 어드민 기능 아키텍처 (기능 목록)

어드민 시스템은 크게 4가지의 논리적 모듈 구조로 구성된다.

### 2.1. IP 자산 및 세계관 관리 (Asset Management)
고객사가 AI 스핀오프 생성의 원료가 되는 본전시 데이터를 업로드하는 영역이다.
*   **전시 기본 정보 등록**: 전시 제목, 작가 프로필, 전시 카테고리, 메인 포스터 이미지 등.
*   **도슨트/오디오 스크립트 업로드**: 작품별 설명 텍스트, 오디오 도슨트 스크립트 파일(.txt, PDF 등).
*   **세계관 텍스트 DB 구축**: 작가 노트, 전시 기획서, 작가의 창작 의도 및 가상 시나리오 설명 등 AI가 학습해야 할 배경 자료 업로드.
*   **핵심 작품 정보 등록**: 작품명, 작품 고유 아이디, 이미지(비전 인식 매칭용), 작가 해설 핵심 키워드 설정.

### 2.2. AI 스핀오프 생성 엔진 제어 (AI Planning Assistant)
업로드된 IP 자산을 RAG(검색 증강 생성) 및 LLM 파이프라인을 거쳐 콘텐츠로 자동 변환하는 코어 컨트롤러다.
*   **AI 큐레이터/페르소나 튜닝**:
    *   대화 스타일(감성적, 논리적, 비평적 등) 템플릿 선택.
    *   스핀오프 전시에 등장할 AI 안내자(페르소나) 이름 및 프로필 설정.
*   **스핀오프 씬(Scene) 스크립트 빌더**:
    *   AI가 세계관 텍스트를 파싱하여 생성한 '외전 스토리(Scene 1~3)' 스크립트 확인 및 자동 생성.
    *   *수동 보정 기능*: AI가 기획한 씬 타이틀, 본문 해설, 연출 가이드를 담당자가 수동으로 직접 편집 및 수정 가능하게 함.
*   **공용 대화/비전 파라미터 매핑**:
    *   비전 인식 엔진에 전달될 '작품별 매칭 정확도 임계값(Threshold)' 설정.
    *   AI 대화 엔진에 주입할 프롬프트 프리셋(System Instruction) 세부 튜닝.

### 2.3. 외부 연계 및 커머스 브릿지 관리 (Referral & Commerce Bridge)
스핀오프 완료 시점의 전환 행동(Conversion) 경로를 설정하고 추적한다.
*   **본전시 예매처 연동**: 본전시 예매 링크(인터파크, 네이버 예약 등 외부 아웃링크) 등록 및 유입 트래킹용 UTM 파라미터 관리.
*   **굿즈숍 연동**: 외부 커스텀 굿즈 제작처(마플, 프린트디 등) 또는 기획사 자체 네이버 스마트스토어의 상품 링크 매칭.
*   **쿠폰/티켓 브릿지 설정**: 스핀오프 참여 관람객 대상 본전시 할인 코드 발급 설정 및 트래킹.

### 2.4. 비즈니스 리포트 및 통계 (B2B Dashboard)
파트너 기획사에게 스핀오프 전시의 성과 데이터를 가시화하여 제공한다.
*   **유입 및 활성 지표**:
    *   독립 유입군(미관람자) vs 본전시 연계 유입군의 일별 유입 추이.
    *   평균 체류 시간 및 AI 세계관 대화 지속도(턴 수별 잔존율).
*   **전환 지표**:
    *   스핀오프 씬 SNS 공유 건수 (공유용 카드 렌더러 전환 건수).
    *   커스텀 굿즈 클릭율 및 구매 전환 건수 (커머스 브릿지 전환).
    *   미관람자의 본전시 예매처 이동 및 실제 예매 전환 기여율.

## 3. 핵심 기획 정책 및 프로세스

### 3.1. AI 콘텐츠 승인 프로세스 (RAG 파이프라인)
고객사가 자산을 등록한 뒤 스핀오프 콘텐츠가 최종 사용자에게 퍼블리싱되기까지의 프로세스 가이드라인이다.
1.  **Draft 상태 (자산 등록)**: 기획사가 텍스트/이미지 자산 업로드 완료.
2.  **AI Generating 상태**: AI 분석 및 5개 공통 모듈용 데이터 셋(대화 컨셉, 씬 스토리라인, 카드 템플릿) 자동 생성.
3.  **Review 상태**: 어드민에서 생성된 스핀오프 씬 스크립트를 시뮬레이터(미리보기)를 통해 검수하고 수동으로 일부 내용 교정.
4.  **Published 상태**: 독립 스핀오프 전용 URL 및 QR 활성화. 유저뷰 오픈.

## 4. 코어 데이터 모델 & 스키마 (초안)

사용자와 어드민이 공통 기획/개발을 진행할 데이터 규격의 뼈대 모델이다.

### 4.1. 전시 IP 스키마 (Exhibition IP Schema)
```json
{
  "exhibition_id": "string (UUID)",
  "title": "string",
  "artist": {
    "name": "string",
    "profile_desc": "string"
  },
  "worldview_context": "string (RAG 학습용 장문 텍스트)",
  "artworks": [
    {
      "artwork_id": "string",
      "title": "string",
      "vision_reference_image_url": "string (URL)",
      "description": "string",
      "audio_script": "string"
    }
  ]
}
```

### 4.2. 스핀오프 씬 스크립트 스키마 (Spinoff Scene Schema)
```json
{
  "spinoff_id": "string (UUID)",
  "exhibition_id": "string (Exhibition UUID)",
  "persona": {
    "name": "string",
    "tone_type": "string (friendly | academic | emotional)",
    "system_instruction": "string (AI 프롬프트)"
  },
  "scenes": [
    {
      "scene_number": "integer",
      "title": "string",
      "story_content": "string (외전 텍스트)",
      "bg_ambient_music_url": "string",
      "visual_preset_color": "string (Hex code)"
    }
  ]
}
```

### 4.3. 유저 체험 세션 스키마 (User Experience Session Schema)
```json
{
  "session_id": "string (UUID)",
  "is_visitor_of_main_exhibition": "boolean",
  "linked_main_session_id": "string (Optional, 본전시 연동 시)",
  "user_sentiment_codes": ["string (감정 코드 리스트)"],
  "spinoff_status": {
    "current_scene": "integer",
    "chat_turn_count": "integer",
    "is_completed": "boolean"
  },
  "conversions": {
    "card_shared": "boolean",
    "goods_clicked": "boolean",
    "main_ticket_clicked": "boolean"
  }
}
```
