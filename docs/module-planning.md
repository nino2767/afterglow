> 버전: v1.1 | 작성일: 2026.06.09 | 업데이트: 공동 작업자 안(경험 모듈) + 기존 안(공간 테마 모드) 통합  
> 연관 문서: [방향성문서 v3.0](https://www.notion.so/AFTERGLOW-34ab01d73440806083a7ef421016c65e)

## 두 안의 관점과 레이어

두 안은 충돌하지 않습니다. **레이어가 다릅니다.**

| | 공간 테마 모드 (기존 안) | 관람객 경험 모듈 (공동 작업자 안) |
|---|---|---|
| **설계 관점** | 팝업 공간 컨셉·테마 중심 | 관람객 경험 행동 중심 |
| **분류 기준** | 어떤 세계를 만드는가 | 무엇을 하게 하는가 |
| **주 활용처** | B2B 기획사 Concept Bot 자동화 | B2C 관람객 UX 설계 |
| **강점** | AI 자동화·팝업 차별화와 직결 | 전시 유형별 범용 적용, 사용자 행동 설계 명확 |

```
[Layer 1]  공간 테마 모드 (Concept Bot)   ← "어떤 성격의 팝업을 만들 것인가"
                      ↓  전시 유형 × 테마 조합으로 자동 추천
[Layer 2]  관람객 경험 모듈               ← "팝업 안에서 관람객이 무엇을 하는가"
                      ↓
[Layer 3]  데이터 파이프라인              ← AI 큐레이터 수집 → 팝업 개인화 → 기획사 인사이트
```

## Layer 1 — 공간 테마 모드 (Concept Bot)

기획사가 Concept Bot 실행 시 선택하는 **팝업 공간의 성격**을 정의합니다.

```
Concept Bot
├── Story Mode     — 스토리형     (서사 축)
├── Sensory Mode   — 감각형       (공감각 축)
├── Archive Mode   — 데이터 초상형 (개인화 축)
├── Community Mode — 커뮤니티형   (관계 축)
├── Role Mode      — 역할극형     (참여 축)
└── Time Mode      — 시간여행형   (맥락 축)
```

| 모드명 | 핵심 축 | 팝업 공간 성격 | MVP 적합성 |
|---|---|---|---|
| Story Mode | 서사·내러티브 | 작품 세계관 또는 작가 삶을 서사로 재현 | ✅ 1순위 |
| Sensory Mode | 공감각·신체 | 시각 외 감각(향·소리·촉감)으로 재해석 | ✅ 1순위 |
| Archive Mode | 개인화·자기표현 | 나의 관람 데이터가 유일한 굿즈가 되는 공간 | ✅ 1순위 |
| Community Mode | 관계·연결 | 같은 취향의 낯선 사람을 연결하는 클럽 | 🔵 v1.0 |
| Role Mode | 참여·몰입 | 관람객이 전시 속 역할을 수행하는 이스케이프룸형 | 🟡 v2.0 |
| Time Mode | 맥락·역사 | 작품의 시대적 배경 속으로 이동하는 공간 | 🟡 v2.0 |

### 테마 모드 상세

#### Story Mode

작품 세계관 확장형(작품 속 가상 세계를 현실 공간으로) 또는 작가 연대기형(작가의 삶을 동선으로 재구성)으로 운영합니다.

```
[입장] 프롤로그 존 → [중간] 절정 존 → [퇴장] 에필로그 존 + 굿즈
```

- **AI 자동화**: 작품 색감·형태 → 공간 시간대·날씨·인물 설정 → 3막 동선 자동 생성
- **굿즈 방향**: "이 세계에서 가져온 물건"이라는 내러티브 부여
- **추가 아이디어**: 엔딩 분기형(AI 대화에 따라 마지막 존 연출 변경), 시즌제 운영

#### Sensory Mode

미디어아트의 시각 경험을 역방향으로 접근합니다.

```
[작품 색감] → AI 번역 → [향기 팔레트]
[작품 리듬] → AI 번역 → [사운드스케이프]
[작품 질감] → AI 번역 → [촉각 오브제]
```

- **AI 자동화**: 작품 이미지 → 색온도·채도를 향기 계열로 매핑 (예: 차가운 딥블루 → 시더우드+민트)
- **결과물**: 팝업용 "감각 레시피 북" 자동 생성 (조향사·사운드 아티스트 브리핑)
- **추가 아이디어**: "블라인드 관람" 체험존 → SNS 챌린지 유도, 감각 구독 박스

#### Archive Mode

AI 큐레이터가 수집한 관람 데이터를 오직 나만의 시각적 아카이브로 변환합니다.

| 데이터 입력 | AI 산출물 | 굿즈/체험 |
|---|---|---|
| 체류 시간 TOP 3 작품 | 나만의 "베스트 컬렉션" 포스터 | A3 리소그래프 인쇄 |
| 반복된 감정 키워드 | 키워드로 생성한 추상 비주얼 | 폰케이스·에코백 |
| 대화 히스토리 전체 | 나의 관람을 요약한 "한 줄 시" | 레터프레스 카드 |
| 동선 데이터 | 내가 만든 전시 지도 | 리소 지도 포스터 |

#### Community Mode

취향 클러스터링으로 관람객을 연결합니다.

- **AI 자동화**: 큐레이터 취향 데이터 → 클러스터링 → 클러스터별 이벤트 초대장 자동 발송
- **공간 장치**: 취향 키워드 뱃지 수령 → 같은 색상 사람과 대화 유도 → 두 사람 데이터 합친 콜라보 포스터 제작
- **추가 아이디어**: 오프라인 → 온라인 커뮤니티 연결, 6개월 후 리유니언 이벤트

#### Role Mode

관람객에게 전시 세계관 안의 직업·역할을 부여합니다.

```
[입장] 역할 배정 → [탐험] 역할별 미션 수행 → [완료] 역할 완수 보상 굿즈
```

- **AI 자동화**: IP 분석 → 직업·역할 목록 자동 생성 → 역할별 미션 시나리오 자동 작성
- **재방문 구조**: 각 역할마다 다른 동선·MD → 한 번 방문으로 모든 역할 체험 불가
- **추가 아이디어**: 그룹 역할극(일행이 협력 미션), 역할 누적 레벨업 시스템

#### Time Mode

작품이 만들어진 시대적·문화적 맥락을 재현합니다.

- **AI 자동화**: 작품 연도·배경 입력 → 해당 시대 문화·유행·사건 조사 → 공간 연출 브리핑 생성
- **공간 설계**: 입장 전 "시대 설정" 존 → 관람객이 어느 시대로 갈지 선택하거나 AI가 자동 배정
- **추가 아이디어**: "미래 시간여행" (이 작품이 100년 후 발견된다면), 시대별 한정 MD

## Layer 2 — 관람객 경험 모듈

팝업 공간 안에서 관람객이 **실제로 하는 행동**을 설계합니다.

### 모듈 10종 분류

| 구분 | 모듈명 | 핵심 목적 | 활용 데이터 | 사용자 경험 | 적용 난이도 |
|---|---|---|---|---|---|
| **회고형** | 감상 회고 모듈 | 나의 관람 경험을 다시 돌아보게 함 | 감정 태그, 저장 작품, 감상 메모, 관람 순서 | "나는 이 전시를 이렇게 느꼈구나"를 리포트로 확인 | 낮음~중간 |
| **표현형** | 감정 담벼락 모듈 | 감정과 생각을 짧게 남기게 함 | 한줄 감상, 감정 태그, 작품 선택 | 한 줄 감상 작성, 다른 관람객의 문장 확인 | 낮음 |
| **개인화형** | 나만의 큐레이션 모듈 | 관람객의 취향으로 전시를 재구성 | 저장 작품, 관심 키워드, 감정 태그 | 내가 고른 작품으로 나만의 작은 전시 생성 | 중간 |
| **공유형** | 전시 포스터/카드 모듈 | 관람 경험을 이미지 콘텐츠로 공유 | 감상 키워드, 저장 작품, 대표 감정 | 나만의 전시 포스터·감상 카드 생성 | 낮음~중간 |
| **시각화형** | 관람 하이라이트 모듈 | 동선과 체류 경험을 콘텐츠화 | 관람 동선, 체류 시간, 공간별 반응 | 내가 오래 머문 구간을 15초 영상/루트로 확인 | 높음 |
| **집단 반응형** | 집단 반응 지도 모듈 | 다른 관람객의 반응과 내 감상을 비교 | 작품별 감정 태그, 저장 수, 체류 시간 | 사람들이 가장 많이 반응한 작품/구간 확인 | 중간 |
| **참여형** | 투표/질문 모듈 | 전시 메시지에 대한 의견을 수집 | 선택 응답, 감상 키워드, 공감/비공감 | 질문에 답하고 다른 사람들의 선택 확인 | 낮음 |
| **수집형** | 디지털 굿즈 모듈 | 관람의 완성감과 소장 욕구를 제공 | 방문 기록, 미션 완료, 작품 저장, 참여 이력 | 배지, 스티커, 디지털 티켓, 기념 카드 획득 | 중간 |
| **학습형** | 관람 노트/퀴즈 모듈 | 전시 내용을 이해하고 기억하게 함 | 작품 정보, 섹션 정보, 퀴즈 응답 | 관람 내용을 정리하거나 퀴즈로 복습 | 중간 |
| **추천형** | 다음 경험 추천 모듈 | 관람 이후 다음 행동으로 연결 | 감정 태그, 저장 작품, 관심 키워드, 클릭 이력 | 다음 전시, 굿즈, 프로그램, 콘텐츠 추천 | 중간 |

### 상위 목적별 묶음

| 상위 목적 | 해당 모듈 |
|---|---|
| **감상을 남기게 하는 모듈** | 감정 담벼락, 감상 회고, 관람 노트 |
| **감상을 콘텐츠로 바꾸는 모듈** | 전시 포스터/카드, 관람 하이라이트, 나만의 큐레이션 |
| **다른 사람과 연결하는 모듈** | 집단 반응 지도, 투표/질문, 감정 담벼락 |
| **다음 행동으로 연결하는 모듈** | 다음 경험 추천, 디지털 굿즈, 프로그램/굿즈 연계 |

## Layer 1 × Layer 2 — 전시 유형별 패키지 추천

기획사가 전시 유형을 선택하면 **테마 모드 + 경험 모듈 패키지**가 자동 추천됩니다.

| 전시 유형 | 추천 테마 모드 | 패키지명 | 기본 경험 모듈 | 확장 모듈 |
|---|---|---|---|---|
| **미디어아트·몰입형** | Sensory + Archive | Immersive Highlight Package | 관람 하이라이트, 감정 타임라인, 집단 반응 지도 | 전시 포스터/카드, 다음 경험 추천 |
| **미술전** | Story (작가 연대기) | Personal Reflection Package | 감상 회고, 나만의 큐레이션, 감정 담벼락 | 다음 경험 추천, 전시 포스터/카드 |
| **팬덤·IP 전시** | Story (세계관 확장) + Community | Fan Memory Package | 팬 메시지 월, 디지털 굿즈, 최애 루트 | 랭킹/참여형, 공유 카드 |
| **역사·아카이브전** | Time | Memory Archive Package | 감상 회고, 관람 노트, 관심 주제 추천 | 퀴즈형, 나만의 큐레이션 |
| **어린이·가족** | Role | Family Play Package | 스탬프북, 미션 완료 카드, 가족 감상 기록 | 퀴즈형, 디지털 굿즈 |
| **브랜드 팝업** | Sensory + 참여형 | Brand Engagement Package | 전시 포스터/카드, 투표/질문, 디지털 굿즈 | 굿즈 추천, 랭킹/이벤트 |
| **문학·텍스트 전시** | Story (세계관 확장) | Words Afterglow Package | 감정 담벼락, 문장 수집함, 나의 문장 카드 | 감상 회고, 집단 반응 지도 |
| **공공·사회 메시지** | Community | Public Voice Package | 생각 담벼락, 투표/질문, 집단 반응 지도 | 캠페인 연결, 공유 카드 |
| **상설·복합문화공간** | Archive + Community | Return Visit Package | 감상 회고, 관람 히스토리, 다음 경험 추천 | 관심 주제 추천, 디지털 굿즈 |
| **사진전** | Sensory | Emotion Sharing Package | 감정 담벼락, 전시 포스터/카드, 감상 회고 | 문장 수집함, 집단 반응 지도 |

## 모드 간 조합 가능성

| 조합 | 시너지 |
|---|---|
| Story + Sensory | 세계관을 시각+후각+청각으로 완전 몰입 |
| Archive + Community | 나의 아카이브로 같은 취향의 사람을 찾는 구조 |
| Role + Time | "그 시대의 탐정"처럼 역할과 시대가 결합 |
| Story + Role | 서사 속 역할을 직접 수행 → 몰입도 최고 |

## 개발 우선순위

| 단계 | Layer 1 테마 모드 | Layer 2 경험 모듈 | 이유 |
|---|---|---|---|
| **MVP** | Story / Sensory / Archive | 감상 회고, 감정 담벼락, 전시 포스터/카드, 다음 경험 추천 | AI 큐레이터 데이터 수집 구조와 바로 연결, 구현 난이도 낮음 |
| **v1.0** | Community | 나만의 큐레이션, 집단 반응 지도, 디지털 굿즈 | 클러스터링 로직 필요, MVP 데이터 축적 후 적용 |
| **v2.0** | Role / Time | 관람 하이라이트, 관람 노트/퀴즈, 투표/질문 | 공간 설계·기술 복잡도 높음, 레퍼런스 확보 후 진행 |
