# AFTERGLOW — 스핀오프 기획 모듈 기획서

> 버전: v1.0 | 작성일: 2026.06.09 | 연관 문서: [방향성문서 v3.0](https://www.notion.so/AFTERGLOW-34ab01d73440806083a7ef421016c65e)

---

## 개요

AFTERGLOW의 스핀오프 팝업 기획을 자동화하는 **Concept Bot의 모드 확장 구조**입니다.  
각 모듈은 관람객 경험의 서로 다른 축을 중심으로 설계되며, 단독 또는 조합으로 운영 가능합니다.

> 기획사(운영자)가 Concept Bot 실행 시 모드를 선택하면,  
> 해당 모드에 맞는 **공간 설계 브리핑 · MD 리스트 · 운영 시나리오**가 자동 생성됩니다.

---

## Concept Bot 전체 구조

```
Concept Bot
├── Story Mode     — 스토리형     (서사 축)
├── Sensory Mode   — 감각형       (공감각 축)
├── Archive Mode   — 데이터 초상형 (개인화 축)
├── Community Mode — 커뮤니티형   (관계 축)
├── Role Mode      — 역할극형     (참여 축)
└── Time Mode      — 시간여행형   (맥락 축)
```

| 모듈명 | 핵심 축 | 관람객 경험 방식 | B2B 자동화 포인트 | MVP 적합성 |
|---|---|---|---|---|
| Story Mode | 서사·내러티브 | 세계관 속 캐릭터, 작가의 삶 동행 | IP 분석 → 3막 구조 자동 생성 | ✅ 1순위 |
| Sensory Mode | 공감각·신체 경험 | 시각 외 감각으로 전시를 재해석 | 작품 → 향·소리·촉감 언어 자동 변환 | ✅ 1순위 |
| Archive Mode | 개인화·자기표현 | 나의 관람 데이터가 유일한 굿즈가 됨 | 관람 데이터 → 개인 아카이브 자동 생성 | ✅ 1순위 |
| Community Mode | 관계·연결 | 같은 취향의 낯선 사람과 연결 | 취향 클러스터링 → 커뮤니티 이벤트 자동 기획 | 🔵 2순위 |
| Role Mode | 참여·몰입 | 관람객이 전시 속 직업·역할을 수행 | IP → 역할 시나리오 자동 설계 | 🟡 v2.0 |
| Time Mode | 맥락·역사 | 작품의 시대적 배경 속으로 이동 | 작품 연도·맥락 → 시대 재현 공간 자동 제안 | 🟡 v2.0 |

## 모듈 상세

### 1. Story Mode — 스토리형

> **핵심 축:** 서사·내러티브

#### 1-A. 작품 세계관 확장형 — "작품 속으로"

특정 작품 하나를 골라 그 안에 존재하는 가상의 세계를 현실 공간으로 끌어냅니다.  
관람객은 "이 작품을 본 사람"이 아니라 "이 세계를 살아가는 존재"로 팝업에 입장합니다.

**공간 구성**
```
[입장] 프롤로그 존 → [중간] 절정 존 → [퇴장] 에필로그 존 + 굿즈
 "이 세계에 도착"     "사건의 중심"      "이 세계를 떠나며 가져가는 것"
```

**AI 자동화**
- 작품 색감·형태 → 공간의 시간대·날씨·인물 설정 자동 추출
- 발단-전개-절정 3막 구조로 팝업 동선 자동 생성

**커스텀 MD 방향**
| 굿즈 유형 | 스토리형 접근 |
|---|---|
| 포스터 | "챕터 2에서 발견된 편지" 형태의 레터 프레스 |
| 향수/인센스 | "이 세계의 오전 6시 공기"라는 이름의 향 |
| 영상 굿즈 | 관람객 본인이 "이 세계의 주인공"으로 등장하는 AI 편집 영상 |

**추가 아이디어**
- 엔딩 분기형: AI 대화 내용에 따라 마지막 존 연출이 달라짐 → SNS 바이럴
- 시즌제 운영: 동일 IP로 봄·여름 챕터 → 재방문 구조
- 미완성 요소 활용: 작가 주석의 단편적 언급 → 팝업 미스터리로 활용

#### 1-B. 작가 연대기형 — "작가의 삶을 걷다"

작품이 아닌 **작가라는 인간**에 집중합니다. 관람객은 "작품의 관객"에서 "작가의 친구"로 전환됩니다.

**공간 구성**
```
[1존] 기원(Origin) → [2존] 전환점(Turning Point) → [3존] 지금 이 작품(The Work)
 작가가 자란 공간·      결정적 만남·사건·               작품이 탄생한
 최초의 감각적 기억      작업 방향을 바꾼 순간             내면의 풍경
```

**추가 데이터 입력 유형**
| 데이터 유형 | 수집 방법 | AI 활용 |
|---|---|---|
| 작가 인터뷰·에세이 | 텍스트 업로드 | 성격 키워드·반복 언어 패턴 추출 |
| 작가의 영향을 받은 레퍼런스 | 링크·이미지 | 미적 계보 분석 |
| 작업실·성장 환경 사진 | 이미지 업로드 | 공간 무드 언어화 |
| 주변 인물 (지인·스승·파트너) | 텍스트 정리 | 관계망 시각화 및 캐릭터화 |

**추가 아이디어**
- 작가의 하루 시간대별 팝업 운영 → 재방문 구조 자동 생성
- 미완성 작가 노트 참여형 굿즈 (관람객이 여백을 채움)
- 작가 연대기 아카이브 구독 모델 → B2C 리텐션

### 2. Sensory Mode — 감각형

> **핵심 축:** 공감각·신체 경험

미디어아트의 압도적 시각 경험을 **역방향**으로 접근합니다.  
전시에서 받은 시각적 인상을 향기·소리·촉감·미각으로 번역해 팝업에서 재경험하게 합니다.

**AI 변환 구조**
```
[작품 색감] → AI 번역 → [향기 팔레트]
[작품 리듬] → AI 번역 → [사운드스케이프]
[작품 질감] → AI 번역 → [촉각 오브제]
```

**AI 자동화 포인트**
- 작품 이미지 분석 → 색온도·채도를 향기 계열로 매핑
  - 예: 차가운 딥블루 → 시더우드 + 민트
  - 예: 따뜻한 앰버 → 샌달우드 + 바닐라
- 작품의 움직임 속도·리듬 → BPM·악기 구성으로 변환
- 결과물: 팝업용 **감각 레시피 북** 자동 생성 (조향사·사운드 아티스트에게 전달할 브리핑)

**개인화 연계**  
AI 큐레이터 감정 키워드 기반으로 나만의 감각 조합 추천:
- "압도적·웅장" → 무거운 우드 베이스 향 + 저음 사운드트랙
- "몽환·고요" → 플로럴+시트러스 향 + 앰비언트 무드

**추가 아이디어**
- "블라인드 관람" 체험존: 시각 차단 후 향+소리만으로 작품 맞추기 → SNS 챌린지
- 감각 구독 박스: "이 전시의 계절별 감각"을 월별 박스로 배송 → 장기 리텐션

### 3. Archive Mode — 데이터 초상형

> **핵심 축:** 개인화·자기표현

AI 큐레이터가 수집한 관람 데이터를 **오직 나만의 시각적 아카이브**로 변환합니다.  
v3.0의 "관심도 데이터·감정 키워드·인터랙션 히스토리"를 수익화 재료로 직접 활용합니다.

**데이터 → 굿즈 변환 구조**

| 데이터 입력 | AI 산출물 | 굿즈/체험 |
|---|---|---|
| 체류 시간 TOP 3 작품 | 나만의 "베스트 컬렉션" 포스터 | A3 리소그래프 인쇄 |
| 반복된 감정 키워드 | 키워드로 생성한 추상 비주얼 | 폰케이스·에코백 |
| 대화 히스토리 전체 | 나의 관람을 요약한 "한 줄 시" | 레터프레스 카드 |
| 동선 데이터 | 내가 만든 전시 지도 | 리소 지도 포스터 |

**공간 설계 원칙**  
팝업 공간이 곧 **"나의 관람 아카이브 인쇄소"** — 방문하는 순간 내 데이터가 실물로 출력됩니다.

**추가 아이디어**
- 관람 유형 분류: "당신은 철학자형 관람객" → 공유 유도
- 기획사 대시보드 연계: 개인 아카이브 데이터 → "어떤 섹션이 가장 깊은 대화를 만들었는가" 집단 인사이트 자동 집계

---

### 4. Community Mode — 커뮤니티형

> **핵심 축:** 관계·연결

같은 전시를 봤지만 서로 모르는 관람객들을 **취향으로 연결**합니다.  
팝업은 단순한 구매 공간이 아니라 "이 전시를 사랑하는 사람들의 클럽"이 됩니다.

**AI 클러스터링 구조**
```
"고요·명상·여백"을 공통으로 가진 관람객 그룹 → 클러스터 A → 드로잉 워크숍 초대
"압도·강렬·서사"를 공통으로 가진 관람객 그룹 → 클러스터 B → 아티스트 토크 초대
```

**공간 내 만남의 장치**
- 입장 시 나의 취향 키워드가 인쇄된 뱃지 수령
- 같은 뱃지 색상의 낯선 사람과 자연스럽게 대화 유도
- 두 사람의 관람 데이터를 합쳐 만드는 콜라보 포스터 제작

**추가 아이디어**
- 오프라인 → 온라인 커뮤니티 연결 → 차기 전시 홍보 채널로 자연 활용
- "다시 만나는 팝업": 6개월 후 같은 클러스터 리유니언 이벤트 → 장기 IP 수익 구조

---

### 5. Role Mode — 역할극형

> **핵심 축:** 참여·몰입

관람객에게 전시 세계관 안의 **특정 직업이나 역할**을 부여합니다.  
이스케이프룸과 전시의 중간 어딘가.

**동선 구조**
```
[입장] 역할 배정 → [탐험] 역할별 미션 수행 → [완료] 역할 완수 보상 굿즈
```

**AI 자동화 포인트**
- IP 분석 → 세계관 내 존재 가능한 직업·역할 목록 자동 생성
- 각 역할별 미션 시나리오 자동 작성 (Ops Automator 확장)
- AI 큐레이터 대화 패턴 → 역할 자동 배정
  - 예: 철학적 질문이 많았던 관람객 → 사서 역할

**재방문 구조**  
각 역할마다 다른 동선·MD → 한 번 방문으로 모든 역할 체험 불가 → 자동 재방문 유도

**추가 아이디어**
- 그룹 역할극: 일행이 각기 다른 역할로 협력 미션 → 단체 방문 유도
- 역할 누적 레벨업 시스템 → 장기 리텐션

---

### 6. Time Mode — 시간여행형

> **핵심 축:** 맥락·역사

작품이 만들어진 시대적·문화적 맥락을 팝업 공간에서 재현합니다.  
"그 시대에 이 작품을 처음 본 사람의 감각"을 체험하게 합니다.

**AI 자동화 포인트**
- 작품의 제작 연도·배경 입력 → 해당 시대의 문화·유행·사건 자동 조사 → 공간 연출 브리핑 생성
- "이 작품이 처음 발표됐을 때 사람들의 반응" 아카이브 텍스트 → 전시 해설 자동 생성
- 시대별 디자인 레퍼런스 자동 추출 → MD 디자인 방향 제안

**공간 설계 원칙**  
공간 진입 전 "시대 설정" 존 — 관람객이 어느 시대로 갈지 선택하거나 AI가 관람 데이터 기반 자동 배정

**추가 아이디어**
- "미래 시간여행": "이 작품이 100년 후에 발견된다면" AI 시나리오 생성
- 시대별 한정 MD → 희소성 기반 재방문 유도

## 모듈 간 조합 가능성

| 조합 | 시너지 |
|---|---|
| Story + Sensory | 세계관을 시각+후각+청각으로 완전 몰입 |
| Archive + Community | 나의 아카이브로 같은 취향의 사람을 찾는 구조 |
| Role + Time | "그 시대의 탐정"처럼 역할과 시대가 결합 |
| Story + Role | 서사 속 역할을 직접 수행 → 몰입도 최고 |

## 개발 우선순위

| 단계 | 모듈 | 이유 |
|---|---|---|
| MVP | Story / Sensory / Archive | AI 큐레이터 데이터 수집 구조와 바로 연결, 구현 난이도 낮음 |
| v1.0 | Community | 클러스터링 로직 필요, MVP 데이터 축적 후 적용 |
