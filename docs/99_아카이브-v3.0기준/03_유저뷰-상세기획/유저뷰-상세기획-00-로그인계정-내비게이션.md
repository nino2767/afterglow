# 유저뷰 상세기획 — 화면 0: 로그인·회원가입 / 화면 7: 마이페이지 / 내비게이션 셸

> 버전: v1.0 | 작성일: 2026.07.03 | 상태: 📋 초안
> 근거: SSOT v1.2 §7(계정 모델·내비게이션 셸)
> 성격: **기존 화면1~6 세트에 대한 증보.** 게스트 우선 원칙은 유지하되, 계정·내비게이션 결핍을 메운다.
> ⚠️ 이 문서는 화면1 §5(MVP OUT: 소셜 로그인·계정 연동)와 화면4 §7을 **오버라이드**한다 — 계정 기능은 MVP IN으로 승격.

---

## 0. 왜 이제 계정이 필요한가

기존 설계는 "QR 스캔 → 순서대로 진행 → 종료"의 1회성 방문을 가정했다. 그런데 실제로는:
- 리포트를 다른 기기(예: PC)에서 다시 보고 싶고
- 전시가 끝난 뒤에도 팝업·차기전시 알림을 받고 싶고
- 기획사는 재방문 관람객을 식별하고 싶다

→ VISITOR(기기 기반, 휘발성)만으로는 이 세 요구를 못 채운다. **ACCOUNT(계정, 영속적)**를 얹되, 온보딩 진입 장벽은 그대로 지킨다(게스트 우선).

## 1. 화면 0: 로그인·회원가입

### 1-1. 진입 경로 (강제 없음 — 전부 선택적 트리거)

| 트리거 | 위치 |
|---|---|
| 상단 계정 아이콘 탭 | 화면1~6 어디서든 (내비게이션 셸 §4) |
| "계정 만들고 저장하기" | 화면4 리포트 완성 직후 (주 전환 지점) |
| "다음에도 알림 받기" | 화면5 랜딩, 팝업 미체크인/오픈 전 상태 |
| "이미 계정이 있으신가요?" 링크 | 화면1 온보딩 하단 (소극적 노출) |

### 1-2. 화면 구성

```
┌──────────────────────────────┐
│  ✕ (닫기 → 이전 화면 복귀)      │
│                                │
│  당신의 여운을 저장해두세요      │  ← 트리거별 문구 다름 (아래 §1-3)
│                                │
│  [ 🟡 카카오로 시작하기 ]       │  ← 원탭, 기본 권장
│                                │
│  ── 또는 ──                    │
│  [ 이메일 ]  [ 비밀번호 ]       │
│  [ 이메일로 계속하기 ]          │
│                                │
│  이미 계정이 있으신가요? 로그인  │  ← 가입/로그인 모드 전환
└──────────────────────────────┘
```

- **닫기(✕) 시 게스트로 계속 진행** — 로그인은 끝까지 선택 사항. 어떤 트리거로 들어와도 닫으면 원래 화면으로 복귀, 데이터 손실 없음.
- 카카오 선택 시 OAuth 완료 후 자동으로 §2 병합 로직 실행.
- 이메일 방식은 가입 시 비밀번호 1회 + 마케팅 동의 체크(선택) 포함.

### 1-3. 트리거별 문구 (전환율 최적화)

| 트리거 | 헤드카피 |
|---|---|
| 화면4 리포트 직후 | "이 여운, 잃어버리지 않게 저장해두세요" |
| 화면5 미체크인/오픈전 | "팝업 소식, 잊지 않고 알려드릴게요" |
| 화면1 소극적 링크 | (문구 없음 — 단순 텍스트 링크) |
| 계정 아이콘 직접 진입 | "다시 만나서 반가워요" (재방문 뉘앙스) |

### 1-4. 게스트→계정 병합 (SSOT §7-2)

- 가입/로그인 성공 즉시: 현재 `visitor_id`의 SESSION·TASTE_PROFILE·SPINOFF_INVITE·PURCHASE를 `account_id`에 연결 (`account.visitor_merged` 발화).
- 병합 후 화면4/5로 복귀 시 "저장됐어요" 토스트만 표시, 화면 자체는 변화 없음 (매끄러운 전환).
- 기존 계정으로 로그인(신규 아님) 시: 병합이 아니라 **해당 계정의 기존 데이터를 이 기기에도 동기화**하여 마이페이지에서 통합 조회 가능하게 함.

### 1-5. 이벤트

`account.signup` / `account.login` / `account.visitor_merged` — SSOT §7-3 그대로.

## 2. 화면 7: 마이페이지

### 2-1. 역할

로그인 사용자의 "계정 홈". 전시 밖에서 진입하는 유일한 화면이자, 하단 탭바(계정 모드)의 종착점 중 하나.

### 2-2. 화면 구성

```
┌──────────────────────────────┐
│  {nickname}님                  │  프로필
│  카카오 계정 · 설정 →           │
├──────────────────────────────┤
│  📖 나의 여운 리포트             │  관람 히스토리 (크로스디바이스 핵심)
│  ┌────────┐┌────────┐        │
│  │빛의 심연 ││(다른 전시)│      │  ← 화면4 카드 축소판, 탭 시 재열람
│  │06.16    ││          │      │
│  └────────┘└────────┘        │
├──────────────────────────────┤
│  💌 나의 초대장                 │  발급된 SPINOFF_INVITE 전체
│  · 어비스 티 라운지 · 유효 D-12  │  (유효기간=운영정책 §3)
├──────────────────────────────┤
│  🔔 알림 설정                   │  차기 전시·팝업 오픈 opt-in
├──────────────────────────────┤
│  로그아웃                       │
└──────────────────────────────┘
├───[홈]─[마이리포트]─[초대장]─[마이페이지●]───┤  하단 탭바(계정 모드)
```

### 2-3. 구성 요소

| 요소 | 동작 | 데이터 |
|---|---|---|
| 여운 리포트 목록 | account_id에 연결된 전체 TASTE_PROFILE(전시별 1개) 카드 그리드. 탭 → 화면4 재열람(read-only) | 읽기: TASTE_PROFILE (account 경유 전체 VISITOR) |
| 초대장 목록 | account_id 하위 전체 SPINOFF_INVITE. 유효기간 임박 순 정렬 | 읽기: SPINOFF_INVITE |
| 알림 설정 | 전시별 opt-in 토글 (화면4/5의 opt-in과 동일 데이터) | ACCOUNT.marketing_consent + 전시별 세부 |

### 2-4. 하단 탭바 4종 (계정 모드)

| 탭 | 내용 |
|---|---|
| 홈 | 참여 가능한 전시 찾기 (v3.0 §8 파트너 확장 시 의미 커짐 — MVP는 안내 위주) |
| 마이리포트 | 여운 리포트 목록 (마이페이지 §2-2와 동일 뷰 단독 진입) |
| 초대장 | 초대장 목록 단독 진입 |
| 마이페이지 | 프로필·설정 |

## 3. 내비게이션 셸 — 화면1~6 공통 적용 (SSOT §7-4)

```
┌──────────────────────────────┐
│ ←   {전시명 또는 화면 컨텍스트}   👤 │  ← 상단 바, 전 화면 공통
├──────────────────────────────┤
│                                │
│        (화면별 본문)            │
│                                │
└──────────────────────────────┘
```

- **뒤로가기(←)**: 각 화면 상세기획의 §진입·이탈 흐름을 따름 (임의 이동 아님 — 정해진 이전 단계로).
- **계정 아이콘(👤)**: 비로그인 = 로그인 유도 아이콘(탭 → 화면0 오버레이). 로그인 = 아바타(탭 → 화면7 오버레이 진입, 가이드 모드 컨텍스트는 유지되어 복귀 가능).
- **하단 탭바 없음** — 화면1~6은 계속 "가이드 모드". 유일한 하단 요소는 각 화면 자체 정의(예: 화면2·3의 "관람 완료" 바)이며 탭바가 아니다.

## 5. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| 카카오 소셜 로그인 | 애플/구글 로그인 |
| 이메일+비밀번호 가입/로그인 | 비밀번호 재설정 플로우 (MVP는 고객센터 안내로 대체) |
| 게스트→계정 병합(단방향, 최초 1회) | 계정 간 데이터 병합(2계정 통합) |
| 마이페이지: 리포트·초대장·알림설정 | 프로필 이미지, 소셜 공유 히스토리 |
| 계정 모드 하단 탭바 4종 | 홈 탭의 전시 탐색·검색 고도화 |

## 6. 남은 결정 포인트

1. **비로그인 상태로 화면4/5 재접속 시(같은 기기, 세션 만료 후) 동작** — visitor_id가 남아있으면 게스트로도 재열람 허용할지, 로그인 요구할지. (제안: 같은 기기는 게스트로도 재열람 허용 — 기존 §7-2 "게스트 상태 데이터 소실 없음" 원칙과 일관)
2. **카카오 vs 이메일 기본 노출 순서** — 카카오 우선 노출로 확정했으나, 이메일 가입자 비중이 낮을 경우 이메일 폼을 접어두는(토글) UI 최적화 여지 있음.
