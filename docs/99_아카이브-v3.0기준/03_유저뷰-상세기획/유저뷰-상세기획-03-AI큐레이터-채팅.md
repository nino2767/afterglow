# 유저뷰 상세기획 — 화면 3: AI 큐레이터 채팅 (Level 2)

> 버전: v1.0 | 작성일: 2026.07.02 | 상태: 📋 초안
> 근거: v3.0 §5(AI 큐레이터), 데이터 계약 SSOT(00_데이터-계약-SSOT.md), B2C-B2B API 명세 §2-3
> 위치: 유저뷰 5화면 중 3번. **스토리 모듈이 화면으로 드러나는 핵심부** — 여기서 만든 데이터가 여운 리포트(화면4)·초대장(화면5)·어드민 Monitor/Report 전부를 먹여 살림.

---

## 1. 화면의 역할

| 관점 | 역할 |
|---|---|
| 관람객(B2C) | 작품에 대한 실시간 질의응답 + 감정을 말로 풀어내는 대화 상대 |
| 데이터(Bridge) | `emotion_chips(leaf+axis)` · `depth_score` 축적 → TASTE_PROFILE 실시간 갱신 |
| 스토리 모듈 | 전시 서사·작가노트(AI_KNOWLEDGE_BASE)를 RAG 소스로 소비. 대화의 "결"은 concept_type 6모드와 동일한 세계관 언어 사용 |
| 기획사(B2B) | 반복 질문 전담(운영 효율) + "어디서 감동했는가" 텍스트 마이닝 소스 |

**MVP 성공 판정 연결(v3.0 9장):** 사용자가 질문 3회 이상 지속하는가 / 마지막 추천(Bridge)에 긍정 반응하는가.

## 2. 진입 · 이탈 플로우

```
[화면2: 작품 정보카드(Level 1)]
   └─ "AI 큐레이터와 대화하기" 버튼
        └─ [화면3: 채팅] ──────────────┐
             ├─ 뒤로가기 → 화면2 복귀    │ 대화 컨텍스트는 세션 내 유지
             ├─ 다른 작품 QR 스캔 → 화면2(새 작품) → 재진입 시 이어짐
             └─ "관람 완료" (전역 하단) → 화면4 여운 리포트
```

- **첫 진입 시** `curator.chat_start` 이벤트 발화 (세션당 1회 — 퍼널 2단계).
- 대화 스레드는 **세션 단위 1개** (작품별 분리 아님). 작품을 옮기면 AI가 컨텍스트 전환을 인지하고 이어 말함. → 취향 서사가 끊기지 않게 하기 위함.

## 3. 화면 구성 (모바일 < 768px)

```
┌──────────────────────────────┐
│ ← 빛의 심연            ⋯     │  헤더: 현재 작품명 (작품 전환 시 갱신)
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ 🤖 첫마디 (온보딩 키워드   │ │  AI 말풍선
│ │ + 작품 결합 개인화 멘트)   │ │
│ └──────────────────────────┘ │
│              ┌─────────────┐ │
│              │ 사용자 말풍선 │ │
│              └─────────────┘ │
│ ┌──────────────────────────┐ │
│ │ 🤖 답변 (RAG 스트리밍)     │ │
│ │ ┌────────┐┌────────┐     │ │  감정 칩 제안 (답변 하단)
│ │ │#고요함  ││#아련함  │     │ │  ← 전시별 EMOTION_LEAF에서만 노출
│ │ └────────┘└────────┘     │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│ [추천 질문 1] [추천 질문 2]   │  퀵 리플라이 (선택)
│ ┌────────────────────┐ [전송]│  입력바
│ └────────────────────┘  🎤   │  음성 입력(MVP OUT, v2)
└──────────────────────────────┘
```

### 구성 요소 상세

| 요소 | 동작 | 데이터 |
|---|---|---|
| **AI 첫마디** | 온보딩 `initial_keywords` + 현재 작품 메타 결합. 예: "S님, 아까 고요함을 기대하셨죠? 이 파란 빛에서 그런 평온함이 느껴지시나요?" | 읽기: SESSION.initial_keywords, ARTWORK 메타 |
| **감정 칩** | AI 답변 하단에 2~3개 제안. 탭 = 해당 감정 확정 입력 (타이핑 없이 데이터 축적) | 쓰기: `{emotion, axis}` → AI_CONVERSATION.emotion_chips |
| **퀵 리플라이** | 작품별 추천 질문 2개 (KB 세팅). 대화 시작 장벽 낮춤 | 읽기: AI_KNOWLEDGE_BASE |
| **스트리밍 답변** | WSS `server_chunk` → 실시간 타이핑 표시 | API §2-3 그대로 |
| **작품 컨텍스트 배너** | 다른 작품 QR 스캔 후 재진입 시 "지금은 '윤슬' 앞이시네요" 시스템 라인 삽입 | 읽기: artwork_id 변경 감지 |

## 4. 대화 엔진 규칙 (스토리 모듈 사양)

### 4-1. Contextual Engine (RAG)
- 소스: 해당 `exhibition_id`의 AI_KNOWLEDGE_BASE (docent_script, 작가노트, artwork_metadata)만. **전시 밖 지식으로 답하지 않음** (할루시네이션 방지 — 모르면 "그건 작가님도 비밀로 하셨어요" 류의 세계관 내 회피).
- 답변 길이: 2~4문장. 마지막 문장은 가능한 한 **관람객 감정을 되묻는 질문**으로 종결 (감정 데이터 수집 트리거).

### 4-2. Persona Matcher (톤 최적화)
- 초기 톤 = 운영자가 KB에 세팅한 TONE_GUIDE.
- 3턴 이후 사용자 말투·질문 유형 분석 → 톤 미세 조정 (친절한 가이드 ↔ 지적 비평가). 조정 결과는 세션 내 유지, 저장 필드 아님(MVP).

### 4-3. 감정 추출 (SSOT §2 준수)
- `server_final`마다 `extracted_emotions`를 `{emotion, axis}` 배열로 반환. leaf는 해당 전시 EMOTION_LEAF 중 매칭, 미등록 감정어는 `pending_review`로 적재(운영자 승인 큐) — **관람객 화면에는 active leaf만 노출**.
- `depth_score`(1~5): 질문 깊이. 사실형=낮음, 해석·감상형=높음. → 세그먼트 산출 입력.

### 4-4. Bridge Trigger 예고 (화면4 연결)
- 대화 누적 중 TASTE_PROFILE 실시간 갱신. 채팅 화면에서는 **초대장을 직접 노출하지 않음** — 관람 완료(화면4)에서만 발급. 단, 대화가 5턴 이상 + 특정 axis 뚜렷하면 AI가 "관람 끝나면 S님 취향으로 준비한 게 있어요" 1회 티저 가능 (스팸 방지: 세션당 1회 상한).

## 5. 상태 · 엣지 케이스

| 상태 | 처리 |
|---|---|
| AI 응답 실패/타임아웃(>10s) | "잠시 생각이 길어지네요" + 재시도 버튼. 3회 실패 시 Level 1 정보카드로 유도 |
| 오프라인(전시장 음영) | 입력 비활성 + "연결이 불안정해요" 배너. 재연결 시 미전송 메시지 복구 |
| 세션 만료(entry QR 재스캔 필요) | 온보딩 화면으로 리다이렉트, 기존 대화는 visitor 기준 복구 |
| 부적절 발화(욕설 등) | 세계관 내 부드러운 회피 + 감정 데이터 미기록 |
| 무응답 이탈 | 별도 푸시 없음 (웹앱, MVP) |

## 6. 이벤트 · 데이터 계약 매핑

| 트리거 | 이벤트/쓰기 | SSOT 근거 |
|---|---|---|
| 첫 메시지 전송 | `curator.chat_start` (세션 1회) | §3 퍼널 2단계 |
| 매 `server_final` | AI_CONVERSATION append: messages, emotion_chips, depth_score | §6 엔티티 |

| 대화 누적 | TASTE_PROFILE.axis_distribution 실시간 갱신 | §4 세그먼트 입력 |

## 7. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| 텍스트 채팅 + 스트리밍 | 음성 입력/출력 |
| 감정 칩 + 퀵 리플라이 | Vision-to-Text (사진으로 작품 인식) |
| 세션 단위 단일 스레드 | 작품별 스레드 분리·히스토리 탐색 |
| 핵심 작품 3~5개 KB (v3.0 9장) | 전 작품 커버 |
| Bridge 티저 1회 | 실시간 팝업 추천 카드 |

## 8. 남은 결정 포인트

1. **감정 칩 제안 개수/빈도** — 매 답변마다 vs 감정 언급 감지 시에만 (현안: 매 답변 2~3개, 과하면 조정)
2. **첫마디 발화 시점** — 채팅 진입 즉시 vs 사용자 첫 입력 후 (현안: 진입 즉시 — Zero-Draft 원칙과 동일하게 빈 화면 금지)
