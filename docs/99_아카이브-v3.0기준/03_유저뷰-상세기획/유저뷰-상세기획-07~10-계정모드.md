# 유저뷰 상세기획 — 화면 7~10 (계정 모드)

> 버전: v1.0 | 작성일: 2026.07.03 | 상태: 📋 초안
> 근거: 유저뷰 IA v1.0, SSOT v1.2 §7, 스핀오프 팝업 운영정책(상태머신)
> 전제: 4화면 모두 **계정 모드**(하단 탭바) 소속. 공통 셸·계정 아이콘 규칙은 화면0 문서 §3 참조. 카드 탭 시 화면4·5로 이동하는 것은 **열람 모드**(IA §2)이며 신규 화면이 아니다.

## 공통 — 카드 컴포넌트 데이터 소스

4화면이 공유하는 원천은 두 개뿐이다. **신규 데이터 모델 불필요** — 기존 엔티티 조합으로 전부 구성된다.

```
visitedExhibitions = TASTE_PROFILE (account_id 경유, 전시별 1개)
invites            = SPINOFF_INVITE (account_id 경유) + 연결된 EXHIBITION(type=spinoff).status
```

`EXHIBITION.status`는 운영정책 상태머신 그대로: `planning|ready|open_concurrent|open_solo|closed|archived`.

## 화면 7: 마이페이지 (프로필·설정 전용 — 범위 축소)

> 기존 화면0 문서 §2는 리포트·초대장까지 뭉쳐 다룬 초안이었다. 본 절이 §2를 대체하며, 프로필/설정만 남긴다.

### 역할
계정 정보 확인·수정, 알림 설정, 로그아웃. **목록성 콘텐츠 없음** (그건 8·9의 역할).

### 화면 구성
```
{nickname}님
카카오 계정 연결됨 / 이메일: xxx@xxx.com
─────────────────
🔔 알림 설정
  전체 마케팅 알림        [ON/OFF]
  전시별 팝업 오픈 알림    [전시별 세부 토글]
─────────────────
계정 전환/연결  (카카오 계정에 이메일 추가 등 — v2)
로그아웃
```

### 이벤트
없음 (설정 변경은 ACCOUNT.marketing_consent 갱신만).

### MVP IN/OUT
IN: 프로필 조회, 전체/전시별 알림 토글, 로그아웃 · OUT: 프로필 사진, 계정 삭제 자동화(고객센터 안내로 대체), 비밀번호 변경(화면0 §5와 동일 사유)

## 화면 8: 마이리포트

### 역할
account에 연결된 전체 여운 리포트(TASTE_PROFILE)를 모아보는 아카이브. **크로스디바이스 요구(로그인 사유①)의 실제 구현부.**

### 화면 구성
```
마이리포트 (총 3개)
┌────────────┐┌────────────┐
│ [axis 그라데이션] ││ [axis 그라데이션] │
│ 빛의 심연    ││ 무한의 정원  │
│ 2026.05.16  ││ 2026.03.02  │
│ "고요한 블루" ││ "따뜻한 봄"  │
└────────────┘└────────────┘
```
- 카드 = 화면4 페르소나 카드 축소판(그라데이션+persona_copy 재사용).
- 탭 → 화면4 **열람 모드** 진입.
- 정렬: 최신 관람순.
- **빈 상태**: "아직 다녀온 전시가 없어요 + QR 스캔하기" (화면10과 동일 CTA 재사용).

### 이벤트
없음 (열람은 조회일 뿐, 화면4 진입 시점의 별도 이벤트는 두지 않음 — 퍼널 오염 방지).

### MVP IN/OUT
IN: 카드 목록, 열람 모드 진입 · OUT: 리포트 검색/필터, 삭제/비공개 처리

## 화면 9: 초대장

### 역할
account에 연결된 전체 초대장(SPINOFF_INVITE) 아카이브. **로그인 사유②(재접속 알림)의 목록 뷰.**

### 화면 구성
```
초대장 (진행 중 1 · 종료 2)

━━ 진행 중 ━━
┌──────────────────────────┐
│ 어비스 티 라운지 · 빛의 심연  │
│ [sensory] D-12 남음         │  ← open_concurrent/open_solo
│ [확인하기 →]                │
└──────────────────────────┘

━━ 종료 ━━
┌──────────────────────────┐
│ 무한의 정원 팝업 (종료)       │  ← closed/archived, 흐리게
│ [다시보기 →]                │
└──────────────────────────┘
```
- 진행 중/종료 그룹 분리, 진행 중 내에서는 유효기간 임박순.
- 탭 → 화면5 **열람 모드** 진입.
- **오픈 전 상태**(EXHIBITION.status = planning/ready): "오픈 알림 신청됨" 뱃지로 별도 그룹 표시(화면4 §5 엣지케이스와 동일 상태 재사용).

### 이벤트
없음 (조회 전용).

### MVP IN/OUT
IN: 상태별 그룹·정렬, 열람 모드 진입 · OUT: 초대장 공유하기, 알림 개별 스누즈

## 화면 10: 홈

### 역할
계정 모드의 기본 진입 탭. **"지금 뭘 해야 하는지" 1가지를 먼저 보여주고, QR 스캔을 항상 열어둔다.** 마켓플레이스(여러 전시 탐색)가 아니다 — v3.0 모델상 전시 발견은 여전히 오프라인 QR이 유일한 입구.

### 화면 구성
```
{nickname}님, 안녕하세요

┌──────────────────────────┐
│  📷  QR 스캔하고 관람 시작    │  ← 상시 최상단 고정 CTA
└──────────────────────────┘

[지금 확인할 것]                ← 우선순위 엔진 1건만 노출 (아래 §A)
┌──────────────────────────┐
│ 어비스 티 라운지 · D-12       │
│ "곧 종료돼요, 지금 방문해보세요"│
│ [확인하기 →]                 │
└──────────────────────────┘

최근 다녀온 전시
┌────────┐┌────────┐
│빛의 심연 ││무한의 정원│        ← 화면8과 동일 카드, 최근 2개만
└────────┘└────────┘
[마이리포트 전체보기 →]
```

### §A. "지금 확인할 것" 우선순위 엔진 (전시 상태머신 기반)

계정에 연결된 모든 초대장을 순회해 **1건만** 골라 노출한다. 여러 건이어도 나열하지 않음(정보 과부하 방지 — 나머지는 화면9로).

| 우선순위 | EXHIBITION(spinoff).status | 노출 문구 | 근거 |
|---|---|---|---|
| 1 (최고) | `open_solo` | "곧 종료돼요 · D-N일 남음" | 단독운영 구간, v3.0 §7 "놓친 사람 유입" 긴급성 |
| 2 | `open_concurrent` | "지금 방문하면 딱이에요" | 관람 직후 전환 최적 구간 |
| 3 | `ready`/`planning` | "팝업 준비 중이에요, 오픈하면 알려드릴게요" | 기대감 유지, CTA는 알림 확인용 |
| 4 (최저) | `closed`/`archived` | (노출 안 함 — 화면9에서만 조회) | 종료된 건 홈에서 긴급성 없음 |

- 동순위 복수 건이면 유효기간 임박순.
- 해당 없음(초대장 0건) → 섹션 자체 숨김, "최근 다녀온 전시"도 없으면 **완전 빈 상태**(§B)로 전환.

### §B. 빈 상태 (가입 직후, 참여 이력 0건)
```
{nickname}님, 안녕하세요

┌──────────────────────────┐
│  📷  QR 스캔하고 관람 시작    │
└──────────────────────────┘

아직 다녀온 전시가 없어요.
전시장 입구 QR을 스캔하면 시작할 수 있어요.
```

### §C. 서비스 확장 단계별 홈의 역할 변화 (간단 버전 — v2 로드맵 메모)

MVP는 파일럿 1~2개 고객사·전시 기준(v3.0 §8)이라, 아래는 지금 구현 대상이 아니라 **성장에 따라 홈이 왜/어떻게 달라질지에 대한 방향 메모**다.

| 단계 | 고객사·전시 규모 | 홈의 역할 |
|---|---|---|
| **Stage 1 · 파일럿 (현재)** | 1~2개사, 전시 1~2개 | 사실상 QR CTA + 빈 상태가 대부분. "지금 확인할 것" 섹션이 채워질 일이 드묾 |
| **Stage 2 · 초기 확장** | 3~5개사, 전시 여러 개 동시 진행 | 우선순위 엔진(§A)이 실제로 의미를 갖기 시작 — 한 계정이 여러 전시 이력을 가짐 |
| **Stage 3 · 성숙** | 다수 파트너, 반복 방문자 다수 | AFTERGLOW 에디토리얼 큐레이션(소량 추천) 섹션 추가 **검토 가능**. 단, 이는 탐색·검색 UI(마켓플레이스)가 아니라 "이런 분이 좋아하실 전시" 식 큐레이션 콘텐츠로 제한 — 마켓플레이스 전환은 별도 비즈니스 의사결정 필요, 본 문서 범위 아님 |

### 이벤트
없음 (조회·CTA 진입점일 뿐, 화면10 자체는 이벤트를 발화하지 않음).

### MVP IN/OUT
IN: QR CTA, 우선순위 엔진(§A) 1건 노출, 최근 리포트 2개 미리보기, 빈 상태 · OUT: 에디토리얼 추천(§C Stage 3), 위치 기반 근처 전시 안내

## 공통 결정 포인트

1. **"지금 확인할 것"이 0건이지만 리포트는 있는 경우** — 섹션을 완전히 숨길지, "모두 확인했어요" 같은 완료 상태를 보여줄지. (제안: 완전 숨김이 더 깔끔 — 빈 성취 배지는 과함)
2. **화면9 종료 그룹의 보존 기간** — 무한정 보관 vs N개월 후 정리. (제안: MVP는 무한정, 데이터 삭제 정책은 별도 논의)
