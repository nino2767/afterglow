# AFTERGLOW 스핀오프 팝업 운영 정책

> 버전: v1.0 | 작성일: 2026.07.02 | 상태: 📋 초안 (결정 포인트 ❓ 포함)
> 근거: v3.0 §7(서비스 타임라인)·§8(파일럿 조건)·§3(비즈니스 구조), SSOT v1.1
> 성격: **운영 규칙의 SSOT.** 어드민 Phase 4(팝업 실행)·초대장 유효기간·퍼널 집계 기간·랜딩 상태 화면이 모두 이 정책을 따른다.

---

## 1. 개설 시점 (When)

v3.0 §7 오버랩 구조를 상태 머신으로 고정:

```
전시 오픈(D-day)                    전시 종료(D+60 예시)      팝업 종료
   │◄──────── 동시 운영 구간 ────────►│◄── 단독 운영 구간 ──►│
   │        (main + spinoff)         │   (spinoff only,      │
   │                                 │    최대 +21일)         │
```

| 항목 | 정책 | 근거 |
|---|---|---|
| 팝업 오픈 시점 | **본전시 오픈과 동시(D-day) ~ 본전시 기간 중 임의 시점.** 운영자가 Phase 4에서 오픈일 설정 | v3.0 §7 "동시 운영 → 관람 직후 이어지는 경험, 전환율 최대" |
| 단독 연장 상한 | 본전시 종료 후 **최대 21일** (`main.end + 21d ≥ spinoff.end` 검증 — 어드민 저장 시 강제) | v3.0 §2 "최대 21일간 단독 연장" |
| 최소 리드타임 ❓ | 계약 → 팝업 오픈까지 최소 **2주** (제안: AI 기획 24h + MD 제작·공간 셋업 실무 기간) | v3.0 미규정 — **결정 필요** |
| 파일럿 개설 조건 | 본전시 잔여 ≥ 4주 / 담당자 데이터·기술 관심 / 공간 확보 가능 / IP 저작권 단순 | v3.0 §8 체크리스트 그대로 |

### 팝업 상태 머신 (EXHIBITION.status, type=spinoff)

```
planning → ready → open_concurrent → open_solo → closed → archived
                        │ (main 종료 시 자동 전이)   │
                        └── 초대장 발급 가능 구간 ────┘
```

- `open_solo → closed` 전이 시: 랜딩은 "종료 화면"(아카이브+차기 알림), 미사용 초대장 자동 만료.
- 상태 전이는 어드민 수동 + 날짜 기반 자동 (main.end 도달 시 `open_concurrent→open_solo` 자동).

## 2. 대상 (Who) — 초대장 발급·입장 정책

| 대상 | 초대장 | 입장/랜딩 | 근거 |
|---|---|---|---|
| **온보딩 관람객** (main 세션 보유) | 관람 완료 시 개인화 초대장 자동 발급 (화면4) | A경로 풀 개인화 | 핵심 루프 |
| **비온보딩 관람객** (관람했으나 AI 미사용) | 출구 QR 축약 온보딩 → default 초대장 | A경로 준개인화 | 화면4 §5 회수 경로 |
| **일반 대중** (본전시 미관람) | 초대장 없음 | C경로 일반 랜딩 — 입장·구매 모두 허용 | v3.0 §7 "놓친 사람 유입" |
| 재방문 관람객 | 기존 초대장 재사용 (재발급 없음) | B경로 | 화면5 §2 |

- ❓ **팝업 입장료**: v3.0 수익모델에 "팝업 입장료 및 체험료"가 있으나 무료/유료 기준 미정. **제안: MVP는 무료 입장 + 체험/굿즈 과금** (전환 장벽 제거, 파일럿 데이터 극대화). 유료화는 고객사 설정 옵션으로 v2.
- 관람객 연령·인원 제한: 전시장 정책 승계 (AFTERGLOW 자체 제한 없음).

## 3. 초대장 유효기간 (화면4 §8-1 확정)

- **발급 시점 ~ 팝업 종료일(closed 전이)까지.** 단독 운영 구간 포함 → "놓친 사람도 초대장으로 재방문" 효과 보존.
- 만료 후 접근: 랜딩 종료 화면 + 차기 전시 opt-in (데이터 루프 지속).
- 웰컴 혜택(셀프 체크인 보상): 초대장과 동일 만료. 1회 사용.

## 4. 개설 단위 (What) — 본전시:팝업 관계

- **MVP: 1 main : 1 spinoff.** (SSOT v1.1은 1:N 허용 구조지만 운영은 1:1로 시작)
- 팝업 존 수: **3~5개** (본전시 핵심 작품 3~5개와 대칭 — v3.0 §9 MVP 범위 준용).
- 장소: 전시 철거 후 유휴 공간 또는 인근 거점 (v3.0 설득 포인트 1 "추가 대관 리스크 없음"). 장소 확보는 AFTERGLOW Total Care 범위.

## 5. 퍼널·지표 집계 기간

| 지표 | 집계 창 |
|---|---|
| 전환율 20% (4단계/1단계) | main 오픈 ~ **spinoff closed** (단독 구간 포함) |
| 팝업 체험 지표 (§3-1 이벤트) | spinoff open ~ closed |
| Insight Reporter 최종 리포트 | closed 후 7일 내 생성 (v3.0 MVP 7주차 일정 대응) |

2. **MVP 무료 입장** — 입장 과금 없이 체험·굿즈 과금으로 시작하는 방향 동의 여부.
