# 유저뷰 IA (Information Architecture)

> 버전: v1.0 | 작성일: 2026.07.03 | 상태: 📋 초안 (홈 탭 성격 = 가정 포함, 하단 §5 확인 필요)
> 근거: SSOT v1.2 §7, 화면0~6 상세기획, v3.0(B2B SaaS 모델 — 마켓플레이스 아님)
> 성격: **전체 구조 마스터 문서.** 개별 화면 상세기획보다 상위 — 진입점·모드·화면 관계를 여기서 먼저 고정하고, 화면별 문서는 이 구조를 따른다.

---

## 1. 왜 필요했나

화면이 0~9로 늘고 하단 탭바가 생기면서 "탭을 누르면 어디로 가는지", "그 화면을 나가면 어디로 돌아가는지"가 문서마다 흩어져 암묵적으로만 정의돼 있었다. 특히 계정 모드에서 과거 리포트를 열람하는 것이 **가이드 모드 재진입인지, 별도 상태인지**가 불명확했다. 본 문서가 그 답을 고정한다.

## 2. 3개 모드

| 모드 | 진입 조건 | 특징 |
|---|---|---|
| **가이드 모드** | QR 스캔 (전시 입구/출구), 초대장 URL 재접속 | 상단 바만, 하단 탭바 없음. 전시/팝업에 스코프됨. 화면1~6 |
| **계정 모드** | 로그인 상태로 앱 재실행, 또는 계정 아이콘→마이페이지 진입 | 하단 탭바 4종. 전시 스코프 없음(전체 계정 뷰). 화면7·8·9·10 |
| 🆕 **열람 모드** | 계정 모드(화면8·9)에서 과거 리포트/초대장 카드 탭 | 화면4·5 UI를 **재사용**하되 read-only 크롬. 신규 화면 아님 — 기존 화면의 상태 변형 |

**열람 모드가 필요한 이유**: 화면4·5는 원래 "지금 이 전시를 막 관람한 사람"을 전제로 한 라이브 상태(관람완료 CTA, 실시간 Bridge 등)다. 마이리포트에서 지난 전시 카드를 눌러 재열람할 때 그 CTA들이 그대로 뜨면 어색하다. 그래서 같은 화면을 쓰되 크롬만 바꾼다:

| 요소 | 가이드 모드(라이브) | 열람 모드(재열람) |
|---|---|---|
| 상단 뒤로가기 | 이전 가이드 화면으로 | 화면8/9(마이리포트/초대장)로 |
| 화면4 초대장 CTA | 정상 노출·클릭 가능 | 유효기간 내면 동일 노출(초대장은 여전히 유효) |
| 화면5 셀프체크인 | 정상 동작 | 이미 redeemed면 "체크인 완료" 상태로 표시, 재클릭 무의미 처리 |
| 상단 배너 | 없음 | "보관함에서 보는 중" 얇은 배너 1줄 |

## 3. 전체 사이트맵

```
AFTERGLOW
│
├─ 진입점 A: QR 스캔 (전시 입구) ─────────────────────► 가이드 모드
│    화면1 온보딩 (게스트 시작 · 하단에 로그인 링크)
│     └► 화면2 스냅&정보카드 ⇄ 화면3 AI 큐레이터 채팅
│          └►[관람완료]► 화면4 여운리포트+Bridge
│               ├►[계정 만들기 CTA]► 화면0 로그인(오버레이) ─► 화면4 복귀(병합)
│               └►[초대장 CTA]► 화면5 스핀오프 랜딩
│                    ├►[셀프체크인]► 화면6 팝업 AI 큐레이터
│                    └►[구매/예약 링크]► 외부 이동
│
├─ 진입점 B: QR 스캔 (전시 출구, 비온보딩 회수) ──────► 화면4 (축약 온보딩 선행)
│
├─ 진입점 C: 초대장 URL 재접속 ───────────────────────► 화면5 (B/C 경로)
│
├─ 진입점 D: 계정 아이콘 탭 (화면1~6 어디서든, 상시) ──► 화면0 로그인(오버레이)
│                                                          └► 성공 시 원래 화면 복귀 + 병합
│                                                          └► 또는 화면7 마이페이지 직행
│
└─ 진입점 E: 로그인 상태로 앱 재실행 ─────────────────► 계정 모드 (기본 탭: 홈)
     하단 탭바 ──┬─ 화면10 홈 (참여 이력 요약 + QR 스캔 CTA)
                 ├─ 화면8 마이리포트 (목록) ─[카드 탭]─► 화면4 열람 모드
                 ├─ 화면9 초대장 (목록) ─[카드 탭]─► 화면5 열람 모드
                 └─ 화면7 마이페이지 (프로필·설정·로그아웃)
```

## 4. 화면 인벤토리 (전체 확정본)

| # | 화면 | 모드 | 상태 |
|---|---|---|---|
| 0 | 로그인·회원가입 | 오버레이(모드 무관) | ✅ 완료 |
| 1 | 온보딩 | 가이드 | ✅ 완료 |
| 2 | 스냅&정보카드 | 가이드 | ✅ 완료 |
| 3 | AI 큐레이터 채팅 | 가이드 | ✅ 완료 |
| 4 | 여운리포트+Bridge | 가이드 / 열람(재사용) | ✅ 완료 (열람 크롬만 §2 추가분) |
| 5 | 스핀오프 랜딩 | 가이드 / 열람(재사용) | ✅ 완료 (열람 크롬만 §2 추가분) |
| 6 | 팝업 AI 큐레이터 | 가이드 | ✅ 완료 |
| 7 | 마이페이지 (프로필·설정 전용으로 **범위 축소**) | 계정 | 🔧 재작성 필요 |
| 8 🆕 | 마이리포트 (리포트 카드 목록) | 계정 | 🆕 신규 작성 필요 |
| 9 🆕 | 초대장 (초대장 카드 목록) | 계정 | 🆕 신규 작성 필요 |
| 10 🆕 | 홈 (참여 이력 요약 + QR 안내) | 계정 | 🆕 신규 작성 필요 |

> 기존 "화면0 문서"의 §2(마이페이지)는 7·8·9를 뭉쳐 다뤘던 **초안**이었다. 본 IA 확정에 따라 7은 프로필/설정만 남기고, 8·9는 독립 문서로 분리한다.

## 5. 화면 10 (홈) — 성격 확정 (가정)

**채택: 미니 대시보드.** 근거 — v3.0은 AFTERGLOW를 특정 고객사 전시에 QR로 진입하는 B2B SaaS 모델로 정의하며, 여러 전시를 가로지르는 탐색·검색 기능(마켓플레이스)은 언급되지 않는다. 전시 발견은 여전히 **오프라인 QR이 유일한 입구**다.

```
┌──────────────────────────────┐
│  {nickname}님, 안녕하세요        │
│                                │
│  ┌──────────────────────────┐ │
│  │  📷  QR 스캔하고 관람 시작   │ │  ← 주 CTA (카메라 오픈)
│  └──────────────────────────┘ │
│                                │
│  최근 다녀온 전시                │
│  ┌────────┐┌────────┐        │
│  │빛의 심연 ││        │        │  ← 화면8과 동일 카드, 최근 2~3개만
│  └────────┘└────────┘        │  탭 시 화면4 열람 모드
│  [마이리포트 전체보기 →]        │  ← 화면8 이동
│                                │
│  진행 중인 초대장 1건            │  ← 있을 때만 노출
│  [확인하기 →]                  │  ← 화면9 이동
└──────────────────────────────┘
```

- **빈 상태(가입 직후, 참여 이력 없음)**: "QR 스캔 CTA"만 크게, 하단 섹션들은 "아직 다녀온 전시가 없어요" 안내로 대체.
- v2 확장 여지: 파트너사가 늘어나면 "홈"에 "제휴 전시 안내" 섹션을 얹을 수 있으나, 이는 탐색 기능이 아니라 AFTERGLOW의 공지/큐레이션 콘텐츠 성격으로 제한 (마켓플레이스 검색 UX는 별도 스코프 논의 필요).

## 6. 모드 전환 규칙 요약

| 전환 | 트리거 | 비고 |
|---|---|---|
| 가이드→계정 | 계정 아이콘 탭 → "마이페이지로 이동" 선택 | 관람 흐름 자연 종료가 아닌 **명시적 이탈** |
| 계정→가이드(신규) | 화면10 "QR 스캔" | 새 전시 스캔 시 화면1부터 |
| 계정→열람 | 화면8/9 카드 탭 | 화면4/5 재사용, §2 열람 크롬 적용 |
| 열람→계정 | 열람 모드 뒤로가기 | 카드를 탭했던 화면(8 또는 9)으로 복귀 |
| 오버레이(화면0) | 계정 아이콘, 화면4 CTA, 화면5 CTA | 완료/취소 시 **원래 컨텍스트로 복귀** (모드 전환 아님) |
