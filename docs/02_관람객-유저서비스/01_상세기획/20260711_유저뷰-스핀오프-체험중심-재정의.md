# 유저뷰 스핀오프 재정의 — 체험 중심 전환

> 버전: v1.0 | 작성일: 2026.07.08 | 상태: ✅ 확정
> 근거: v3.0 §2(스핀오프 = "전시 IP를 활용한 2차 **체험 공간**")·컨셉 샘플 A/B/C(체험이 본체, 굿즈는 결과물)·APPENDIX 3(아르떼뮤지엄 TEA BAR = 체험 4,000원), SSOT v1.2, 화면5·6 상세기획
> 성격: **방향 수정.** 기존 화면5·6·퍼널 설계가 굿즈 판매에 기울어 있던 것을 체험 중심으로 되돌린다.
> ⚠️ 화면5·6 문서, SSOT §3(퍼널), 통합리포트 문서를 **오버라이드**함.

## 1. 문제 — 스핀오프가 "굿즈샵"으로 설계돼 있었다

v3.0의 스핀오프는 **2차 체험 공간**이다. 컨셉 샘플도 전부 체험이 본체다.

| 컨셉 | 체험(본체) | 굿즈(결과물) |
|---|---|---|
| A 딥블루 레코드 | 청음 부스에서 나만의 사운드트랙 큐레이션 | 그 트랙이 담긴 LP |
| B 루미너스 랩 | AI와 나만의 심해 생물 디자인 | 그 생물이 3D 프린팅된 피규어 |
| C 어비스 티 라운지 | 감정에 맞춘 티 페어링 체험 | 인센스 스틱 |

벤치마킹한 아르떼뮤지엄 TEA BAR도 **"찻잔을 놓으면 꽃이 피어나는 경험"을 4,000원에 파는 것**이지, 찻잔을 파는 게 아니다.

그런데 기존 화면 설계는 이렇게 돼 있었다:

- 화면5 랜딩: 주 CTA가 "예약/구매하러 가기", 추천 MD 카드가 상단 진열
- 화면6 존 해설: 각 존이 "연계 MD 카드 → 구매"로 종결
- 퍼널 6단계: 여정의 종착점이 `purchase.link_clicked`
- 통합 리포트: "나의 소장품" 섹션

→ **존(SPINOFF_ZONE)이 체험 단위가 아니라 MD 진열대처럼 다뤄졌다.**

## 2. 재정의 — 굿즈를 빼는 게 아니라, 체험에 종속시킨다

```
기존 구조: 존 체험 → 연계 MD 진열 → 구매        (체험이 굿즈로 가는 통로)
개정 구조: 존 체험 → 체험의 결과물이 곧 굿즈       (굿즈가 체험의 산물)
```

- **진열식 MD 카드 제거.** 체험하기 전에는 굿즈가 보이지 않는다.
- 굿즈는 **체험 완료 후에만** 등장한다 — "방금 만든 그 생물, 피규어로 받아보시겠어요?"
- 이렇게 하면 v3.0의 굿즈 서사(MD Matcher·마플 POD·커스텀 MD)를 **버리지 않으면서** 스핀오프의 정체성이 체험 공간으로 유지된다.

### 수익 구조 (MVP 파일럿)

| 항목 | MVP | 근거 |
|---|---|---|
| 팝업 입장 | **무료** | 전환 장벽 제거, 파일럿 데이터 극대화 |
| **체험** | **유료** (주 수익원) | v3.0 아르떼뮤지엄 벤치마킹(체험 자체가 상품) |
| 굿즈 | 체험 결과물의 부차적 제작 (외부 커머스) | 실제 결제·제작 연동은 Phase 2 |

## 3. 화면5 (스핀오프 랜딩) 개정

### 3-1. 구성 변경

```
기존                          개정
① 히어로 (키비주얼)            ① 히어로 (키비주얼 + 체험 카피)
② 추천 체험·굿즈 카드 2~3종  →  ② 이 공간의 체험 (존 소개)      ← 주 콘텐츠
③ [예약/구매하러 가기 →]        ③ [방문 예약하기 →]              ← CTA 변경
④ 팝업 정보 (위치·기간)        ④ 팝업 정보 (위치·기간·체험료)
⑤ 셀프 체크인                  ⑤ 셀프 체크인
                              ⑥ (하단) 이곳에서 만들 수 있는 것  ← 굿즈, 최하단·부차
```

### 3-2. 핵심 변경점

| 요소 | 기존 | 개정 |
|---|---|---|
| ② 주 콘텐츠 | 추천 MD 2~3종 (가격·이미지) | **존별 체험 소개 2~3개** (무엇을 하는가, 소요시간, 체험료) |
| ③ 주 CTA | "예약/구매하러 가기" → 외부 커머스 | **"방문 예약하기"** → 예약 페이지 |
| ⑥ 굿즈 | 상단 진열 | **최하단 안내 수준** — "체험 후 제작 가능한 것들" 소개, 가격·구매버튼 없음 |

- `purchase.link_clicked` 이벤트는 ⑥에서 발화 가능하되, **퍼널 단계가 아닌 참고 지표**로 강등(§5).

## 4. 화면6 (팝업 큐레이터) 개정

### 4-1. 존 해설의 무게중심 이동

```
기존: [존 설명] → [연계 MD 카드] → [구매]
개정: [존 설명] → [체험 안내·참여] → [체험 완료] → (선택) [결과물 제작 안내]
```

- 존 상세(화면6 §3-③)에서 **MD 카드를 제거**하고, 대신 **체험 참여 안내**를 넣는다: 무엇을 하는지, 얼마나 걸리는지, 체험료.
- 체험 완료 후에만 결과물 안내 노출: "방금 만든 심해 생물을 피규어로 받아보시겠어요?" (외부 링크)

### 4-2. 신규 — 체험 참여·완료 추적

체험이 상품이 되었으므로, **체험 자체를 데이터로 잡아야 한다.**

```
SPINOFF_ZONE
  + experience_type: enum        🆕 free | paid
  + experience_price: int?       🆕 유료 체험 시 가격
  + duration_min: int?           🆕 예상 소요 시간
  + has_output: bool             🆕 결과물(굿즈화 가능) 존재 여부

ZONE_EXPERIENCE                  🆕 신규 엔티티
  experience_id (PK)
  exhibition_id (FK, spinoff), session_id (FK), zone_id (FK)
  started_at, completed_at?
  paid: bool
  output_claimed: bool           결과물 제작 신청 여부
```

### 4-3. 신규 이벤트

| 이벤트 | 시점 | 의미 |
|---|---|---|
| `experience.started` | 존 체험 시작 | 체험 참여 |
| `experience.completed` | 체험 완료 | **새 성공 지표** |
| `experience.output_claimed` | 결과물 제작 신청 | 굿즈 연결(부차) |

## 5. 퍼널 개정 (SSOT §3) — 가장 중요한 변경

### 5-1. 6단계 재정의

| # | 기존 | 개정 |
|---|---|---|
| 1 | `session.start` | 동일 |
| 2 | `curator.chat_start` | 동일 |
| 3 | `invite.issued` | 동일 |
| 4 | `invite.landing_viewed` | 동일 **(MVP 성공지표 20% 판정 기준 — 불변)** |
| 5 | `invite.redeemed` (셀프 체크인) | 동일 |
| 6 | ~~`purchase.link_clicked`~~ | **`experience.completed`** 🆕 |

- **`purchase.link_clicked`는 퍼널에서 제외** → 참고 지표(`landing.organic_viewed`와 같은 층위)로 강등.
- 근거: MVP에서 구매는 외부 커머스 링크 클릭까지만 추적하므로 **실제 결제 여부를 알 수 없다.** 검증 불가능한 것을 퍼널 최종 단계로 두는 것은 부적절. 반면 `experience.completed`는 **현장에서 직접 측정 가능**하고, 스핀오프의 존재 이유(2차 체험)와 정확히 일치한다.
- **MVP 성공지표(전환율 20% = 4단계/1단계)는 변경 없음** — 퍼널 6단계 교체는 핵심 지표에 영향을 주지 않는다.

### 5-2. 참고 지표로 이동

```
퍼널 외 참고 지표
  landing.organic_viewed      (기존)
  spinoff.session_start / zone_viewed / curator_chat   (기존)
  purchase.link_clicked       🆕 강등 — 굿즈 관심도 프록시
  experience.output_claimed   🆕 결과물 제작 신청률
```

| ④ | "나의 소장품" (PURCHASE 기반) | **"내가 한 체험"** (ZONE_EXPERIENCE 기반) |

- 결과물 제작을 신청했다면 부차 표기: "· 피규어 제작 신청함"

## 9. 남은 결정 포인트

1. **체험료 결제 수단** — 현장 결제 vs 사전 예약 결제. MVP는 자체 결제 없음(v3.0 §9 OUT)이므로 현장 결제 유력.
2. **무료 체험 존과 유료 체험 존의 혼재** — 한 팝업 안에 둘 다 있을 때 UI 표기 규칙.
3. **v3.0 수익 시뮬레이션 재계산 필요** — 기존 객단가 1.5만 원은 굿즈 구매 전제. 체험료 중심으로 바뀌면 단가·전환율 가정을 다시 잡아야 함(사업 문서 영향).
