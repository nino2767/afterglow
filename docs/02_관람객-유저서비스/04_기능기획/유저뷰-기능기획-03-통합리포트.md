# 유저뷰 기능기획 03 — 본전시 + 스핀오프 통합 리포트

> 버전: v1.0 | 작성일: 2026.07.08 | 상태: 📋 초안
> 우선순위: **3순위** (Sol 논의 신규항목 6건 중) — 브랜드 완결성. 다중 스핀오프 확정으로 설계 가능해짐
> 근거: v3.0 서비스명 정의("AFTERGLOW — 전시가 끝난 후에도 남는 빛")·§6 여정 5단계(사후 경험: 디지털 아카이브), 화면4·6·8, SSOT v1.2, 스핀오프 확장 스펙개정
> 성격: **누적 아카이브** — 신규 화면이 아니라 화면8(마이리포트)의 카드가 성장한 결과물

---

## 1. 문제 정의 — 지금 리포트가 두 개, 그런데 여정은 하나

| 현재 | 시점 | 담는 것 | 한계 |
|---|---|---|---|
| 화면4 여운 리포트 | 본전시 관람 완료 직후 | 취향 카드, 하이라이트, 초대장 | 스핀오프를 아직 안 갔으므로 **미완성** |
| 화면6 미니 리포트 | 팝업 체험 종료 직후 | 팝업에서의 반응 1카드 | 본전시 맥락이 빠진 **파편** |

관람객의 실제 여정은 `본전시 → 초대장 → 팝업 → 굿즈`로 이어지는 **하나의 서사**인데, 그 서사를 통째로 되돌아볼 수 있는 곳이 없다. v3.0의 서비스명 자체가 "전시가 끝난 후에도 남는 빛"이고 여정 5단계가 "디지털 아카이브 확인"인데, **그 아카이브의 실체가 정의된 적이 없다.**

---

## 2. 설계 결정 — "새 화면이 아니라, 카드가 자라는 것"

### 2-1. 왜 신규 화면을 만들지 않는가

통합 리포트를 별도 화면으로 만들면 "리포트가 세 개"가 된다. 관람객 입장에선 어느 게 진짜인지 혼란스럽다.

**대신 화면8(마이리포트)의 전시 카드가 여정 진행에 따라 성장하는 구조**로 간다. 카드를 탭해 들어가면(열람 모드, IA §2) 그 시점까지의 **가장 완전한 버전**이 보인다.

```
여정 진행에 따른 카드 상태 변화

[본전시만 관람]        [팝업 방문 후]          [굿즈 구매 후]
┌──────────┐        ┌──────────┐        ┌──────────┐
│ 빛의 심연  │   →    │ 빛의 심연  │   →    │ 빛의 심연  │
│ 취향카드   │        │ + 팝업 1곳 │        │ + 팝업 1곳 │
│           │        │           │        │ + 소장품 2 │
└──────────┘        └──────────┘        └──────────┘
  = 화면4 그대로        = 통합 리포트 v1      = 통합 리포트 v2 (완결)
```

- 본전시만 봤으면 지금의 화면4와 동일하게 보인다 — **하위 호환**.
- 스핀오프를 다녀오면 그 섹션이 **추가로 붙는다**. 리포트가 "다시 쓰이는" 게 아니라 "이어 쓰인다".
- 다중 스핀오프(2개 이상 방문)면 방문한 만큼 섹션이 쌓인다.

### 2-2. 진입 경로

| 경로 | 도착 |
|---|---|
| 화면8 마이리포트 → 카드 탭 | 통합 리포트 (열람 모드) |
| 화면10 홈 → 최근 다녀온 전시 카드 탭 | 동일 |
| 화면6 미니 리포트 → "전체 여정 보기" | 동일 (팝업 직후 자연스러운 연결) |
| 알림 N4(차기 전시) 딥링크 | 화면10 경유 |

**화면4(관람 직후)에서는 통합 리포트로 갈 수 없다** — 아직 스핀오프 데이터가 없으므로 통합할 것이 없다. 화면4는 여전히 "그날의 리포트 + 초대장"이다.

## 3. 통합 리포트 구성

```
┌──────────────────────────────┐
│ ✨ 빛의 심연에서의 여정         │  ① 헤더 (전시명 기준)
│ 2026.05.16 — 06.02 · 3일간   │     첫 방문 ~ 마지막 활동
├──────────────────────────────┤
│ [페르소나 카드]                │  ② 취향 (화면4 재사용)
│ "Deep Ocean Dreamer"          │     최종 TASTE_PROFILE 반영
│ #고요함 #윤슬 #심오함          │     ※ 팝업 반응까지 누적된 최신값
├──────────────────────────────┤
│ 🕰 여정 타임라인                │  ③ 신규 — 통합 리포트의 핵심
│ ● 05.16 본전시 관람            │
│   └ '윤슬' 앞에서 12분         │
│ ● 05.16 AI와 8번의 대화        │
│ ● 05.24 어비스 티 라운지 방문   │
│   └ '딥블루 바' 존 체험        │
│ ● 05.24 심해 인센스 구매        │
├──────────────────────────────┤
│ 🎁 나의 소장품                 │  ④ 구매/획득물 (있을 때만)
│ · 심해 인센스                  │
├──────────────────────────────┤
│ 📊 이 전시에서의 나            │  ⑤ 감정 축 분포 (누적)
│ serene ████████░░ 42%         │     본전시+팝업 합산
│ dreamy ██████░░░░ 31%         │
├──────────────────────────────┤
│ [이미지로 저장] [공유]          │  ⑥ 공유 (화면4와 동일 정책)
└──────────────────────────────┘
```

### 섹션별 데이터 출처

| # | 섹션 | 출처 | 비고 |
|---|---|---|---|
| ② | 페르소나 카드 | `TASTE_PROFILE` (main) | SSOT v1.1: 팝업 반응도 main 프로필을 **갱신**하므로 최신값이 곧 통합값 |
| ③ | 여정 타임라인 | `SESSION`(main+spinoff), `ARTWORK_INTERACTION`(artwork+zone), `AI_CONVERSATION`, `PURCHASE` | **신규 조합** — 시간순 정렬만 하면 됨 |
| ④ | 소장품 | `PURCHASE` | 없으면 섹션 숨김 |
| ⑤ | 감정 축 분포 | `TASTE_PROFILE.axis_distribution` | 본전시+팝업 누적 (SSOT v1.1 §4 이어받기 원칙) |

**중요**: 새로운 데이터 모델이 거의 필요 없다. SSOT v1.1에서 이미 *"spinoff는 신규 프로필 생성 없이 main 프로필을 갱신"*, *"ARTWORK_INTERACTION.target_type = artwork | zone"*으로 설계해둔 덕분에, **통합 리포트는 기존 데이터를 시간순으로 읽기만 하면 된다.**

## 4. 데이터 계약 (SSOT 개정 — 최소)

```
TASTE_PROFILE
  + journey_completed_at: datetime?   🆕 마지막 활동 시각 (타임라인 종료점 표시용)
  + spinoff_visited_count: int        🆕 방문한 스핀오프 수 (카드 미리보기 뱃지용)
```

- 신규 엔티티 없음. 타임라인은 조회 시점에 **집계 쿼리로 조립**한다(비정규화 저장 불필요 — 데이터가 적고 조회 빈도가 낮음).
- `journey_completed_at`은 UX용(타임라인 마지막 점 표시)이며, 여정이 "끝났다"는 판정은 하지 않는다 — 팝업 재방문 시 계속 갱신.

### 신규 이벤트

| 이벤트 | 시점 | 용도 |
|---|---|---|
| `report.journey_viewed` | 통합 리포트 열람 | 사후 재방문 지표 (퍼널 외) |
| `report.shared` | 이미지 저장·공유 | 바이럴 측정 (v1.0에서 후보로만 남겨뒀던 것 — 여기서 확정) |

## 6. 공유 정책 (화면4 §3 계승 + 확장)

- 공유 단위는 여전히 **페르소나 카드 이미지**. 타임라인 전체를 이미지로 만들면 세로로 너무 길고, 구매 내역 등 사적 정보가 포함됨.
- 단, 통합 리포트에서 공유 시 카드에 **"팝업까지 다녀온 여정"** 뱃지가 추가된다(자연 유입 유도).
- 소장품(④)·구매 내역은 **공유 이미지에 절대 포함하지 않는다.**

## 7. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| 타임라인 (본전시+팝업 시간순) | AI 편집 관람 영상 (v3.0 여정 4단계) |
| 누적 감정 축 분포 | NFT·블록체인 아카이브 (v3.0 여정 5단계 언급분) |
| 소장품 목록 (구매 링크 클릭 기준) | 실제 결제 연동 소장품 검증 |
| 페르소나 카드 공유 + 여정 뱃지 | 타임라인 이미지 공유 |
| 스핀오프 다중 방문 누적 | 전시 간 통합(여러 전시를 아우르는 연간 리포트) |

> ⚠️ **소장품(④)의 한계**: MVP에서 구매는 외부 커머스 링크 클릭(`purchase.link_clicked`)까지만 추적한다. 실제 결제 완료 여부를 모르므로, 소장품 섹션은 **"관심 상품"에 가깝다.** 문구를 "나의 소장품"이 아니라 **"눈여겨본 것들"** 로 완화하거나, 현장 결제 연동 후 활성화할지 결정 필요(§8-1).

## 8. 남은 결정 포인트

1. **소장품 섹션 MVP 포함 여부** — 결제 완료를 모르는 상태에서 "소장품"이라 부르면 사실과 다를 수 있음. (제안: MVP에선 "눈여겨본 것들"로 표기하거나 섹션 자체를 OUT)
2. **타임라인 세밀도** — 작품별 개별 항목까지 나열하면 길어짐. (제안: 하이라이트 위주 5~7개 항목, "더보기"로 전체)
3. **여러 스핀오프 방문 시 순서** — 시간순 vs 매칭 rank순. (제안: 시간순 — 여정 서사가 목적이므로)
