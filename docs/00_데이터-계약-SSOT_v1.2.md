## 7. 🆕 계정 모델 (ACCOUNT) · 내비게이션 셸

### 7-1. 설계 원칙 — "게스트 우선, 전략적 전환"

- 온보딩(화면1)에서 **로그인을 강제하지 않는다** (30초 룰 유지). VISITOR는 기존과 동일하게 기기 기반으로 즉시 생성.
- 로그인은 **애착이 최고조인 시점**(화면4 리포트 완성 직후)에 전략적으로 유도. 그 외에도 언제든 상단 계정 아이콘으로 진입 가능.
- 로그인 사유(확정): ① 크로스디바이스 리포트/초대장 열람 ② 전시 종료 후 재접속·팝업/차기전시 알림 ③ 기획사 CRM·재방문 트래킹 ④ 일반적인 계정 체계 기대.
- 로그인 방식(확정): **카카오 소셜 로그인**(기본 권장, 원탭) + **이메일/비밀번호**(정식 계정) 병행 제공.

### 7-2. ACCOUNT 엔티티 · 병합(merge) 규칙

```
ACCOUNT (신규)
  account_id (PK)
  auth_provider        kakao | email
  email                (email 가입 시) / kakao_uid (카카오 시)
  nickname
  marketing_consent
  created_at

VISITOR
  visitor_id (PK)       기존과 동일 — 기기 기반, 게스트도 즉시 생성
  account_id (FK, nullable)   🆕 로그인/가입 시점에 연결됨
```

- **병합 규칙**: 로그인·가입 성공 시, 현재 기기의 `visitor_id`에 연결된 모든 SESSION·TASTE_PROFILE·SPINOFF_INVITE·PURCHASE 레코드를 그대로 두고 `VISITOR.account_id`만 채운다 (데이터 이관 아님 — 소급 연결). 이후 다른 기기에서 같은 계정 로그인 시, 해당 account_id에 연결된 모든 VISITOR(=기기별)의 데이터를 통합 조회.
- 한 계정에 여러 visitor_id(기기)가 연결될 수 있음 (1 ACCOUNT : N VISITOR).
- 게스트 상태로 종료해도 데이터는 소실되지 않음 — 추후 같은 기기 재방문 시 여전히 visitor_id로 이어짐. 단, 다른 기기에서는 로그인해야 이관됨.

### 7-3. 신규 이벤트

| 이벤트 | 발화 시점 |
|---|---|
| `account.signup` | 최초 가입 (provider 포함) |
| `account.login` | 기존 계정 로그인 |
| `account.visitor_merged` | 게스트 visitor_id → account_id 최초 연결 시 |

### 7-4. 내비게이션 셸 — 2모드

| 모드 | 적용 화면 | 구성 |
|---|---|---|
| **가이드 모드** | 화면1~6 (전시·팝업 체험 중) | 상단 바만: 뒤로가기 / 현재 컨텍스트명 / 계정 아이콘(항상 노출 — 탈출구). **하단 탭바 없음** (몰입 유지) |
| **계정 모드** | 화면7 마이페이지 및 전시 밖 진입 | 하단 탭바 4개: 홈(전시 찾기) · 마이리포트(히스토리) · 초대장 · 마이페이지 |

- 계정 아이콘(가이드 모드 상단 우측): 비로그인 시 로그인 유도 아이콘, 로그인 시 아바타 → 탭하면 마이페이지로 이동 (가이드 모드 이탈 없이 오버레이 권장).
- 어느 화면에서든 계정 아이콘 하나로 "마이페이지"까지 도달 가능해야 한다 — 이게 §의 "상하단 네비 부족" 해소의 핵심 규칙.
