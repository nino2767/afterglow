# 스핀오프 유저뷰 화면 플로우 v0.1 (Phase 1)

작성일: 2026.07.11 | 기반: 방향성재정의 v4.0 draft 7장
목업: `03_와이어프레임/20260711_스핀오프-유저뷰-목업-v0.1.html` (브라우저로 열기 — 화면별 폰 목업 + 논의 노트)

## 설계 원칙

1. **전환 퍼널이 아니라 하나의 전시 관람 경험** — 발견→입장→관람→기념→여운의 관람 서사로 설계
2. **본전시는 전제가 아니라 선택적 입력값** — 안 본 사람도 완결된 경험, 본 사람은 한 겹 더 (additive)
3. **제로 에포트** — 관람객이 뭔가를 '작성'하는 순간을 최소화. 사진·대화·머무름이 곧 데이터

## S-1. 서비스 홈 — 극장 입구 (직접 검색 유입)

- **대상**: "AFTERGLOW"를 직접 검색·주소 입력으로 접속한 사람 (특정 전시 링크가 아닌 브랜드 유입) + **고객사에게 플랫폼 구조를 보여주는 쇼케이스 역할**
- **구조**: 극장 문법 — **"현재 상영 중인 스핀오프"** 라인업 + Coming Soon → 선택하면 해당 전시의 S0 포스터로
- **Phase 1 구현 (라이트 라인업)**: 라인업 카드 수 장 수준의 가벼운 홈. 상영작은 2개로 채움:
  1. 실제 파트너 전시 스핀오프 1개
  2. **상설 데모 전시 1개** — v3.0의 '심해의 환상' 시뮬레이션을 실제 동작하는 스핀오프로 구현. 고객사 미팅에서 "직접 입장해보세요"가 최고의 세일즈 자료 + 빈 극장 문제 해소 + 장면 제작 파이프라인의 첫 테스트베드
- URL 구조: `/`(홈)와 `/전시slug`(개별 스핀오프) 분리
- **B2B 설명은 홈에 섞지 않음**: 모듈 구조·판매 조합·데이터 리포트 샘플은 별도 B2B 랜딩/제안 자료로 — 관람객 홈은 세계관 몰입 유지
- Phase 2+: 멀티 IP 라인업 확장 + 전시 검색 (MVP OUT시킨 '전시 검색 기능'이 여기서 부활)

## S0. 발견 — 디지털 포스터

- **진입 경로**: 본전시 퇴장 초대장(QR/링크) · SNS 공유 카드 · 미니 스핀오프 전시 입구 QR · 서비스 홈(S-1) 라인업 · (Phase 2+) 전시 정보 플랫폼
- **화면**: 서비스 소개 페이지가 아니라 **전시 포스터** — 세계관 키비주얼 + 스핀오프 전시 제목 + 한 줄 소개 + [입장하기]
- 포인트: S3의 공유 카드가 곧 이 포스터 → 공유가 신규 유입이 되는 바이럴 루프

## S1. 입장 — 온보딩 (세계관 진입 의식)

- 인트로 씬 5~10초 (스킵 가능)
- **분기 질문 단 1개**: "본전시 〈제목〉, 보셨나요?"
  - **아니오** → 감정/취향 키워드 2~3개 선택 (예: 고요함·기괴함·압도됨) → 기본 경험 (※ 본전시 소개/예매 링크 노출)
  - **예** → 기억에 남는 장면 1개 선택 (Phase 3에서 관람 데이터 자동 연동으로 대체)
- 데이터 수집 동의 (라이트하게, 필수)
- **로그인 없음** — 익명 세션. 연락처는 S3 아카이브 저장 시점에만 1개 요청 (여운 리마인드용)

## S2. 관람 — 씬 뷰어 (핵심 화면)

- 세계관 신규 장면 3~5개를 세로 스와이프로 관람 (장면 = 비주얼 + 사운드 + 짧은 내러티브)
- 장면마다 **AI 세계관 안내자** 대화 진입 — 풀스크린 채팅이 아니라 장면 위 오버레이 (몰입 유지)
- 입장 시 선택한 키워드가 장면 연출에 변주로 반영 (라이트 개인화)
- **오프라인 병행 모드**: 미니 스핀오프 전시 현장에서 작품 사진 촬영 → 인식 → 해당 장면 열림. 온·오프가 동일한 씬 구조를 공유

## S3. 기념 — 나의 아카이브

- 관람 궤적 **자동** 생성: 머문 장면 · 대화 발췌 · 감정 키워드 비주얼 (버튼 하나, 작성 없음)
- SNS 공유 카드 → 발산에서 '핵심'으로 지목한 "사진이 리뷰가 된다"의 구현 지점
- **출구 굿즈**: 내 아카이브 기반 커스텀 굿즈 1~2종 (마플 POD 연계 후보) · 외부 결제 링크
- **본전시 연계 CTA (조건부)**: S1에서 '아니요, 처음이에요'(본전시 미관람)를 선택한 유저에게만 하단에 **[본전시 〈심해의 환상〉 예매/소개 보기 ↗]** 버튼을 노출하여 예매 전환 유도. (관람 완료 고객에게는 미노출)

## S4. 여운 — 이어지는 대화

- D+1, D+3 리마인드 (채널 미정: 알림톡 vs 이메일)
- 재접속 시: 세계관 대화 계속 + **새 장면 1개 추가 공개** (재방문 유인 장치)
- 아카이브는 대화할수록 자라남

## 구현 우선순위 (화면 단위)

**S2 (씬 뷰어+대화) > S3 (아카이브) > S1 (온보딩) > S4 (리마인드) > S0 (포스터)**
— S2·S3가 본질 검증의 몸체. S0은 정적 페이지로 최소 구현 가능

## 미결정 사항

- 리마인드 채널 (알림톡 비용 vs 이메일 도달률)
- 세계관 장면 제작 파이프라인: AI 생성 vs 수동 큐레이션 비율 (어드민 Phase 2와 연결)
- 익명 세션 식별 방식 (브라우저 기반 한계 — 재접속 측정 정확도에 영향)
