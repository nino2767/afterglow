## 1. 개요 및 목적 (Overview & Goals)

### 1.1. 서비스 정의
**AFTERGLOW 어드민 플랫폼**은 전시 기획사, 갤러리, 작가 등 B2B 고객사(운영자)와 플랫폼 슈퍼 관리자(AFTERGLOW 마스터)를 위한 통합 백오피스 솔루션이다. 

원천 전시 IP(작품, 작가 노트, 오디오/세계관)를 업로드하면 AI 파이프라인을 통해 독립된 **'두 번째 작품(스핀오프 전시)'**의 씬 스크립트, AI 안내자(페르소나) 대화, 추천 굿즈 및 연출 프리셋을 자동으로 기획·생성·운영하고, 전계정의 라이선스 계약 및 정산 수수료를 통합 관리한다.

### 1.2. 📚 핵심 용어 정리 표 (Terminology Glossary)

| 용어 (Term) | 약자/원어 | 정의 및 AFTERGLOW 플랫폼 내 역할 |
|---|---|---|
| **RAG** | **Retrieval-Augmented Generation**<br>(검색 증강 생성) | AI의 엉뚱한 거짓말(환각/Hallucination)을 방지하기 위해, 기획자가 입력한 작가의 원천 세계관 장문 텍스트(`worldview_context`)를 실시간으로 탐색·학습하여 작가의 창작 의도와 100% 일치하는 대화 및 씬 스크립트를 생성하는 **핵심 AI 기술 엔진**. |
| **본전시 원천 IP** | Main Exhibition IP | 기획사/작가가 오프라인 갤러리나 미술관에서 운영하는 **오리지널 본전시 지식재산권**. 스핀오프 전시의 기초 원천 데이터 자산이 됨. |
| **스핀오프 전시** | Spin-Off Exhibition | 본전시와 세계관/캐릭터를 공유하되, 모바일 웹 및 미니 부스에서 관람객과 1:1 대화를 나누는 **독립된 디지털 2차 작품 전시**. |
| **Concept Bot** | AI Concept Pipeline | 작가의 세계관 텍스트만 등록하면 AI가 씬 1~3 스토리라인과 AI 안내자 "레아" 페르소나 대화 규칙을 30초 이내 0번째 초안으로 자동 기획해 주는 **AI 자동 생성 파이프라인**. |
| **AI 안내자 "레아"** | AI Persona Guide | 스핀오프 전시 씬 1~3에서 관람객과 1:1로 실시간 대화를 나누며 관람 몰입 및 티켓 예매를 유도하는 **AI 페르소나 캐릭터**. |
| **독립 유입군** | Independent New Visitors | 본전시를 관람하지 않고 스핀오프 전시에만 독자적으로 진입한 **신규 타깃 관람객 (목표 40%+)**. 본전시 티켓 예매처 역전환 유도 대상. |
| **본전시 연계군** | Linked Visitors | 본전시를 관람한 후 스핀오프 전시에 유입된 **기존 관람객 (52%)**. 소장용 씬 카드 발급 및 커머스 굿즈 구매 유도 대상. |
| **1-Click Cloning** | Version Cloning | 기존 스핀오프 회차(`v1.0`)의 씬 스토리와 페르소나 설정을 한 번의 클릭으로 그대로 복사하여 차기 팝업/전시(`v2.0`)로 빠르게 재활용하는 **자산 재활용 기능**. |
| **RS (수수료 배분)** | Revenue Share | 스핀오프 연계 커머스 굿즈 판매 매출에서 플랫폼과 B2B 기획사가 계약된 비율(예: 20%)로 수익을 분배하는 **수익 공유 정산 모델**. |
| **RBAC** | **Role-Based Access Control**<br>(역할 기반 권한 통제) | 사용자 계정의 역할(`SUPER_ADMIN`, `CLIENT_ADMIN`, `OPERATOR`, `VIEWER`)에 따라 데이터 접근 및 조작 권한을 차등 제어하는 **보안 시스템**. |

### 1.3. 핵심 전환 배경 (v4.0 방향성)
* **기존 (v3.0):** 본전시의 연장 장치이자 팝업 굿즈 판매점 연결 목적의 서브 플랫폼.
* **재정의 (v4.0):** **본전시와 세계관을 공유하는 독립된 두 번째 작품 (스핀오프)**.
* **핵심 가치:** 
  1. 기획 대행사 없이 어드민 업로드만으로 1시간 내 스핀오프 웹 전시/미니 오프라인 부스용 콘텐츠 생성.
  2. Phase 1 단계에서는 관람객의 본전시 데이터 없이도 **본전시 미관람자(독립 유입군 40%+)가 독자적으로 유입되어 몰입**할 수 있는 완성도 높은 독립 스핀오프 경험 제공.
  3. 모든 IP 자산, 작가 세계관 장문(`worldview_context`), AI 씬 스토리, 큐레이터 톤 설정이 **디지털 자산(Assetization)**으로 박제되어 차기 전시나 2차 팝업 제작 시 1-Click 복사(Cloning) 재활용 지원.

## 2. 양대 어드민 분리 & 1:N:N 계층 아키텍처 (Architecture)

AFTERGLOW 어드민은 **운영자 B2B 어드민**과 **고객사 관리 슈퍼 어드민** 2개의 독립 도메인/빌드 모드로 동작하며, 데이터는 `Client(고객사) 1 ──< Main Exhibition IP(본전시 IP) N ──< Spinoff Version(스핀오프 회차/버전) N` 의 1:N:N 구조로 연결된다.

```
                      ┌─────────────────────────────────────────┐
                      │    AFTERGLOW ADMIN PLATFORM (Vite App)  │
                      └────────────────────┬────────────────────┘
                                           │
             ┌─────────────────────────────┴─────────────────────────────┐
             ▼                                                           ▼
┌──────────────────────────────┐                           ┌──────────────────────────────┐
│  1. 운영자 B2B 어드민          │                           │  2. 고객사 관리 슈퍼 어드민    │
│  (Operator Admin)            │                           │  (Super Admin)               │
├──────────────────────────────┤                           ├──────────────────────────────┤
│ • 도메인: afterglow-admin    │                           │ • 도메인: afterglow-super    │
│ • 환경변수: MODE=operator     │                           │ • 환경변수: MODE=super        │
│ • 대상: 고객사 전시 기획자     │                           │ • 대상: AFTERGLOW 마스터      │
│ • 기능: IP 관리, 스핀오프 관리,│                           │ • 기능: 전 고객사 라이선스,   │
│   관람 모니터, 성과 리포트   │                           │   운영 어드민 실시간 점검 뷰 │
└──────────────────────────────┘                           └──────────────────────────────┘
```

## 3. 4단계 이용자 권한 (RBAC: Role-Based Access Control) 스펙

| 역할 (Role) | 타깃 사용자 | 데이터 접근 스코프 | 주요 권한 범위 | UI 제어 규칙 |
|---|---|---|---|---|
| **1. `SUPER_ADMIN`** | AFTERGLOW 마스터 | 전계정 통합 (`all clients`) | 전 파트너사 계정 승인/정지, RS 수수료율 설정, 전 고객사 운영 어드민 뷰 실시간 스위칭 | 슈퍼 어드민 대시보드 전체 접근 |
| **2. `CLIENT_ADMIN`** | 고객사 총괄 책임자 (예: 피플리 김민준 대표) | 자사 `client_id` 전체 | 자사 계정 프로필 관리, 자사 운영자 계정 초대, 본전시 IP 생성 및 스핀오프 최종 퍼블리싱 승인 | 운영자 어드민의 모든 기능 읽기/쓰기 가능 |
| **3. `OPERATOR`** | 기획사 담당자 / 현장 운영자 | 자사 지정 전시 IP & 스핀오프 | IP 자산(`worldview_context`) 편집, 스핀오프 씬 1~3 스토리 & 페르소나 튜닝, 모니터링 | 자사 지정 전시의 실무 작업 가능 |
| **4. `VIEWER`** | 파트너 스태프 / 외부 검수자 | 자사 지정 전시 IP | 관람 모니터 및 성과 리포트 열람 전용 | IP 및 스핀오프 수정/저장 버튼 비활성화 (Read-Only) |

## 4. 로그인 & B2B 파트너 입점 온보딩 (Authentication & Onboarding)

### 4.1. 로그인 프로세스 (Login.jsx)
* 이메일/비밀번호 기반 로그인 지원.
* **원클릭 데모 접속:** 4단계 RBAC 역할(`SUPER_ADMIN`, `CLIENT_ADMIN`, `OPERATOR`, `VIEWER`)별 사전 정의된 데모 계정으로 원클릭 스위칭 테스트 가능.

### 4.2. B2B 파트너 입점 가입 신청 (Signup.jsx)
* **신청 플로우:** 기획사명, 담당자 연락처, 이메일, 입점 희망 본전시 정보를 입력하여 제출 (`status: pending`).
* **슈퍼 관리자 승인 플로우:** 슈퍼 관리자가 `ClientManagement.jsx` 승인 큐에서 신청서를 확인 후 플랜 및 RS 수수료율을 설정하고 **`[가입 승인]`** 시 활성화 및 비밀번호 자동 발급.

---

### 🅰️ [어드민 1] 운영자 B2B 어드민 메뉴 스펙 (6개 모듈)

#### 📁 1. 대시보드 (Dashboard)
* **주요 요소:** 
  - **[구역 1] 범위별 4대 KPI 성과 카드:** 조회 범위 스위처(`viewScope`: 자사 전체 합산 vs 현재 선택 전시)
  - **[구역 2] 스핀오프 전시 관리 카탈로그 카드:** 진행 중인 전시(`active`, 기본 표출) vs 종료/아카이브(`archived`) 명시적 필터 탭 및 `[IP 자산 관리]`, `[씬/페르소나 튜닝]` 직관적 이동 버튼
  - **[구역 3] 실시간 관람객 몰입 감정 요약:** 관람객 주요 감정 코드(Calm 62%), 평균 대화 턴 수(6.3턴), 역전환 클릭 요약 퀵 프리뷰

#### 📁 2. IP 관리 (IP Management) [명칭 변경]
* **주요 요소:**
  - **본전시 IP 카탈로그:** 자사 본전시 IP 목록 관리 (활성/종료 아카이브 필터)
  - **RAG 작가 세계관 장문 (`worldview_context`):** AI 안내자 학습용 작가 창작 의도/비하인드 텍스트 에디터
  - **핵심 작품 갤러리 (`artworks`):** 작품별 비전 reference 이미지, 오디오 도슨트 스크립트 자산 라이브러리 저장

#### 📁 3. 스핀오프 관리 (Spinoff Management) [명칭 변경]
* **주요 요소:**
  - **회차/버전 관리 & Cloning:** 전시당 1:N 스핀오프 버전 생성 및 기존 버전 1-Click 복사 재활용
  - **RAG 4단계 파이프라인:** Draft ➔ Generating ➔ Review ➔ Published 상태 제어
  - **씬 & 페르소나 에디터:** Scene 1~3 스토리라인, Ambient 사운드/컬러 연출, AI 안내자 "레아" 톤앤매너 튜닝
  - **모바일 시뮬레이터:** 모바일 화면을 통한 씬 스토리 감상 및 1:1 AI 대화 실시간 미리보기

#### 📁 4. 관람 모니터 (Monitor)
* **주요 요소:** 유입군 이원화 토글 (전체 / 독립 유입군 48% / 본전시 연계군 52%), 씬 1~3 잔존율/체류시간 히트맵, 실시간 AI 대화 피드.

#### 📁 5. 성과 리포트 (Report)
* **주요 요소:** v4.0 4대 검증 KPI 지표 카드, 여정 퍼널 분석, B2B 기획사 의사결정 권고안 PDF/CSV 출력.

#### 📁 6. 정산 관리 (Settlement)
* **주요 요소:** 자사 개별 전시의 B2B Revenue Share 정산 내역 조회.

* **주요 요소:** 전 파트너사의 라이브 전시 및 스핀오프 퍼블리싱 상태 총괄 모니터링.

#### 📁 4. 시스템 설정 (System Configuration)
* **주요 요소:** 플랫폼 공통 AI 파라미터 및 마스터 계정 관리.
