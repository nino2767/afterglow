# AFTERGLOW 방향성 재정의 & 상세기획 메모 (v4.0 draft)

작성일: 2026.07.11 | 트랙: A (퀵 실험형) | 기반: v3.0 문서 재검토

## 1. 재정의된 본질 (상위기획 결과)

**한 줄 정의**

> AFTERGLOW는 전시가 끝난 후에도 그 세계관에 계속 머물 수 있게, 전시 IP를 '두 번째 작품(스핀오프)'으로 만들어 이어주는 서비스다.

**핵심 전환**: "본전시의 연장 장치" → **"세계관을 공유하는 독립된 두 번째 작품"**

**핵심가치 키워드**: 세계관의 연장 · 여운의 경험화(정보가 아닌) · 관람 전 구간의 동반

**원점 (동기)**
- 영화/예능의 '스핀오프' 개념을 전시로 — 본편이 끝나도 세계관이 이어진다
- 개인 경험: 전시 후 작가·작품을 검색해도 정보 파편만 있고 허전했음 (이해+감동+연결 모두 미해소)
- 시장 공백(⚠️ 재검증 필요): "전시/미술은 IT 낙후"라는 초기 인상은 수정 필요. 관람 중 가이드 영역은 이미 경쟁 존재 — 오프라인 도슨트 보편화, 국립중앙박물관은 IoT 비콘 기반 위치 자동안내(5개 언어)에 이어 2025.9 AI 검색·지능형 큐레이션 플랫폼 공개, 네이버는 바이브(VIBE) 오디오 탭에서 전시 도슨트 콘텐츠 운영(NCT 127 도슨트, 마티스전 등 — 전시 종료 후에도 콘텐츠 제공), 오디오클립은 2025.12 서비스 종료 확정. 다만 이들은 모두 **관람 중** 서비스이며, **관람 후 세계관 연장(스핀오프)** 영역은 여전히 공백 → AFTERGLOW의 공백 정의를 '전시 IT 전반'이 아니라 '관람 후 구간'으로 좁혀야 함. 상세는 별도 문서 **AFTERGLOW_시장조사리포트_v1.0.md** 참조

**핵심 동작**: 전시 IP(작품, 작가 세계관, 관람객의 감정 데이터)를 재료로, AI가 스핀오프 경험 — 이어지는 대화, 개인 아카이브, 커스텀 굿즈, 팝업 — 을 만들어낸다. **관람객의 사진·대화가 그 사람만의 스핀오프 입장권이 된다.**

**B2B의 위상**: 본질이 아니라 전달 경로. 세계관의 주인이 기획사이므로 그들을 통해 관람객에게 닿는다.

| 지표 "팝업 전환율 20%" | ❌ 폐기 | '연장 장치' 관점 지표 → **잔존(머무름) 지표**로 교체 |

| 타깃 파트너 (피플리, 세라핌컴퍼니) | ✅ 유지 | 세계관 주인 확보는 전제조건. 접근 전략(중소→대형) 유지 |

## 3. MVP 가설

> **"전시가 끝난 후에도 관람객은 그 세계관에 머물고 싶어하며, 스핀오프 경험(이어지는 대화 + 개인 아카이브 + 미니 오프라인 체험)이 '검색하고 허전한' 경험을 대체할 수 있다."**

### 검증 플로우

1. **관람 중**: **작품 사진 촬영이 기본 진입점** — 작품을 찍으면 AI가 사진을 인식해 설명 제공 + AI 큐레이터 대화 시작 (사진·감정 키워드가 개인 데이터로 축적). QR은 보조 진입 수단(선택) — QR이 없는 전시장도 많음. 촬영 불가 전시는 작품 리스트에서 직접 선택하는 폴백 구조
2. **퇴장 시점**: 사진+대화가 자동으로 **나만의 아카이브(리뷰 초안)** 로 생성 → **두 경로를 독립적으로 초대** (① 온라인 스핀오프 ② 미니 스핀오프 전시) — 각각 분리 측정
3. **[오프라인 경로] 미니 스핀오프 전시 (전시장 내/인근 1개 공간)**: 굿즈숍이 아니라 **'또 하나의 작은 전시'를 보는 경험** — 본전시 세계관을 가져와 다른 이야기·연출로 구성한 새 장면을 관람. 예능 스핀오프 문법을 따름 (아는형님→방과후교실, 나혼자산다→여은파: 원작의 세계관·인물로 **다른 판**을 벌이는 독립 콘텐츠). 내 관람 데이터는 연출에 반영 (예: 내 감정 키워드가 공간의 빛·사운드·장면으로). **굿즈는 관람 동선 출구의 기념품 — 본체가 아님**
4. **[온라인 경로] 관람 후 (D+1~7)**: 웹앱 재접속 → 세계관 대화 계속, 아카이브 확인·공유

**온라인/오프라인 분리 검증 설계**: 두 경로는 서로 독립 — 부스를 안 가도 온라인 스핀오프는 온전히 즐길 수 있고, 그 역도 성립. 결과 조합별 판정:

| 온라인 재접속 | 부스 방문 | 해석 → 다음 방향 |
|---|---|---|
| ↑ | ↑ | 풀 스핀오프 검증 → 정식 팝업 확장 + 디지털 병행 |
| ↑ | ↓ | 머무름 욕구는 디지털로 충족 → 디지털 스핀오프 중심, 팝업은 선택 상품 |
| ↓ | ↑ | 오프라인 체험이 본체 → 팝업 중심(v3.0 원형에 근접), 웹은 브릿지로 축소 |
| ↓ | ↓ | 본질 가설 재검토 (상위기획 복귀) |

### 성공 기준 (숫자)

| 지표 | 기준 | 의미 |
|---|---|---|
| **[핵심] 관람 후 48시간 내 재접속률** | **30% 이상** (파일럿 30명 중 9명) | 세계관에 머물고 싶은가 |
| 재접속 시 대화 지속 | 재접속자의 50% 이상이 3턴 이상 대화 | 형식적 방문이 아닌 실질 머무름 |
| 미니 스핀오프 전시 방문율 | AI 큐레이터 사용자 중 20% 이상 | 오프라인 스핀오프로의 확장 의지 |
| 아카이브 공유(SNS 등) | 10% 이상 | 리뷰화 가설 검증 |

### 기능 범위 IN / OUT

**IN**
- 웹앱형 AI 큐레이터 (핵심 작품 3~5개 대화 해설) — v3.0 유지, 단 진입은 **사진 인식 기반**(QR은 보조, 촬영 불가 시 작품 선택 폴백)
- 사진 촬영 → 자동 아카이브/리뷰 초안 생성 — 신규
- **관람 후에도 이어지는 세계관 대화** (작가·작품 심층 대화) — 신규, 본질 직결
- 미니 스핀오프 전시 1개 공간 (세계관 기반 새 장면 관람 + 출구 굿즈 1~2종)
- 상설 데모 전시 1개 (v3.0 '심해의 환상' 시뮬레이션의 실구현) — 홈 라인업 채움 + 고객사 세일즈 데모 + 장면 제작 파이프라인 테스트베드
- 내부용 AI 기획 어시스턴트 (부스 컨셉·굿즈 생성용, 노출 안 함)

**OUT**
- 정식 팝업 공간 운영 대행 (21일 연장 운영 등) → 검증 후
- 네이티브 앱, 자체 결제, 실시간 도면 생성, 다국어 — v3.0 OUT 유지
- 전시 검색 기능
- 기획사용 대시보드 (파일럿에선 수동 리포트로 대체)

## 4. 비즈니스 모델 (가볍게)

- MVP: 파트너 기획사 1곳 **무료 파일럿** (IP 제공 ↔ 데이터 리포트 무상 제공)
- 검증 후: v3.0의 RS 모델 재검토 — 단, 수익원 우선순위가 바뀔 수 있음 (팝업 매출 중심 → 관람객 구독/굿즈 + 기획사 리포트 구독 가능성). 재접속 데이터를 보고 결정
- 부스 굿즈는 현장 결제/외부 링크 (v3.0 OUT 기준 유지)

## 5. 치명적 리스크 (트랙 A — 이것만)

1. **IP 2차 활용 라이선스**: 작품 이미지·세계관을 AI가 재가공·개인화하는 것에 대한 기획사/작가의 명시적 동의 필수. 파일럿 협약서에 재가공 범위 명시
2. **개인정보 (사진·감정 데이터)**: 온보딩 시 수집·활용 동의 플로우 필수. 관람객 사진에 타인이 찍히는 경우의 처리 기준 필요

## 6. 파일럿 타임라인 (7주, v3.0 골격 유지)

- 1~2주: 파트너 1곳 확정, IP 자산 확보, 라이선스 협약
- 3~4주: AI 큐레이터 + 아카이브 웹앱 구축, 미니 스핀오프 전시 기획 (내부 AI 어시스턴트 활용)
- 5~6주: 현장 운영 (QR 배치, 부스 오픈), 데이터 수집
- 7주: 지표 분석 → 성공 기준 대비 판정 → 다음 단계 결정

### 판정 후 분기

- **재접속률 ≥ 30%**: 본질 검증 성공 → 오프라인 스핀오프 확장(정식 팝업) + B2B 모델 구체화
- **10~30%**: 상세기획 내 조정 (아카이브 UX, 대화 품질, 리마인드 방식 개선 후 재시도)
- **< 10%**: 상위기획으로 복귀 — "관람 후에도 머물고 싶다"는 전제 자체 재검토

## 7. 구현 우선순위 & 시스템 모듈 구조 (2026.07.11 추가)

### 구현 순서

| 순위 | 대상 | 내용 |
|---|---|---|
| **1** | 스핀오프 전시 유저뷰 | AFTERGLOW의 메인 서비스. IP 세계관만으로 성립하는 독립 경험 |
| **2** | 스핀오프 운영 어드민 | 고객사가 본전시 IP 자산을 업로드 → AI가 스핀오프 콘텐츠 생성·운영 (기획 모듈 엔진 편입) |
| **3** | 본전시 유저뷰 | AI 큐레이터, 사진→아카이브. 어드민은 공통 코어라 약간의 고도화로 충분 |

### 핵심 설계 원칙: Phase 1 스핀오프는 본전시 데이터 없이 성립해야 한다

- 1순위 시점에는 관람객의 본전시 데이터(사진·대화)가 존재하지 않음
- 따라서 Phase 1 스핀오프 = **본전시를 안 본 사람도 즐길 수 있는 IP 세계관 경험** (진짜 스핀오프의 정의와 일치)
- 개인화(관람 데이터 연동)는 Phase 3에서 본전시 모듈이 붙을 때 켜지는 **프리미엄 기능**
- 지표 조정: Phase 1은 "관람 후 재접속률" 대신 **스핀오프 자체의 체류·재방문·공유**로 측정. "관람 후 48시간 재접속 30%"는 Phase 3(A+B 결합) 시점의 지표

### 시스템 모듈 구조

```
[공통 코어]
  IP 세계관 엔진: 자산 업로드 → 세계관 DB → AI 대화/콘텐츠 생성
  관람객 프로필·아카이브
  고객사·전시 관리 어드민
        │
   ┌────┴─────────────────┐
[모듈 A] 본전시            [모듈 B] 스핀오프 (메인)
  AI 큐레이터 유저뷰         세계관 경험 유저뷰
  사진→아카이브              이어지는 대화, 미니 스핀오프 전시, 출구 굿즈
  (Phase 3)                 (Phase 1~2)
```

### 판매 조합 (B2B 상품 구조)

| 조합 | 제공 가치 | 비고 |
|---|---|---|
| A만 (본전시) | AI 큐레이터 가이드 고도화 | 기존 오디오가이드 대체 시장 |
| **B만 (스핀오프)** | 전시 IP의 두 번째 작품화 | **메인 상품** — 종료된 전시 IP에도 판매 가능 |
| A+B | 데이터 루프 완성 — 관람 데이터 기반 초개인화 스핀오프 | 최상위 상품. v3.0의 개인화 시나리오가 여기서 부활 |
