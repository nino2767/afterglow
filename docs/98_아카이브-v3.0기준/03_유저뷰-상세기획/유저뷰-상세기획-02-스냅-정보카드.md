# 유저뷰 상세기획 — 화면 2: 스냅 & 정보 카드 (Level 1)

> 버전: v1.0 | 작성일: 2026.07.02 | 상태: 📋 초안
> 근거: v3.0 §2(AI 큐레이터: QR 태깅 작품 해설)·§9(MVP IN: 핵심 작품 3~5개 AI 대화 해설 / OUT: Vision-to-Text는 v2), SSOT v1.1 §2·§6, 화면3 상세기획(Level 2 진입점)
> 위치: 유저뷰 6화면 중 2번. **관람 중 홈 화면** — 작품 인식 → 즉시 해설(Level 1) → 대화(Level 2, 화면3) 진입의 허브. `dwell_sec` 등 행동 데이터의 주 수집 지점.

---

## 1. 화면의 역할

| 관점 | 역할 |
|---|---|
| 관람객(B2C) | 작품 앞에서 QR 한 번 → 3초 안에 핵심 해설. 더 궁금하면 대화로 |
| 데이터 | ARTWORK_INTERACTION 축적 (view, dwell_sec, 감정 퀵 반응) — 화면4 하이라이트·Monitor 히트맵의 원천 |
| 스토리 모듈 | Level 1 해설 = KB docent_script의 카드 버전. 세계관 톤 유지 |
| 어드민(B2B) | 섹션별 체류·인기 지수(Monitor Phase 3)의 데이터 공급원 |

**설계 원칙 — "Level 1은 읽는 것, Level 2는 나누는 것":** 정보 카드는 스크롤 없이 한 화면, 15초 안에 소화. 깊이는 전부 화면3(대화)으로 위임. 두 레벨의 역할 분리가 흐려지면 카드가 길어지고 대화 진입률이 죽는다.

---

## 2. 진입 · 이탈 플로우

```
[화면1 온보딩 완료] → 화면2 (대기 상태: 작품 목록/안내)
   ├─ 작품 QR 스캔 → 정보 카드 표시 (interaction 기록 시작)
   │    ├─ 감정 퀵 반응 탭 (선택)
   │    ├─ [AI 큐레이터와 대화하기] → 화면3 (Level 2)
   │    └─ 다른 작품 QR 스캔 → 카드 교체 (이전 작품 dwell 확정)
   ├─ 작품 목록에서 탭 → 동일 카드 (viewed_via: list)
   └─ 하단 고정 [관람 완료] → 화면4
```

- **dwell_sec 산정**: 카드 오픈 ~ 다음 작품 스캔/화면 이탈. 백그라운드 전환 시 일시정지 (부풀림 방지).

## 3. 화면 구성 (모바일)

```
[대기 상태]                      [정보 카드 상태]
┌──────────────────────┐      ┌──────────────────────┐
│ 빛의 심연 · {nickname}님│      │ [작품 이미지]           │
│                       │      │ 윤슬 — 김OO            │
│  📷 [작품 QR 스캔]      │      │ ─────────────────    │
│                       │      │ 핵심 해설 3~4문장       │
│  또는 작품 골라보기      │      │ (docent_script 요약)   │
│  ┌────┐┌────┐┌────┐  │      │ ─────────────────    │
│  │작품1││작품2││작품3│  │      │ 지금 어떤 느낌이세요?    │
│  └────┘└────┘└────┘  │      │ [고요함][아련함][+]     │ ← 감정 퀵 반응
│  (핵심 작품 3~5, MVP)  │      │ ─────────────────    │
│                       │      │ 💬 [AI 큐레이터와       │
│ [관람 완료]            │      │     대화하기]           │ ← 화면3 진입
└──────────────────────┘      │ [관람 완료]             │
                               └──────────────────────┘
```

### 구성 요소 상세

| 요소 | 동작 | 데이터 |
|---|---|---|
| QR 스캔 | 카메라 → 작품 QR 인식 → 카드 즉시 표시 (3초 룰) | 쓰기: ARTWORK_INTERACTION(view, viewed_via: qr) |
| 작품 목록 | KB 등록 핵심 작품 3~5개 썸네일 (v3.0 §9). QR 없이도 접근 가능한 보조 경로 | viewed_via: list |
| 정보 카드 | 이미지+제목+작가+해설 3~4문장. 스크롤 없는 1카드 | 읽기: AI_KNOWLEDGE_BASE |
| 감정 퀵 반응 | 카드 하단 leaf 칩 2~3개(작품별 default_emotion_chips) + [+]로 전체 leaf. 탭 = 즉시 기록, 대화 없이도 감정 수집 | 쓰기: `{emotion, axis}` → ARTWORK_INTERACTION.auto_tags |
| 대화 CTA | 화면3 진입. 카드에서 가장 눈에 띄는 액션 (Level 2 전환이 핵심 지표 — MVP "질문 3회 지속" 검증의 입구) | — |
| 사진 촬영 | 카드 내 촬영 버튼 → photo_url 저장 (화면4 리포트 소재) | 쓰기: photo_url |

## 4. 상태 · 엣지 케이스

| 상태 | 처리 |
|---|---|
| KB 미등록 작품 QR | "이 작품은 준비 중이에요" + 등록 작품 목록 유도 (MVP 3~5개 한계 안내) |
| 카메라 권한 거부 | 작품 목록(list 경로)으로 자동 대체 — QR 없이도 전 기능 사용 가능 |
| 전시장 음영(오프라인) | 카드 콘텐츠 최초 로드 시 전체 캐시 → 오프라인에도 카드 열람 가능, interaction은 큐잉 후 재전송 |
| 연속 스캔 남용(동일 작품 반복) | 동일 작품 재스캔은 view 미중복, dwell만 누적 |
| 세션 없음(QR 직접 진입) | 화면1 온보딩으로 리다이렉트 (작품 QR에 exhibition_id 포함 → 온보딩 후 해당 작품 카드로 복귀) |

## 5. MVP IN / OUT

| IN | OUT (v2+) |
|---|---|
| 작품 QR 스캔 + 목록 보조 경로 | Vision-to-Text 사진 인식 (v3.0 §5 명시 기능이나 MVP OUT — 화면3 §7과 일치) |
| 핵심 작품 3~5 카드 | 전 작품 커버 |
| 감정 퀵 반응 + 사진 저장 | 위치 기반 자동 해설 (큐피커 벤치마킹 기능) |
| 오프라인 카드 캐시 | 오디오 해설(TTS) |

## 6. 남은 결정 포인트

1. **감정 퀵 반응과 화면3 감정 칩의 관계** — 동일 데이터 스키마(SSOT §2)지만 수집 맥락이 다름(카드=즉흥, 대화=심화). 세그먼트 산출 시 가중치 차등 둘지. (현안: v1은 동일 가중, 파일럿 후 튜닝)
2. **작품 목록 노출 순서** — 관람 동선 순 vs 인기순. (현안: 운영자 Phase 1 지정 순서 — 동선은 전시마다 달라 운영자가 앎)
