# 유저 테스트 & 피드백 수정사항 이슈 트래커

본 문서는 지라(Jira) 등 별도의 이슈 관리 툴을 사용하지 않는 환경에서, **B2C 유저뷰 및 B2B 어드민 사용자 테스트(QA)** 중 발견된 수정사항, 버그, 개선 의견을 통합 관리하고 추적하기 위한 문서입니다.

---

## 🚦 상태 정의 및 필터링 가이드
- **우선순위 (Priority)**:
  - `P0 (Critical)`: 핵심 프로세스 불가 (예: 빌드 오류, QR 인식 불가, 회원가입 루프 등)
  - `P1 (High)`: 주요 기능 및 기획 사양 누락 또는 핵심 화면 레이아웃 깨짐
  - `P2 (Normal)`: 일반 기능 버그, 텍스트 오타, 단순 미관 개선
  - `P3 (Low)`: 마이너한 애니메이션 최적화 및 장기 보류 개선안
- **상태 (Status)**:
  - `📋 대기 (Todo)`: 신규 등록되어 검토 대기 중인 이슈
  - `⚙️ 진행 중 (In Progress)`: 담당 개발자가 배정되어 수정 중인 이슈
  - `✅ 완료 (Done)`: 수정 완료되어 검증 및 실주소 배포까지 완료된 이슈
  - `🛑 보류 (Hold)`: 기획 재논의 또는 외부 요인(데이터 레이어 동결 등)으로 보류된 이슈

## 🛠️ 이슈 신규 등록 템플릿
테스터분들은 문제를 등록하실 때 아래 템플릿을 복사하여 작성해 주시면 개발자가 원인을 빠르게 분석하는 데 큰 도움이 됩니다.

```markdown
### [이슈 #ID] 이슈 제목
- **발견 화면**: (예: 유저뷰 / report 페이지)
- **우선순위**: (P0 / P1 / P2 / P3)
- **현상 상세**:
  - 어떤 행동을 했을 때 어떤 버그나 디자인 어긋남이 발생하는지 적어주세요.
- **재현 경로**:
  1. `https://afterglow-user.vercel.app/home` 진입
  2. ...
  3. ...
- **희망 수정안**:
  - (예: 버튼 클릭 범위 확장, 컬러 테두리 변경 등)
- **스크린샷/첨부**: (있을 경우 이미지 경로 또는 Notion 이미지 링크 업로드)
```
