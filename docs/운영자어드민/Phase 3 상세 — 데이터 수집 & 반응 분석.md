# Phase 3 상세 — 데이터 수집 & 반응 분석

> 전시 오픈 후 B2C 관람객의 행동이 실시간으로 유입되는 단계입니다. 운영자는 이 화면을 보며 전시 반응을 파악하고, 이 데이터가 Phase 4 스핀오프 기획의 근거가 됩니다.

---

## 1. Phase 목표

관람 중 생성되는 행동 데이터를 실시간으로 수집·시각화하여 **"어떤 작품에서 어떤 감정이 얼마나 발생했는가"**를 운영자가 파악할 수 있게 합니다. 이 데이터는 Phase 4 컨셉 카드의 유일한 근거가 됩니다.

---

## 2. 수집 데이터 항목

| 데이터 항목 | 수집 방식 | 저장 엔티티 | 활용처 |
| --- | --- | --- | --- |
| **QR 스캔 타임라인** | QR 접속 시 자동 기록 | SESSION | 입장 시간대·동선 분석 |
| **작품별 체류 시간** | 세션 내 작품 화면 활성 시간 | ARTWORK_INTERACTION | 몰입도 측정 |
| **촬영 횟수** | 셔터 이벤트 기록 | ARTWORK_INTERACTION | 시각적 관심도 |
| **좋아요·저장** | 버튼 탭 이벤트 | ARTWORK_INTERACTION | 선호도 측정 |
| **감정 태그 선택** | Chip UI 탭 이벤트 | AI_CONVERSATION | 감정 데이터 |
| **AI 대화 횟수·깊이** | 메시지 수·depth_score | AI_CONVERSATION | 지적 몰입도 |
| **이탈 구간** | 세션 종료 시점 작품 위치 | SESSION | 전시 동선 개선 |
| **AI Push 반응** | Push 수신 후 Level 2 진입 여부 | SESSION | 전환 트리거 효과 |

---

## 3. 화면 구성 (탭 구조)

### 탭 1. 실시간 현황

- 현재 전시 중인 관람객 수 (실시간)
- 오늘 총 방문자 / 누적 방문자
- QR 스캔 타임라인 차트 (시간대별)
- 현재 가장 많은 관람객이 있는 섹션 표시
### 탭 2. 작품별 반응 히트맵

- 공간 구조 위에 작품별 반응 강도 색상으로 표시
- 클릭 시 작품별 상세 데이터 펼침
    - 평균 체류 시간 / 촬영 횟수 / 좋아요 수
    - AI 대화 진입률 / 평균 대화 깊이 점수
    - 가장 많이 선택된 감정 태그 TOP 5
### 탭 3. 키워드 & 감정 분석

- 전체 감정 태그 빈도 랭킹 (막대 차트)
- 카테고리별 분류 (감정 / 분위기 / 매체 반응 / 서사)
- 시간대별 키워드 트렌드 변화
- 세그먼트별 감정 분포 비교 (연령대·방문 요일 등)
### 탭 4. 세그먼트 분석

| 세그먼트 축 | 분류 방식 | 활용 |
| --- | --- | --- |
| **반응 유형** | 감성형(감정 중심) / 탐구형(질문 중심) / 소비형(촬영 중심) | 컨셉 카드 타겟 설정 |
| **몰입 깊이** | 라이트(QR만) / 미드(Level 1) / 딥(Level 2 진입) | 팝업 전환 가능성 예측 |
| **선호 축** | 작가 관심 / 서사 관심 / 감각 관심 / 기술 관심 | Bridge 트리거 메시지 개인화 |

> 💡 세그먼트는 Phase 4 컨셉 카드에 "데이터 근거"로 자동 연결됩니다. 예: "감성형 세그먼트 42% — 감정/분위기 확장형 컨셉 적합도 높음"

---

## 4. 데이터 수집 시작 조건

- Phase 2 완료 (최소 조건 충족) 후 "전시 오픈" 버튼 클릭 시 자동 활성화
- B2C QR 코드가 자동으로 활성 상태로 전환
- 데이터 수집 기간 = 전시 기간 (Phase 1에서 설정한 날짜 기준)
---

## 5. Phase 4 진입 조건

| 조건 | 최소 (MVP) | 권장 |
| --- | --- | --- |
| 세션 수 | 50개 이상 | 200개 이상 |
| 감정 태그 수집 | 100개 이상 | 500개 이상 |
| AI 대화 수 | 30개 이상 | 100개 이상 |

최소 조건 충족 시 Phase 4 "스핀오프 자동 기획" 버튼 활성화.

전시 오픈 후 **최소 3~7일 데이터 축적 권장** (전시 기간의 10% 이상).

---

## 6. 검토 의견 & 보강 제안

**1. 데이터 수집 동의(Consent) 관리**

B2C 앱 온보딩에서 받은 consent_scope와 실제 수집 항목이 일치해야 합니다. 운영자 화면에서 동의 항목별 수집 현황을 확인할 수 있어야 하며, 미동의 데이터는 집계에서 자동 제외되어야 합니다.

**2. 최소 데이터 없을 때 Phase 4 진입 처리**

전시 초반이나 관람객이 적은 경우 Phase 4 진입 조건을 충족하지 못할 수 있습니다. 이 경우 "데이터 부족 시 AI 일반 추천 모드"로 Phase 4에 진입하는 대안 플로우가 필요합니다.

**3. 실시간 vs 배치 처리 구분**

탭 1(실시간 현황)은 스트리밍 처리, 탭 2~4(분석)는 배치 처리(1시간 단위 갱신)로 구분하는 것이 현실적입니다. 운영자에게 "마지막 업데이트 시각"을 명시해야 합니다.

**4. 알림 기능 필요**

운영자가 대시보드를 항상 열어두지 않으므로, 주요 이벤트(특정 작품 체류 급증, 이탈 구간 발생 등) 발생 시 이메일·인앱 알림 발송 기능이 필요합니다.
