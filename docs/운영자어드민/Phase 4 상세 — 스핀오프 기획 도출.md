# Phase 4 상세 — 스핀오프 기획 도출

> Phase 3에서 축적된 관람 데이터를 기반으로 AI가 스핀오프 팝업 컨셉을 자동 제안하는 단계입니다. 운영자는 검토하고 선택하고 승인만 합니다. 흰 종이(Blank Page)에서 시작하지 않습니다.

---

## 1. Phase 목표

관람 데이터 기반으로 24시간 내에 스핀오프 팝업 기획안 초안을 도출하는 것. 단순히 운영 시간을 줄이는 것을 넘어, **관람객이 실제 반응한 데이터에서 나온 컨셉**이기 때문에 전환율도 함께 높아지는 구조.

## 2. 컨셉 카드 구조 (Anatomy)

운영자가 받아보는 카드 1장에 담기는 정보입니다.

| 섹션 | 내용 | 목적 |
| --- | --- | --- |
| **헤더** | 컨셉명 + 유형 태그 + 운영 난이도 ★☆☆ | 빠른 식별 |
| **한 줄 요약** | 예: "전시의 고요한 감성을 향기로 소장하는 센소리 팝업" | 컨셉 즉시 이해 |
| **데이터 근거** | 예: "관람객 34%가 '고요한·평온한' 키워드 반응 / 해당 섹션 평균 체류 11분 (전체 평균의 1.8배)" | 신뢰도 확보 |
| **관람객 가치 훅** | 예: "전시가 남긴 감정을 오감으로 소장할 수 있는 공간" | B2C 전환 근거 |
| **Bridge 트리거 문구** | 예: "'고요한'이라는 단어를 3번 말씀하셨어요. 그 감각을 향기로 담아드릴게요." | 개인화 초대 메커니즘 |
| **예상 수익 시뮬** | 전환율 시나리오 3단계(보수/기본/낙관) × 객단가 | 의사결정 근거 |
| **추천 MD 미리보기** | 3종 썸네일 + "+ 더 보기" | 굿즈 방향성 빠른 확인 |
| **운영 준비 기간** | 예: "준비 기간 3주 / 공간 구조 변경 불필요" | 실행 가능성 즉시 판단 |
| **액션 버튼** | [이 컨셉 선택] [변형안 보기] [저장] | 다음 단계 진입 |

## 3. 6가지 컨셉 유형 × B2C 가치 레이어

| 컨셉 유형 | 관람객 욕구 | 가치 훅 | Bridge 트리거 문구 예시 |
| --- | --- | --- | --- |
| **① 작가 확장형** | 작가 세계에 더 깊이 들어가고 싶다 | 전시장 밖에서 작가의 작업실을 직접 경험 | "S님이 가장 오래 머문 작품 — 그 작가가 직접 설계한 공간이 있어요." |
| **② 서사 확장형** | 전시 이야기가 끝나지 않았으면 좋겠다 | 미완으로 끝난 이야기를 내가 직접 완성 | "오늘 탐험한 심연의 끝은 아직 열려있어요. 팝업에서 결말을 직접 써보세요." |
| **③ 감정/분위기 확장형** | 전시에서 느낀 감정을 오래 붙잡고 싶다 | 전시가 남긴 감정을 오감으로 소장 | "'고요한'이라는 단어를 3번 말씀하셨어요. 그 감각을 향기로 담아드릴게요." |
| **④ 매체 확장형** | 미디어아트 기술을 직접 만져보고 싶다 | 관람객에서 창작자로 — 기술을 직접 조작 | "S님이 촬영한 빛의 굴절, AI가 당신만의 작품으로 렌더링해드릴게요." |
| **⑤ 테마 재구성형** | 같은 주제를 다른 감각으로 경험하고 싶다 | 눈으로 본 것을 미각·청각·촉각으로 재경험 | "오늘 눈으로 느낀 심해의 온도 — 이번엔 혀끝으로 경험해보세요." |
| **⑥ 개인화 추천형** | 나만을 위한 무언가를 갖고 싶다 | 내 관람 데이터로 만들어진 세상에 하나뿐인 굿즈 | "오늘 S님의 관람 여정으로 만들어진 단 하나의 아카이브가 준비됐어요." |

## 4. 컨셉 카드 탐색 & 선택 UX

| 단계 | 인터페이스 | 세부 내용 |
| --- | --- | --- |
| **기본 뷰** | 카드 3종 나란히 (추천순) | AI가 데이터 적합도 기준으로 정렬. 좌측 상단 카드 = 최우선 추천 |
| **정렬 필터** | 수익 중심 / 체험 중심 / 빠른 실행 | 운영자의 이번 팝업 우선순위에 따라 정렬 순서 변경 |
| **카드 확장** | 카드 클릭 → 풀스크린 상세 뷰 | 데이터 근거 전체 / 공간 레이아웃 초안 / MD 전체 리스트 확인 |
| **저장(핀)** | ♡ 저장 버튼 | 결정 보류 상태로 복수 카드 저장 후 나중에 비교 |
| **변형 요청** | [변형안 보기] 클릭 | 같은 유형 내에서 다른 방향성의 서브 컨셉 3종 추가 생성 |
| **커스텀 추가** | [+ 직접 추가] | 운영자 아이디어를 카드 형태로 직접 입력 → AI가 MD·Bridge 문구 자동 완성 |

## 5. 컨셉 카드 확장 3가지 경로

**① 운영자 직접 추가**

- "+커스텀 컨셉" 버튼 → 타이틀·방향성 입력
- AI가 추천 MD 5종 + Bridge 트리거 문구 자동 완성 제안
- 운영자 검토 후 확정
**② AI 데이터 기반 자동 추가**

- 관람 데이터에서 기존 6종에 없는 새 반응 패턴 감지 시 "새 컨셉 감지됨" 알림
- 운영자 승인 후 신규 카드 추가
- 예: "관람객 28%가 사운드에 집중 반응 → '청각 확장형' 신규 컨셉 제안"
**③ 운영 중 컨셉 진화**

- 팝업 운영 중 유입되는 신규 데이터로 기존 카드 자동 정교화
- 운영자에게 "컨셉 카드 업데이트 가능" 알림 발송
- 승인 시 Bridge 트리거 메시지·MD 리스트 자동 갱신

## 6. 선택 후 분기 플로우

몇 개를, 어떤 방식으로 선택하느냐에 따라 다음 단계가 달라집니다.

| 선택 패턴 | AI 대응 | 결과물 |
| --- | --- | --- |
| **1종 단독 선택** | 해당 컨셉 기반 상세 기획안 즉시 생성 | 단일 컨셉 스핀오프 운영 가이드북 |
| **2종 복합 선택** | "메인 컨셉 + 서브존" 배치 기획안 생성 | 메인 체험 존 + 굿즈 판매 존 2-레이어 구조 기획안 |
| **미결정 상태** | "변형안 3종 더 생성" 또는 "운영자 직접 커스텀" 분기 제안 | 탐색 계속 또는 빈 카드에서 직접 작성 |
| **전체 재생성 요청** | 수집 기간 또는 세그먼트 기준 변경 후 재분석 | 새로운 조건 기반 컨셉 카드 전체 교체 |

> 💡 **2종 복합 선택 예시** — "감정/분위기 확장형" (센소리 라운지) + "개인화 추천형" (커스텀 MD 스테이션)을 함께 선택하면, AI가 공간을 두 존으로 나눠 동선까지 설계한 복합 기획안을 출력합니다.

## 7. 선택 후 자동 생성 산출물

컨셉 1~2종 선택 후 "상세 기획안 생성" 클릭 시 AI가 자동으로 생성합니다.

| 산출물 | 내용 | 활용처 |
| --- | --- | --- |
| **공간 레이아웃 초안** | 존 배치 텍스트 설계 (입구/체험/구매/출구 동선) | 현장 세팅 기준 |
| **커스텀 MD 리스트** | 컨셉·데이터 기반 추천 굿즈 5~8종 + 예상 단가 | MD Matcher 연동 |
| **Bridge 트리거 메시지 세트** | 세그먼트별 개인화 초대 문구 3종 변형 | 관람 종료 시 자동 발송용 |
| **현장 운영 시나리오** | 스태프 응대 스크립트 + 고객 동선 안내 | Ops Automator 연동 |
| **마케팅 카피 초안** | SNS 홍보 문구 3종 + 팝업 소개 1문단 | 기획사 자체 마케팅 활용 |
| **예상 수익 상세 시뮬** | 전환율 시나리오 3단계(보수/기본/낙관) × 객단가 | 내부 보고용 |
| **제안서 초안** | 제안 제목·배경·근거 데이터·컨셉안 자동 구성 (PDF 출력) | 사내 보고 및 파트너 공유 |

## 8. 운영자 경험 원칙: "검토하고 승인만 하세요"

컨셉 제안 방식 전반에 흐르는 핵심 UX 원칙입니다.

- **Zero-Draft 원칙**: 운영자는 절대 빈 화면에서 시작하지 않습니다. AI가 항상 0번째 초안을 먼저 내놓습니다.
- **근거 선행 원칙**: 모든 제안에는 "왜"가 먼저 나옵니다. 데이터 근거 없는 컨셉 카드는 표시되지 않습니다.
- **수정 최소화 원칙**: 운영자의 개입은 선택, 승인, 미세 조정에 집중됩니다. 처음부터 작성하는 일이 없어야 합니다.
- **관람객 가치 내장 원칙**: 모든 카드에는 B2C 가치 훅이 포함됩니다. 운영자가 "이 컨셉으로 관람객이 올까?"를 카드 안에서 바로 판단할 수 있어야 합니다.

## 9. 검토 의견 & 보강 제안

**1. 수익 시뮬레이션 근거 투명화 — 투자사 설득용**

단계별 목표 수치를 현실적으로 조정하고 (MVP: 5%, v1.0: 15%, v2.0: 20%), 아르떼뮤지엄 TEA BAR·팀랩 등 실제 사례 기반 시뮬레이션 근거 섹션을 제안서에 포함시켜야 합니다. 추상적인 %보다 "월 관람객 1만 명 × 전환율 10% × 객단가 1.5만원 = 월 1,500만원" 형태의 구체적 수치가 설득력 있습니다.

**2. 데이터 없을 때 컨셉 카드 품질 저하 문제**

Phase 3 데이터가 부족한 전시 초반에는 컨셉 카드가 "일반 추천" 수준으로 제공됩니다. 운영자에게 이 사실을 명시하고, 데이터 축적 후 "컨셉 카드 고도화" 알림을 보내는 플로우가 필요합니다.

**3. 제안서 공유 권한 관리**

제안서 초안은 Tier 2(고객사 어드민)까지만 공유되어야 합니다. 외부 공유 시 워터마크 또는 만료 링크 방식 고려 필요.

**4. 컨셉 확정 후 취소·변경 정책**

컨셉을 확정한 후 Bridge 트리거가 발동되기 전에 변경하는 경우의 처리 정책이 필요합니다. (예: 전시 운영 중 컨셉 변경 시 이미 발송된 초대장 처리 방법)
