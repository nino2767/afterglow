# AFTERGLOW External Document Styling & Confirmation Skill

이 스킬은 대외용 서비스 소개서, IR 스펙, 제안서 등의 HTML 문서를 변환하거나 생성할 때 사용자의 디자인 선호도 컨펌을 받고 명확한 디자인 토큰을 적용하기 위한 지침입니다.

## 1. ⚠️ 필수 사전 컨펌 규칙 (Mandatory Pre-confirmation)
사용자로부터 대외용 HTML 문서 제작 요청을 받으면, **자체 판단하여 임의의 스타일로 바로 작성하지 말고, 반드시 작성 착수 전 사용자에게 1안과 2안 중 어떤 디자인을 사용할지 먼저 질문하고 컨펌을 받아야 합니다.**

### 🎨 대외용 디자인 2대 스타일 옵션
1. **대외용 1안 (Nocturne · 딥 다크 &amp; 웜 어센트)**
   * **특징:** 딥 다크 모드 그라데이션 헤더, 라운드 카드 모듈, 은은하고 감성적인 라인과 웜 어센트 하이라이트.
   * **추천:** 브랜드 비전, 감성적 스토리텔링, B2C 서비스 경험 강조 문서.
2. **대외용 2안 (Industry · 바우하우스 굵은 헤더 &amp; 블루프린트 코너)**
   * **특징:** `#1d2d3d` 다크 네이비 톤 배너, 52px Barlow Condensed 대담한 타이틀, 십자선 모퉁이 코너(Blueprint) 및 고대비 인포그래픽.
   * **추천:** B2B 아키텍처, 데이터 파이프라인, 엔지니어링 스펙, 정산 모델 중심 문서.

---

## 2. 렌더링 무결성 작성 가이드 (Direct Embed Style)
* 외부 스크립트나 커스텀 태그(`<x-dc>`, `<helmet>`) 의존 시 로컬 파일 열람 환경에서 스타일 미적용 현상이 발생할 수 있습니다.
* 반드시 선택된 안의 디자인 토큰과 핵심 CSS 클래스(`.blueprint`, `.table`, `.tag` 등)를 HTML `<head>` 내 `<style>` 태그에 **직접 내장(Direct Embed)**하여 작성하여 어떤 환경에서도 100% 원본 디자인이 선명하게 표출되도록 합니다.
